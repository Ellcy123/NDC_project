# 证据 / 道具 / 环境叙事规则

改 `ItemStaticData` 前必须读本文件。这里是编辑器配置的证据分类规则，尤其用于判断 `itemType`、英文内部名、资源名和美术需求写法。

## ItemStaticData.itemType

| itemType | 英文类型 | 中文 | 收集 / 背包 | 规则 |
|---|---|---|---|---|
| 1 | clue | 线索 | 拍照取证后进入证据栏 / 背包类 UI | 现场痕迹、现场状态、不可直接带走的证据。美术需求描述“痕迹 / 物件状态”，不要写成照片素材。 |
| 2 | envir | 环境叙事 | 不收集，不进背包 | 只承担氛围、背景、情绪暗示或非直接证据功能。可以点击查看，但不作为可持有证据。 |
| 3 | item | 道具 / 实物 / 文件 | 可获得，可进入背包 | 可带走的实物、文件、报告、纸条、证物袋、工具等。 |

## Unit 大纲标记

- `🟣` / `Envir` / `环境叙事`：必须配置为 `itemType = "2"`。
- `🟢 ... - Clue`：通常配置为 `itemType = "1"`。
- `🟢 ... - Item` / `文件` / `实物` / `报告`：通常配置为 `itemType = "3"`。
- 若大纲、证据美术表、场景文档冲突：先以更细的证据美术表 / 场景文档为准；仍冲突时先问用户，不硬猜。

## envir 规则

- `envir` 不配置 `iconPath`；如果已有 `iconPath`，清空或删除。
- `envir` 不应进入 `DoubtConfig.condition` 或 `ExposeData.item`。它可以作为场景可点击对象挂在 `SceneConfig.ItemIDs[]` 下。
- `envir` 的 `canAnalyzed` / `canCombined` 必须为 `"false"`。
- `ArtRequirement` 写环境画面、氛围暗示、情绪信息，不写“关键证据 / 指证主力”。

## 命名规则

英文名或资源名要带类型前缀，避免只看英文时分不清收集方式：

- `clue_wall_scratches`
- `item_nov20_evening_paper`
- `envir_foster_pipe_medical_photo`

推荐字段：

- `Name[1]`：使用短英文内部名，前缀为 `clue_` / `item_` / `envir_`。
- `desSpritePath` / `mapSpritePath`：资源名同样使用 `clue` / `item` / `envir`，如 `SC3005_envir_3109_big`。
- `folderPath` 仍按场景 / 章节目录组织，不强制带类型。

## Clue 美术需求规则

`clue` 在游戏内 UI 通常可以用照片卡承载，但美术需求不要写成“黑白照片 / 拍摄照片 / 10x15 / 标注”。应描述场景里必须画出的痕迹或物件状态：

- 写：外墙抓痕、鞋印方向、油渍位置、压痕形状。
- 不写：黑白照片、照片旁标尺、现场照片、拍摄角度。
- 本体确实是照片 / 报告 / 文件的条目除外。

## 通用证据规则

### ID 编码

- EPI01 = 1xxx（loop1=11xx, loop2=12xx, ..., 派生=17xx）。
- EPI02 = 2xxx（loop1=21xx, loop2=22xx, ..., 派生=27xx）。
- EPI03 = 3xxx（loop1=31xx, loop2=32xx, ..., 派生=37xx）。

### 物理属性

- 所有证据的物理属性（尺寸、时间、距离、品牌、价格）必须前后一致。
- 跨循环引用时不能出现数值矛盾。

### 获取链

- 直接获取：对话 `script:"get"` 或场景物品点击。
- 分析派生：对直接证据执行“分析”。
- 合成派生：两个证据组合。

### note 字段

- `关键——{用途}`：Expose 关键证据。
- `场景道具` / `环境叙事`：环境性证据。
- 空：普通证据。

### 核心原则

- 单一证据不定案（推荐但不强制）：关键结论推荐用两种不同来源的信息交叉验证，以增强推理厚度；单件强证据即可定案的情形也允许，由设计师判断。
- 证据获取时间不能早于其存在时间。
- 证词原文与提取摘要信息密度对等，不丢失关键限定词。

### 疑点挂载

详见 `docs/游戏系统/核心玩法/疑点系统.md`。

- 疑点 condition 优先 2 件双来源交叉，允许 1 件，禁止 >= 3 件。
- 碎片 condition 严格 1 件，2-3 个碎片合并成疑点。
- 一对一：每件证据 / 证言只能挂在唯一一个疑点 / 碎片的 condition 里。
- 时序：疑点 / 碎片的出现 Loop <= 它 condition 里任何证据在 expose 里使用的最早 Loop。
