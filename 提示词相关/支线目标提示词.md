支线任务设计工作提示词
📋 任务背景
我需要为NDC游戏设计支线任务系统。游戏分为多个循环，每个循环有主线任务。支线任务用于丰富叙事、角色深度和玩家体验。
🎯 任务系统结构
三大类型
1. A类：角色动机探索
  - 触发：玩家发现角色第一个私人物品时
  - 完成：循环结束时自动完成
  - 作用：帮助玩家理解角色的背景和动机
2. B类：隐藏真相追踪
  - 触发：发现第一条相关线索时
  - 完成：整个Episode结束时完成
  - 作用：跨循环追踪，制造悬念，为后续Episode铺垫
3. C类：表面疑点追踪
  - 触发：玩家听到证词或发现现象时产生疑问
  - 完成：循环结束时自动解答
  - 作用：引导玩家注意关键疑点，循环结束时主线解答
⚠️ 关键设计原则
触发条件逻辑
- ✅ 正确：触发时刻是玩家产生疑问的瞬间
- ❌ 错误：触发时刻是发现答案证据的瞬间
- 例子： 
  - ✅ 正确：听到Rosa说在地下室 → 触发"Rosa当晚在哪里工作？"
  - ❌ 错误：发现工作记录卡证明她撒谎 → 触发任务
任务命名原则
- ✅ 使用中性疑问句，不引导答案方向
  - 正确："11:30的枪声是什么？"
  - 错误："11:30真的是死亡时间吗？"（暗示答案是"不是"）
- ✅ 不透露玩家当前不知道的信息
  - 正确："Morrison作为警探为何欠下欠款？"（发现欠条后触发）
  - 错误："Morrison为何从英雄变成腐败警官？"（玩家还不知道他腐败）
- ✅ A类任务基于已发现的物品提问
  - 正确：发现赌债欠条 → "Morrison作为警探为何欠下欠款？"
  - 错误：发现女儿照片 → "Rosa为何冒险说谎？"（玩家还不知道她说谎）
触发条件与任务名的对应关系
- 触发条件必须是玩家刚好能产生这个疑问的时机
- 任务名必须是玩家在那个时机会自然产生的疑问
- 例子： 
  - 发现女儿照片 → "Rosa背负着什么压力？"（看到照片自然好奇她的压力）
  - 发现工作记录卡 → 配合C类任务"Rosa当晚在哪里工作？"

🔍 自检清单
输出前请自检：
1. [ ] A类触发条件是否为"发现第一个角色物品"
2. [ ] C类触发条件是否为"产生疑问的时刻"而非"发现答案的时刻"
3. [ ] 任务名称是否中性，不引导答案方向
4. [ ] 任务名称是否包含玩家尚未知道的信息
5. [ ] B类任务是否标注为"Episode结束"完成
6. [ ] 所有A/C类任务是否标注为"循环结束"完成
