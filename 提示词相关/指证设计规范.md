# NDC游戏指证系统设计规范(Markdown版)

## 一、核心设计理念

**引用来源**:`NDC游戏策划0620.docx` - "游戏的特色"部分

指证系统是NDC游戏的核心玩法,目标是实现类似《逆转裁判》和《弹丸论破》的游戏推进体验:

1. **核心目标**:揭穿NPC的谎言,通过剧情演绎增强玩家爽快感
2. **终极体验**:对最终凶手进行连续指证,将他的谎言逐一击穿,形成类似ACT游戏"连击"的爽快感

## 二、指证流程核心机制

**引用来源**:`NDC Episode 1 循环3/4/5 场景证据布局设计.docx`中的指证系统设计部分

### 2.1 谎言层级递进机制

每个嫌疑人的指证流程必须设计为**多层谎言防御体系**(通常3-4轮),具体结构如下:

```
初始谎言(第1层) 
   ↓ 
玩家用证据A部分驳倒 
   ↓ 
嫌疑人被迫编造新谎言(第2层,有更严重漏洞) 
   ↓ 
玩家用证据B+C继续攻击漏洞 
   ↓ 
嫌疑人继续防御/掩盖(第3层,防线崩溃)
   ↓ 
玩家用证据D最终击穿 
   ↓ 
嫌疑人退无可退,被迫承认真相
```

### 2.2 证据使用节奏

**引用来源**:`NDC Episode 1 循环4` - Jimmy三层防御拆解设计

每一轮指证的证据配置规则:

| 指证轮次 | 证据数量 | 证据作用 | 逻辑矛盾类型 |
|---------|---------|---------|------------|
| 第1轮 | 1-2个核心证据 | 戳破表层谎言 | 直接矛盾(事实对比) |
| 第2轮 | 2-3个证据组合 | 攻击新谎言的逻辑漏洞 | 间接矛盾(行为不合理) |
| 第3轮 | 1个关键证据 | 彻底击穿防线 | 致命矛盾(无法解释) |
| 最终轮 | 证据链总结 | 迫使承认 | 系统性崩溃 |

## 三、具体设计模板

### 3.1 三轮指证模板(参考循环3-Tommy)

**引用来源**:`NDC Episode 1 循环3` - Tommy古董密码网络指证

#### 第一轮指证:表层谎言戳破

```markdown
**初始谎言内容**:[嫌疑人声称的虚假身份/事件/时间]

**需要证据**:
- 证据A:[证据编号] - [证据名称] 
- 证据B:[证据编号] - [证据名称]

**指证过程**:
1. Zack展示证据A,质问矛盾点
2. 嫌疑人试图解释/抵赖
3. Zack追加证据B,形成证据链
4. **逻辑矛盾暴露**:[具体描述嫌疑人前后矛盾的地方]

**嫌疑人被迫升级谎言**:
[编造一个新的说法来掩盖,但这个新说法有更明显的破绽]
```

#### 第二轮指证:新谎言漏洞攻击

```markdown
**嫌疑人新谎言**:[第一轮被戳穿后编造的新说法]

**需要证据**:
- 证据C:[证据编号] - [证据名称]
- 证据D:[证据编号] - [证据名称] 
- (可选)证据E:[补充证据]

**指证过程**:
1. Zack展示证据C,指出新谎言的不合理之处
2. 嫌疑人开始慌张,试图辩解
3. **逻辑矛盾暴露**:[新谎言与已知事实的冲突]
4. Zack追加证据D,进一步压迫

**嫌疑人再次防御**:
[勉强编造第三层说法,但已经破绽百出]
```

#### 第三轮指证:防线彻底崩溃

```markdown
**嫌疑人最后防线**:[已经自相矛盾的辩解]

**需要证据**:
- 证据F:[决定性证据] - 无法辩驳的致命证据

**指证过程**:
1. Zack展示致命证据F
2. 嫌疑人脸色剧变/彻底慌乱
3. **防线崩溃**:[嫌疑人无法自圆其说]
4. Zack步步紧逼,直击核心问题

**嫌疑人最终妥协**:
[被迫承认部分或全部真相,但可能仍有保留]
```

## 四、游戏交互设计

**引用来源**:`NDC游戏策划0620.docx` - "INVESTIGATE 调查指证"部分

### 4.1 EXPOSE指证操作流程

```
1. 玩家在INVESTIGATE面板选择嫌疑人
2. 选择嫌疑人的某条陈述(时间/身份/事件)
3. 点击EXPOSE按钮
4. 从证据栏选择1-3个证据进行指证
5. 系统判定:
   - 成功 → 显示"LIE EXPOSED!"红色印章
   - 失败 → 显示"evidence insufficient to expose the lie"
6. 成功后NPC头像显示感叹号,可前往场景观看指证剧情
```

### 4.2 指证成功反馈

**引用来源**:`NDC游戏策划0620.docx` - "指证成功"模块

- **时长**:每次指证成功后的剧情演绎约36秒
- **内容**:
  - Emma的正向反馈(夸奖/总结)
  - 嫌疑人认输的剧情演出
  - 揭露新的游戏内容(新场景/新NPC/新证据)

## 五、设计检查清单

在设计每个循环的指证流程时,必须确保:

- [ ] **谎言层级**:至少3层谎言,每层被戳穿后嫌疑人被迫升级
- [ ] **证据配置**:每轮指证有1-3个明确证据支撑
- [ ] **逻辑自洽**:每个谎言被戳穿后,新谎言的破绽更明显
- [ ] **节奏把控**:3-4轮指证总时长控制在5-6分钟
- [ ] **情感递进**:嫌疑人从自信→慌张→崩溃的情绪变化
- [ ] **玩家引导**:NORMAL模式下高亮矛盾点,EXPERT模式需玩家自行发现

## 六、禁止事项

**引用来源**:基于`NDC游戏策划0620.docx`核心玩法总结

❌ **禁止一击必杀**:不能用一个证据直接让嫌疑人认罪,必须有递进过程

❌ **禁止证据堆砌**:不能要求玩家一次性选择5个以上证据

❌ **禁止逻辑跳跃**:每轮指证必须基于上一轮的信息递进

❌ **禁止无效防御**:嫌疑人的新谎言不能是毫无意义的重复

❌ **禁止剧透性任务**:指证前的任务描述不能透露谎言内容

## 七、证据链条设计原则

**引用来源**:基于指证玩法的核心机制总结

### 7.1 常规指证 vs 终极指证的本质区别

指证玩法的核心特色在于**渐进式真相揭露**,但不同类型的指证对证据完整性的要求存在重要差异:

#### 常规指证(循环中的嫌疑人指证)

**核心机制**:
- 玩家可能**并不拥有完全确凿的直接证据**
- 依靠**对话战术**击破谎言,诱导嫌疑人产生说话漏洞
- 通过**语言压力**和**心理攻势**的双重作用,迫使对方在矛盾中暴露真相
- 强调**逻辑推理**和**心理博弈**的过程

**证据要求**:
- 证据可以是**间接证据**或**部分证据**
- 允许通过**证据组合**和**逻辑推理**形成压力
- 重点在于**击破谎言的过程**,而非直接证明犯罪

**玩法体验**:
- 类似《逆转裁判》的法庭辩论
- 享受层层剥开谎言的智力对抗
- 在不完全信息下进行推理的成就感

---

#### 终极指证(每章最后的凶案指证)

**核心要求**:
- ⚠️ **玩家必然已经拥有完美的证据链条**
- 所有关键环节都有**实质性证据支撑**
- 证据链条必须**形成闭环**,无懈可击

**证据标准**:
- 必须包含**直接证据**(物证/确凿证词/不可辩驳的记录)
- 证据之间能够形成**完整的逻辑链**
- 每个推理步骤都有对应证据,**不存在逻辑跳跃**

**设计目标**:
- 给玩家"铁证如山"的**正义伸张感**
- 最终对决不是靠话术,而是靠**真相的力量**
- 实现类似ACT游戏的"连击终结技"般的**爽快感**

---

### 7.2 设计检查清单

在设计指证流程时,必须明确区分:

**常规指证设计时**:
- [ ] 证据可以是间接的,但逻辑链要完整
- [ ] 对话设计要有层层递进的压迫感
- [ ] 允许嫌疑人在压力下逐步暴露破绽

**终极指证设计时**:
- [ ] 确保拥有犯罪动机的直接证据
- [ ] 确保拥有犯罪手段的物证支撑
- [ ] 确保拥有时间/地点的确凿证词
- [ ] 所有证据形成完整闭环,凶手无法辩驳
- [ ] 不依赖"对话压力",仅凭证据即可定罪

---

### 7.3 案例对比

**常规指证案例**(循环3 - Tommy古董网络):
- 谎言1:"只做酒类生意" → 证据:财务报告**矛盾**(间接)
- 谎言2:"古董是正常买卖" → 证据:现场照片**不合理**(推理)
- 谎言3:"暂时寄存保管" → 证据:档案编号**显示系统性**(逻辑)
- **特点**:每轮证据都在**制造矛盾**,迫使升级谎言

**终极指证案例**(Episode 1 - 最终凶手):
- 动机:受害者日记+财务记录 → **直接证明仇恨/利益**
- 手段:作案工具+指纹/DNA → **物证确凿**
- 时间:监控录像+目击证词 → **时间线吻合**
- 掩饰:伪造证据的破绽 → **反向证明有罪**
- **特点**:每个环节都有**铁证**,形成无懈可击的证据链
