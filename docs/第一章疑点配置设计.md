# 第一章疑点配置设计

## 设计原则

- 疑点标题以**问号结尾**，暗示玩家"这里有问题"，而非直接揭示答案
- 疑点描述提供线索，引导玩家思考

### 疑点分类逻辑

疑点按**解决时机**分为两类：

| 分类 | 说明 | 作用 |
|------|------|------|
| **本循环解决** | 在当前循环发现并解答，用于解锁当前指证 | 推动当前剧情 |
| **跨循环悬疑** | 在当前循环发现，但要到后续循环才能解答 | 埋伏笔、维持悬念 |

### 跨循环悬疑线索链

```
Whale身份之谜（贯穿全章的核心悬疑）：
├── Loop2：Whale是谁？（发现Whale给Morrison的纸条）
├── Loop3：Webb日记里的"大鱼"是谁？（Whale被标记为DANGER）
├── Loop4：Jimmy家的Whale纸条意味着什么？（纸张纹样相同）
└── Loop6：Jimmy收了Whale多少钱？（最终揭示Whale收买了Jimmy）

Vivian的处境（为Loop5动机铺垫）：
├── Loop3：Tommy为什么同情Vivian？（暗示Vivian被迫做不光彩的事）
└── Loop5：Vivian的日记里写了什么？（揭示仇恨根源）

音乐掩护诡计（为Loop6真相铺垫）：
├── Loop5：为什么要把演出曲目换成最吵的？（表面指向Vivian预谋）
└── Loop6：11点的枪声为什么没人听到？（揭示真凶Jimmy利用了这一点）
```

### 本循环解决

#### DBT_C1L1_001：Rosa当晚究竟在哪工作？

| 字段 | 值 |
|------|-----|
| DoubtID | DBT_C1L1_001 |
| LoopID | loop1 |
| DoubtType | timeline |
| DoubtTitle | Rosa当晚究竟在哪工作？ |
| DoubtContent | Rosa说她案发时一直在地下室酒窖整理，但工作记录卡显示她23:00-01:00的工作地点是后台走廊。两个地方，同一时间，她不可能分身。 |

**触发条件**：
```json
{
  "type": "timeline_place",
  "required_testimonies": [
    "TESTI_ROSA_BASEMENT",
    "TESTI_ROSA_CORRIDOR"
  ],
  "trigger_when": "all_placed"
```

**触发流程**：
1. 玩家与Rosa对话 → 获得证词"案发时在地下室酒窖"
2. 玩家在储藏室找到工作记录卡 EV1115

4. 系统检测到同一时间段的地点矛盾 → 产出疑点

## 玩家体验设计

### 疑点标题的表述原则

| 原则 | 示例 |
|------|------|
| 以问号结尾 | "Rosa当晚究竟在哪工作？" |
| 暗示而非揭示 | "这条毛巾为什么有奇怪的味道？"（而非"毛巾上有氯仿"） |
| 引发好奇心 | "Whale是谁？" |
