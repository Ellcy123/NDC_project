Architecture Reference · v1.0

AI工作流架构全图

Project → Agent → Skill → Tool · 含横向状态层
L0 Project 项目层 整个工作流的边界与目标定义
定义
项目目标 & 约束
明确任务边界、成功标准、时间/资源限制。是整个系统的"宪法"，所有子层的行为都以此为准绳。
任务描述 验收标准 约束条件
输出
任务分解计划
将大目标拆成可分配给各Agent的子任务序列。决定并行 vs 串行，定义各子任务的输入/输出接口。
任务拓扑图 优先级排序 依赖关系
L1A Knowledge 知识层 静态结构化知识 · 外置于上下文
世界观库
Domain Knowledge
项目特有的背景知识，结构化存储，按需检索。避免塞入上下文窗口导致注意力稀释。
人物档案 设定文档 风格指南
向量检索
RAG 检索增强
将知识库向量化，Agent在生成前先检索相关片段注入上下文。是解决长上下文问题的核心手段之一。
Embedding 相似度检索 重排序
规则引擎
约束 & 规范
不依赖LLM判断的硬规则：格式规范、禁用词表、输出结构要求。用代码约束，不用prompt祈祷。
格式校验 禁用规则 模板引擎
L1B Memory 记忆层 动态状态 · 跨session持久化
工作记忆
Session State
当前任务执行中的短期状态。Agent完成一个节点后将关键决策写入，供下游Agent读取，不依赖上下文传递。
当前进度 中间结果 临时决策
长期记忆
Persistent Memory
跨项目/跨任务积累的经验。记录哪些策略有效、哪些失败过，让系统随时间自我改进。
成功案例 错误记录 用户偏好
状态压缩
Summarization
长链条执行中主动摘要历史、归档旧状态，防止Memory层膨胀。类似人类的"睡眠巩固"机制。
自动摘要 历史归档 优先级剪枝
L2 Agent 执行层 目标导向的自主执行单元 · 含调度与协作
调度者
Orchestrator Agent
整个Agent网络的大脑。接收Project层的任务计划，将其拆解为具体指令分发给Worker。监控执行进度，处理异常和失败重试。这是目前最脆弱的环节。
任务分发 进度监控 异常处理 重试策略
执行者
Worker Agents
负责具体的生产性任务。每个Worker专注单一领域，接收明确的输入，产出明确的输出。不做判断，只做执行——判断交给Orchestrator和Critic。
剧情写作 场景描述 对话生成 资产生成
评审者
Critic Agent
独立于Worker的质量把关层。检查输出是否满足知识层的约束（一致性、准确性、风格）。建议第一个搭建——输入输出最清晰，容错成本最低。
一致性检查 事实核验 风格审查 评分反馈
⚠ 关键设计原则
Agent之间通过结构化消息（JSON/Markdown）传递状态，而非自然语言。自然语言传递在多跳后必然产生语义漂移。Orchestrator的任务分解质量决定整个系统上限，这部分目前仍高度依赖prompt工程，是最需要投入的地方。
L3 Skill 技能层 有判断的复合能力 · Agent的行为库
生成类
Content Generation
结合知识层约束，生成符合风格与格式要求的内容。内含提示词模板和输出后处理逻辑。
对话生成 场景描述 旁白写作
检索类
Knowledge Retrieval
从知识层精准取回相关信息，构建精简的上下文窗口。核心在于检索策略——避免把整个知识库塞进去。
语义检索 关键词过滤 上下文组装
分析类
Analysis & Evaluation
对输入或输出进行结构化评估。返回有判断依据的评分和改进建议，而非模糊的"好"或"不好"。
情感分析 一致性评估 质量评分
转换类
Transform & Format
在不同格式、模态或粒度之间转换数据。例如将剧情大纲展开为对话，或将对话压缩为摘要。
格式转换 摘要提取 结构展开
Skill vs Tool 的核心区别
Skill = 有判断逻辑的复合行为（包含prompt策略 + 后处理 + 错误处理）。Tool = 无判断的原子执行（调用一个API，读一个文件）。混淆两者会导致调试时无法定位问题层级。
L4 Tool 工具层 无判断的原子执行 · 与外部世界的接口
📁 文件 & 存储
File R/W
读写本地/云端文档、脚本、配置文件
Vector DB
向量数据库读写（Pinecone / Chroma / Weaviate）
KV Store
键值缓存，存储Agent状态与中间结果
Git Ops
版本控制，追踪内容变更历史
🎨 生成 API
Image Gen
Midjourney / DALL-E / SD · 场景图与角色图
TTS / Voice
ElevenLabs / Fish Audio · NPC语音合成
SFX Gen
ElevenLabs SFX · 音效自动生成
Video Gen
Seedance / Kling · 动态预演与宣传素材
🔍 数据 & 检索
Web Search
实时检索史料、市场数据、竞品信息
Web Fetch
抓取指定页面完整内容进行深度分析
SQL Query
结构化数据库查询，如游戏事件日志
Embedding API
将文本转为向量，支撑RAG检索管道
🔗 外部集成
Code Exec
沙盒代码执行，处理数据、生成图表
Unity Bridge
与Unity编辑器交互，自动化场景配置
Notification
Slack / 邮件 · 任务完成或异常告警
LLM API
调用Claude / GPT-4 等模型完成推理

L0 · Project
《凝视暗影》第X章制作
目标：完成第三章剧情 + 配套场景图 + NPC语音
L1A · Knowledge
Ashford世界观库
人物档案、1928-29年史料、Art Deco风格指南、已有剧情索引
L1B · Memory
创作状态追踪
已写章节摘要、角色当前状态、待解决的剧情矛盾列表

制作调度Agent
将"写第三章"拆成：大纲→场景分镜→对话→旁白→图像需求→语音需求

游戏专项技能
