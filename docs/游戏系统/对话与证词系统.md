# 对话与证词系统

## 一、对话链结构

每句对话是 `Talk` 表中的一条记录，通过 `next` 字段形成链式结构：

```
Talk[105001001] ──next→ Talk[105001002] ──next→ Talk[105001003] ──next→ ...
  Tommy: "谁？"      Zack: "我是Zack"      Tommy: "楼下出了大事"
```

### Talk ID 编码

- **普通对话**：`{NPC3位}{对话组3位}{句序3位}`，如 `105001001` = Tommy 第1组对话 第1句
- **指证对话**：`{轮次2位}{序号4位}`，如 `110001` = Loop1 指证第1句

## 二、对话脚本类型（script 字段）

| script 值 | 行为 | Parameters 用法 |
|-----------|------|-----------------|
| `""` (空) | 普通对话，自动跳 `next` | — |
| `"1"` / `"branches"` | **分支选择**，弹出 2-3 个选项 | ParameterStr=选项文本, ParameterInt=跳转 Talk ID |
| `"end"` | **对话结束** | — |
| `"get"` | **获取证据/道具** | ParameterInt=物品/证词 ID |
| `"expose"` / `"Lie"` | **指证谎言标记** | ParameterStr0=正确证据 ID（逗号分隔）, ParameterInt0=成功跳转 ID |
| `"change_scene"` | **切换场景** | ParameterStr=目标 sceneId |
| `"interrupt"` | **中断对话** | — |

## 三、分支对话实例

```json
{
  "id": "103001005",
  "Speaker": {"id": "101", "Name": ["扎克·布伦南", "Zack Brennan"]},
  "Words": ["放轻松，我就随便问问。"],
  "script": "branches",
  "Parameters": [
    {"ParameterStr": "有人能证明吗？", "ParameterInt": "103001006"},
    {"ParameterStr": "刚才发生了什么？", "ParameterInt": "103001012"},
    {"ParameterStr": "你在害怕什么？", "ParameterInt": "103001010"}
  ]
}
```

游戏中：
```
Zack: "放轻松，我就随便问问。"
  ❶ 有人能证明吗？    → Talk 103001006
  ❷ 刚才发生了什么？  → Talk 103001012
  ❸ 你在害怕什么？    → Talk 103001010
```

分支最终必须汇合到同一节点继续主线。

## 四、证据获取实例（script="get"）

```json
{
  "id": "103001004",
  "Speaker": {"id": "103"},
  "Words": ["没...没有！我一直在地下室走廊打扫...从十点开始...不，是九点半..."],
  "script": "get",
  "ParameterInt": "1031002"
}
```

→ 玩家自动获得 TestimonyItem 1031002「Rosa 从 9 点半开始在地下室走廊打扫」

## 五、证词系统

### 5.1 两层结构

```
Testimony（完整证词段落）
└── TestimonyItem（可提取的单条证据）× N
```

- **Testimony** = NPC 的完整陈述，含 `<special>` 标签高亮关键信息
- **TestimonyItem** = 从证词中提取的独立证据条目，可用于时间线、关系网和指证

### 5.2 实例：Rosa 的证词

**完整证词**（Testimony id=103001004）：
> 我是<special=1>蓝月亮的清洁工</special>，<special=2>9点半起在地下室走廊打扫</special>。地下室隔音好，什么都没听到。

**提取的 TestimonyItem**：

| ID | 类型 | 内容 | 触发方式 |
|----|------|------|----------|
| 1031001 | 自述(1) | Rosa 是蓝月亮的清洁工 | 关系网：需与 Rosa(103) 和 Tommy(105) 都谈过 |
| 1031002 | 自述(1) | Rosa 从 9 点半开始在地下室走廊打扫 | 时间线：场景 1103，21:30-25:00 |

### 5.3 证词类型（testimonyType）

| 值 | 名称 | 说明 |
|----|------|------|
| 1 | **SelfStatement（自述）** | NPC 对自己行为的陈述 |
| 2 | **Hearsay（见闻）** | NPC 对他人行为的描述 |

### 5.4 证词触发条件（triggerType）

证词需要满足特定条件才能"提取"为可用证据：

#### triggerType=0：None（无条件）
```json
{"triggerType": "0", "triggerParam": "103"}
```
直接从 NPC 103(Rosa) 获取，无需额外操作。

#### triggerType=1：Timeline（时间线验证）
```json
{"triggerType": "1", "triggerParam": "103,1103,2130,2500"}
```
格式：`NPC_ID, 场景ID, 开始时间(HHMM), 结束时间(HHMM)`
操作：在 INVESTIGATE → TIMELINE 面板中将证词拖到对应时间段。

#### triggerType=2：RelationNetwork（关系网交叉验证）
```json
{"triggerType": "2", "triggerParam": "103,105"}
```
格式：`NPC_ID_1, NPC_ID_2`
操作：在 INVESTIGATE → RELATIONSHIP 面板中将证词拖到 NPC 头像。

### 5.5 TestimonyItem ID 编码

格式：`{NPC 3位}{轮次 1位}{序号 3位}`

| ID | 含义 |
|----|------|
| 1031001 | Rosa(103) Loop1(1) 第 001 条 |
| 1051003 | Tommy(105) Loop1(1) 第 003 条 |
| 1052001 | Tommy(105) Loop2(2) 第 001 条 |

## 六、时间线系统

### 交互流程
1. 打开 INVESTIGATE → TIMELINE
2. 从证词列表拖拽时间相关证词到时间线面板
3. 证词转化为时间条（NPC 在何时何地）
4. 多条证词放入后系统自动检测：时间重叠、地点矛盾、时间空白
5. 满足条件 → 产出疑点

### triggerParam 格式
```
"103,1103,2130,2500"
  │    │     │     └── 结束时间 25:00（= 次日 01:00，24h+制）
  │    │     └── 开始时间 21:30
  │    └── 场景 ID 1103
  └── NPC ID 103
```

## 七、关系网系统

### 交互流程
1. 打开 INVESTIGATE → RELATIONSHIP
2. 将人物证词拖到 NPC 头像上
3. 匹配成功 → 身份确认（头像高亮 + 标签），可能发现异常关系
4. 匹配失败 → 提示 "这条证词似乎不属于这个人"

### 视觉状态
- 未确认身份：灰色边框
- 已确认身份：高亮边框 + 身份标签
- 发现关系疑点：连线变红 + 感叹号
