# 疑点系统

## 一、核心概念

**疑点（Doubt）** = 玩家通过主动整理信息后发现的矛盾点。

疑点是指证的前置条件：收集完本轮**全部当前目标疑点**后，才能解锁 EXPOSE 功能。

**设计哲学**：系统在后台做矛盾检测，玩家只需做拖拽/分析/合成操作。操作量高但思考负担低——玩家感觉自己在推理，但实际不会卡关。

## 二、五种获取途径

| # | 途径 | 操作 | 示例 |
|---|------|------|------|
| ① | 整理人物关系 | 将证词拖到 NPC 头像 | 发现 Vivian 和 Webb 的关系存疑 |
| ② | 整理时间线 | 将证词拖到时间线面板 | Rosa 的时间线与死亡时间重叠 |
| ③ | 获取物品 | 场景探索/对话获得关键物品 | 获得带血的工作记录卡 |
| ④ | 分析物品 | CASE BOARD 化验/指纹分析 | 毛巾上检测出氯仿残留 |
| ⑤ | 合成物品 | CASE BOARD 合并两个物品 | 医疗账单+工作记录→动机 |

## 三、疑点解锁条件（DoubtConfig 表）

每个疑点有一组 `condition`，**所有条件都满足**时疑点自动解锁：

```json
{
  "id": "疑点ID",
  "condition": [
    {"type": "条件类型", "param": "目标ID"}
  ],
  "text": "疑点文本"
}
```

| condition.type | 名称 | param 含义 |
|---------------|------|-----------|
| 1 | **Item（道具）** | 需拥有 ItemStaticData 中的指定物品 |
| 2 | **RelationNetwork（关系网）** | 需在关系网面板建立指定连接 |
| 3 | **Timeline（时间线）** | 需在时间线面板标记指定事件 |

## 四、完整实例：Loop1 的 4 个疑点

### 疑点 1101：Rosa 当晚究竟在哪工作？

```json
{"id": "1101", "condition": [{"type":"3","param":"1031002"}, {"type":"1","param":"1115"}], "text": "Rosa当晚究竟在哪工作？"}
```

**解锁路径**：
1. 与 Rosa 对话 → 获得证词 1031002「Rosa 从 9 点半在地下室走廊打扫」
2. 在时间线面板放入证词 → 满足 type=3
3. 在 Tommy 办公室获取物品 1115「工作排班表」→ 满足 type=1
4. 两条件满足 → **疑点解锁**（排班表显示 Rosa 不该在地下室）

### 疑点 1102：这条毛巾为什么有奇怪的味道？

```json
{"id": "1102", "condition": [{"type":"1","param":"1701"}, {"type":"1","param":"1121"}], "text": "这条毛巾为什么有奇怪的味道？"}
```

**解锁路径**：
1. 获取物品 1114「沾湿的毛巾」→ 分析 → 得到 1701「带有氯仿气味的毛巾」
2. 获取物品 1121「走廊清洁记录」
3. 两件物品都拥有 → **疑点解锁**

### 疑点 1103：是谁拖得我？

```json
{"id": "1103", "condition": [{"type":"1","param":"1702"}], "text": "是谁拖得我？"}
```

**解锁路径**：合成 1701 + 1121 → 得到 1702 → **疑点解锁**

### 疑点 1104：11 点 30 分的枪声是案发时间吗？

```json
{"id": "1104", "condition": [{"type":"3","param":"1051003"}], "text": "11点30分的枪声是案发时间吗？"}
```

**解锁路径**：与 Tommy 对话获得证词 1051003 → 放入时间线 → **疑点解锁**

## 五、疑点分类

| 分类 | 说明 | 作用 |
|------|------|------|
| **当前目标疑点** | 本轮指证必须收集 | 全部收集 → 解锁 EXPOSE 按钮 |
| **其他疑点** | 属于后续循环 | 进入对应循环后自动移入"当前目标" |

玩家可以提前收集后续循环的疑点，但系统不提示具体用途，产生"这个什么时候用？"的好奇心。

## 六、UI 表现

```
EXPOSE 按钮状态：
├── 疑点未收集完整 → 灰色，提示"收集更多疑点后可进行指证"
└── 疑点收集完整   → 红色，可点击进入指证
```

获得新疑点时左下角弹出提示：
```
┌──────────────────────┐
│ ！获得新疑点          │
│ Rosa的时间线存疑      │
└──────────────────────┘
```

## 七、疑点 ID 编码

格式：`{unit}{loop}{seq}`
- 1101 = Unit1 Loop1 疑点 #1
- 1203 = Unit1 Loop2 疑点 #3
- 2101 = Unit2 Loop1 疑点 #1
