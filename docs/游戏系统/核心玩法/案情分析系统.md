# 案情分析系统

## 一、核心概念

案情分析系统（Case Board）是玩家整理调查成果、进行推理操作的核心界面。它本身没有独立的配置表，而是汇聚多个系统的数据——已收集的物品（ItemStaticData）、已获取的证词（TestimonyItem）、已解锁的疑点（DoubtConfig）——提供统一的操作面板。

玩家在 Case Board 中完成三类推理操作：查看和加工证据、整理时间线、梳理人物关系。这些操作的结果会触发疑点解锁，进而推动指证环节的开启。

## 二、三个子面板

### EVIDENCE（证据清单）

展示玩家已收集的所有物品和证词，提供分析和合成功能。

| 功能 | 操作 | 数据来源 |
|------|------|----------|
| 物品浏览 | 查看已收集物品的详细描述和图片 | ItemStaticData（已收集的） |
| 物品分析 | 对 `canAnalyzed=true` 的道具进行化验 | ItemStaticData.analysedEvidence |
| 物品合成 | 拖拽两个物品到合成槽 | ItemStaticData.combineParameter[] |
| 证词浏览 | 查看已获得的证词道具 | TestimonyItem（已收集的） |

**分析流程**：选中物品 → 点击分析 → 原物品变为分析后的新物品

```
ItemStaticData 1114（沾湿的毛巾）
  canAnalyzed = true
  analysedEvidence = 1701
      ↓ 分析操作
ItemStaticData 1701（带有氯仿气味的毛巾）
  beforeAnalysedEvidence = 1114
```

**合成流程**：将两个物品放入合成槽 → 点击合成 → 获得新物品

```
ItemStaticData 1121（清洁剂）+ ItemStaticData 1701（氯仿毛巾）
  combineParameter = ["1121","1701"]
      ↓ 合成操作
生成新物品 1702
```

### TIMELINE（时间线）

用于整理 NPC 在案发期间的行踪。玩家将时间相关的证词拖入时间线面板，系统自动检测矛盾。

| 操作步骤 | 说明 |
|----------|------|
| 1. 打开 TIMELINE 面板 | 显示案发时间轴（横轴）和各NPC（纵轴） |
| 2. 从证词列表拖拽证词 | 只有 `triggerType=1`（Timeline）的证词可放入 |
| 3. 证词转为时间条 | 根据 triggerParam 中的时间段自动定位 |
| 4. 系统检测 | 多条证词放入后自动检测时间重叠、地点矛盾 |
| 5. 触发结果 | 满足条件的证词被标记为"已验证"，可能参与疑点解锁 |

**triggerParam 格式**（triggerType=1）：
```
"103,1103,2130,2500"
  │    │     │     └── 结束时间 25:00（= 次日 01:00，24h+制）
  │    │     └── 开始时间 21:30
  │    └── 场景 ID 1103
  └── NPC ID 103 (Rosa)
```

**实例**：将 TestimonyItem 1031002（"Rosa从9点半开始在地下室走廊打扫"）拖入时间线 → 系统在 Rosa 行上显示 21:30-25:00 的时间条 → 该证词被标记为已在时间线上验证。

### RELATIONSHIP（关系网）

展示 NPC 之间的关系图谱。玩家通过将见闻类证词拖到 NPC 头像上来验证身份和关系。

| 操作步骤 | 说明 |
|----------|------|
| 1. 打开 RELATIONSHIP 面板 | 显示各嫌疑人头像节点 |
| 2. 将证词拖到 NPC 头像 | 只有 `triggerType=2`（RelationNetwork）的证词可交互 |
| 3. 匹配验证 | triggerParam 中的两个 NPC 是否匹配 |
| 4. 成功 → 确认关系 | 头像高亮 + 显示身份标签，连线建立 |
| 5. 失败 → 提示不匹配 | "这条证词似乎不属于这个人" |

**triggerParam 格式**（triggerType=2）：
```
"103,105"
  │    └── NPC ID 105 (Tommy)
  └── NPC ID 103 (Rosa)
```

**实例**：将 TestimonyItem 1031001（"Rosa是蓝月亮的清洁工"）拖到 Tommy 的头像 → triggerParam="103,105" 匹配成功 → Rosa 与 Tommy 的雇佣关系连线建立。

**头像视觉状态**：
- 未确认身份：灰色边框
- 已确认身份：高亮边框 + 身份标签
- 发现关系疑点：连线变红 + 感叹号

## 三、数据来源汇总

Case Board 不拥有自己的配置表，它读取以下表的运行时数据：

| 数据 | 来源表 | 过滤条件 | 用途 |
|------|--------|----------|------|
| 已收集物品 | ItemStaticData | 运行时标记为已收集 | EVIDENCE 面板展示、分析和合成 |
| 已获取证词 | TestimonyItem | 运行时标记为已获取 | TIMELINE 和 RELATIONSHIP 面板操作 |
| 已解锁疑点 | DoubtConfig | condition 全部满足 | 疑点列表展示 |
| NPC信息 | NPCStaticData | 本章角色 | RELATIONSHIP 面板头像和标签 |

## 四、Case Board 与疑点解锁的关系

玩家在 Case Board 中的操作是疑点解锁的主要驱动力。DoubtConfig 的三种条件类型都与 Case Board 操作相关：

| DoubtConditionType | 对应操作 | 面板 |
|--------------------|----------|------|
| type=1 (Item) | 拥有该道具（可能需要先分析/合成） | EVIDENCE |
| type=2 (RelationNetwork) | 在关系网中验证该证词 | RELATIONSHIP |
| type=3 (Timeline) | 在时间线中放入该证词 | TIMELINE |

**实例**：疑点 1101（"Rosa当晚究竟在哪工作？"）的解锁条件：
```json
{
  "id": "1101",
  "condition": [
    {"type": "3", "param": "1031002"},
    {"type": "1", "param": "1115"}
  ],
  "text": "Rosa当晚究竟在哪工作？"
}
```
→ 条件1：在 TIMELINE 面板放入证词 1031002（Rosa的时间线证词）
→ 条件2：在 EVIDENCE 面板拥有道具 1115（排班表）
→ 两个条件都满足时，疑点自动解锁

## 五、核心流程

```
场景探索 + NPC对话
    ↓ 收集物品和证词
打开 Case Board
    ├── EVIDENCE：查看/分析/合成物品
    ├── TIMELINE：拖拽证词整理时间线
    └── RELATIONSHIP：拖拽证词梳理人物关系
    ↓ 操作触发条件满足
疑点自动解锁（DoubtConfig.condition 全部达成）
    ↓ 本轮全部疑点解锁
EXPOSE 按钮亮起 → 进入指证环节
```

## 六、与其他系统的关联

| 系统 | 关联方式 |
|------|----------|
| **搜证与物品系统** | 提供 EVIDENCE 面板的数据源；分析/合成操作改变 ItemStaticData 的持有状态 |
| **对话与证词系统** | 提供 TIMELINE 和 RELATIONSHIP 面板的数据源（TestimonyItem） |
| **疑点系统** | Case Board 操作是疑点解锁的主要途径；解锁的疑点在 Case Board 中展示 |
| **指证系统** | 全部疑点解锁后从 Case Board 进入指证；指证时使用的证据来自 EVIDENCE |
| **NPC系统** | RELATIONSHIP 面板使用 NPCStaticData 的头像和名称信息 |
