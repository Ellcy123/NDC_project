# 场景系统

## 一、核心概念

场景是玩家进行调查的空间载体。SceneConfig 是数据的"超级容器"——NPC 和道具仅通过 SceneConfig 的 `NPCInfos` 和 `ItemIDs` 字段与场景关联。场景本身不存储背景图和地点名称，而是通过 `location` 外键引用 LocationConfig 获取这些信息。

同一物理地点在不同 Loop 中是不同的 sceneId，但引用同一个 LocationConfig。例如 Rosa 储藏室在 Loop1 的 sceneId 是 `1101`，在 Loop2 是 `1201`，但都引用 LocationConfig id=`101`。

## 二、三表关系

```
SceneConfig（场景实例）
├── location → LocationConfig（地点信息：名称、背景图）
├── NPCInfos → NPCLoopData（NPC 实例：对话、立绘、位置）
└── ItemIDs → ItemStaticData（道具 ID 列表）

MapConfig（地图点位）
└── 通过 ChapterConfig.map2Scenes 映射到 SceneConfig
```

| 表 | 职责 | 主键 |
|----|------|------|
| SceneConfig | 场景实例，绑定具体的 NPC 和道具 | sceneId |
| LocationConfig | 物理地点的静态信息（名称、背景图、环境音效） | id |
| MapConfig | 游戏地图上的可交互地点（图标、坐标） | id |

## 三、数据结构

### 3.1 SceneConfig

| 字段 | 含义 | 示例值 |
|------|------|--------|
| sceneId | 场景 ID（4 位编码） | `"1103"` |
| location | 地点信息（引用 LocationConfig） | `{"id":"103", "Name":["Tommy办公室","TommyOffice"], ...}` |
| backgroundMusic | 背景音乐资源路径 | `"test"` |
| NPCInfos | 场景中的 NPC 实例列表（引用 NPCLoopData） | 见下方示例 |
| ItemIDs | 场景中的道具 ID 列表（引用 ItemStaticData） | `["110022"]` |

### 3.2 LocationConfig

| 字段 | 含义 | 示例值 |
|------|------|--------|
| id | 地点 ID | `"103"` |
| Name | 地点名称 [中, 英] | `["Tommy办公室","TommyOffice"]` |
| sceneType | 场景类型 | `"1"` (dialogue) |
| backgroundImage | 背景图资源路径 | `"Art\Scene\Backgrounds\EPI01\SC003_bg_TommyOffice"` |
| ambientSound | 环境音效路径（可选） | `""` |

### 3.3 MapConfig

| 字段 | 含义 | 示例值 |
|------|------|--------|
| id | 地图点 ID | `"101"` |
| Name | 地名 [中, 英] | `["蓝月亮酒吧","Blue Moon Cabaret"]` |
| icon | 图标资源路径 | `"Art\UI\map\cabaret"` |
| position | 地图坐标 [x, y, z] | `["0","0","0"]` |

**第一章地图点位**：

| id | 地名 | 英文名 |
|----|------|--------|
| 101 | 蓝月亮酒吧 | Blue Moon Cabaret |
| 102 | Morrison 家 | Morrison's Home |
| 103 | Jimmy 家 | Jimmy's Home |
| 104 | Zack 办公室 | Zack Office |
| 105 | 警局 | Police Station |

## 四、同地点跨 Loop 的关系

同一个 LocationConfig 被多个 SceneConfig 引用，每个 Loop 的 NPC 和道具不同：

| sceneId | Loop | location.id | 地点名 | NPC | 部分道具 |
|---------|------|-------------|--------|-----|---------|
| 1101 | 1 | 101 | Rosa 储藏室 | 无 | 1111, 1112, 1115 |
| 1201 | 2 | 101 | Rosa 储藏室 | Rosa (103) | 120021 |
| 1103 | 1 | 103 | Tommy 办公室 | Tommy (105) | 110022 |
| 1203 | 2 | 103 | Tommy 办公室 | Tommy (105) | -- |

Loop1 的 Rosa 储藏室（1101）没有 NPC，只有物品可拾取；Loop2 的同一地点（1201）出现了 Rosa 且道具不同。这体现了循环推进对场景内容的影响。

## 五、完整实例

### 场景 1103（Loop1 Tommy 办公室）

```json
{
  "sceneId": "1103",
  "location": {
    "id": "103",
    "Name": ["Tommy办公室", "TommyOffice"],
    "sceneType": "1",
    "backgroundImage": "Art\\Scene\\Backgrounds\\EPI01\\SC003_bg_TommyOffice"
  },
  "backgroundMusic": "test",
  "NPCInfos": [{
    "id": "1001",
    "NPC": {"id": "105", "Name": ["Tommy","Tommy"], "role": "2"},
    "TalkInfo": {"id": "105001001", ...},
    "LoopTalkInfo": {"id": "105002001", ...},
    "ResPath": "Art\\Scene\\NPC\\EPI01\\SC103_npc_Tommy1",
    "ClickResPath": "Art\\Scene\\NPC\\EPI01\\SC103_npc_Tommy2",
    "PosX": "1101", "Posy": "681", "PosZ": "-1"
  }],
  "ItemIDs": ["110022"]
}
```

**解读**：
- 场景引用 LocationConfig 103 获取背景图 `SC003_bg_TommyOffice`
- 包含 1 个 NPC 实例（NPCLoopData 1001）：Tommy，首次对话从 105001001 开始，重复对话从 105002001 开始
- 包含 1 个装饰物品 110022

### 场景 1101（Loop1 Rosa 储藏室，无 NPC）

```json
{
  "sceneId": "1101",
  "location": {
    "id": "101",
    "Name": ["Rosa储藏室", "RosaStorageRoom"],
    "sceneType": "1",
    "backgroundImage": "Art\\Scene\\Backgrounds\\EPI01\\SC001_bg_RosaStorageRoom"
  },
  "backgroundMusic": "test",
  "ItemIDs": ["110021", "1111", "1112", "1115", "1801", "1803"]
}
```

**解读**：
- 没有 NPCInfos 字段——本场景无 NPC
- 包含 6 个道具：装饰物(110021)、线索/道具(1111, 1112, 1115)、门(1801, 1803)

## 六、场景 ID 编码

格式：4 位数 `{unit}{loop}{location}`

| 位 | 含义 | 范围 |
|----|------|------|
| 第 1 位 | 章节(Unit) | 1=第一章, 2=第二章 |
| 第 2 位 | 循环(Loop) | 1-6 |
| 第 3-4 位 | 地点编号 | 01-23+ |

| sceneId | 含义 |
|---------|------|
| 1101 | Unit1 Loop1 地点 01（Rosa 储藏室） |
| 1201 | Unit1 Loop2 地点 01（Rosa 储藏室） |
| 1117 | Unit1 Loop1 地点 17（酒吧外街道） |
| 1210 | Unit1 Loop2 地点 10（酒吧大堂） |

LocationConfig.id 的编码与 sceneId 的第 3-4 位对应，EPI01 地点范围为 `101`-`125`。

## 七、与其他系统的关联

| 关联系统 | 关联方式 |
|---------|---------|
| 核心循环 | ChapterConfig.initScene 指定每轮初始场景；map2Scenes 映射地图点位到场景 |
| 对话系统 | NPCInfos 中的 TalkInfo/LoopTalkInfo 指向 Talk 表的对话链 |
| 搜证系统 | ItemIDs 定义场景中可交互的物品列表 |
| 地图系统 | MapConfig 定义地图 UI 上的点位，通过 ChapterConfig.map2Scenes 关联到具体场景 |
