# 时间线系统

## 一、核心概念

**时间线（Timeline）** = 玩家将 NPC 的时间相关证词拖放到时间轴上，还原案发当晚各角色的行动轨迹，从而发现时间重叠、地点矛盾等疑点。

时间线是三大整理工具之一（时间线 / 关系网 / 案情板），核心作用是**验证 NPC"什么时间在什么地点"的陈述是否成立**。验证成功后产出证词道具，可进一步用于疑点解锁和指证。

## 二、核心流程

```
1. 对话获得时间相关证词
       ↓
2. 打开 INVESTIGATE → TIMELINE
       ↓
3. 从证词列表拖拽证词到时间线面板
       ↓
4. 证词转化为时间条（显示 NPC 在某时段某地点的活动）
       ↓
5. 多条证词放入后，系统自动检测矛盾
       ↓
6. 验证成功 → 产出可用的证词道具（TestimonyItem）
```

### 矛盾检测类型

| 检测类型 | 说明 | 示例 |
|----------|------|------|
| 时间重叠 | 同一 NPC 同时段出现在不同地点 | Rosa 声称 21:30 在地下室，但另一证词说她 23:00 在走廊 |
| 地点矛盾 | 不同 NPC 的证词互相排斥 | Morrison 声称 23:30 接到线报赶来，但 Vivian 说他 00:00 才到 |
| 时间空白 | NPC 在关键时段无法说明去向 | Jimmy 在 23:10-00:00 声称一直在厨房，但存在空隙 |

## 三、数据结构

### 3.1 DayTimeConfig — 时间范围定义

定义时间线面板显示的时间范围。

| 字段 | 类型 | 含义 | 示例 |
|------|------|------|------|
| day | int(string) | 日期标识（关联场景 ID） | `"1103"` |
| startTime | int(string) | 开始时间（HHMM 格式） | `"2100"` |
| endTime | int(string) | 结束时间（HHMM 格式） | `"2500"` |

当前配置：场景 1103（蓝月亮酒吧），时间窗口 21:00 ~ 25:00。

### 3.2 时间格式说明（HHMM，24h+ 制）

| 表示 | 实际时间 | 说明 |
|------|----------|------|
| `2100` | 当晚 21:00 | 普通 24h 格式 |
| `2330` | 当晚 23:30 | |
| `2400` | 午夜 00:00 | 24:00 = 次日零点 |
| `2430` | 次日 00:30 | 超过 2400 表示跨天 |
| `2500` | 次日 01:00 | 25:00 = 次日 01:00 |

采用 24h+ 制（不回绕到 00:00）是为了让时间轴上的数值始终递增，避免跨天显示混乱。

### 3.3 TestimonyItem 中的时间线数据

证词道具中 `triggerType="1"` 的条目为时间线类型，其 `triggerParam` 驱动时间线验证：

| 字段 | 含义 |
|------|------|
| triggerType | `"1"` = Timeline（时间线验证） |
| triggerParam | 格式：`"NPCID,sceneId,startTime,endTime"` |

**triggerParam 拆解**：
```
"103,1103,2130,2500"
  │    │     │     └── 结束时间 25:00（次日 01:00）
  │    │     └──────── 开始时间 21:30
  │    └────────────── 场景 1103（蓝月亮酒吧）
  └─────────────────── NPC 103（Rosa）
```

当 startTime = endTime 时，表示**时间点事件**而非时间段：
```
"105,1103,2330,2330"   → Tommy 在 23:30 听到枪响（瞬时事件）
```

## 四、完整实例：第一章时间线证词

以下是第一章（EPI01）中所有 triggerType=1 的 TestimonyItem，即可放入时间线的证词道具：

### 4.1 NPC 活动时间段

| ID | NPC | 内容 | 时间段 | 场景 |
|----|-----|------|--------|------|
| 1031002 | Rosa(103) | 从 9 点半开始在地下室打扫 | 21:30-25:00 | 1103 |
| 1051002 | Tommy(105) | 从晚上 10 点在办公室整理账本 | 22:00-25:00 | 1103 |
| 1071003 | Jimmy(107) | 约晚上 9 点到达蓝月亮上夜班 | 21:00-25:00 | 1103 |
| 1071004 | Jimmy(107) | 11:10 到近 12 点一直在厨房 | 23:10-24:00 | 1103 |
| 1061003 | Vivian(106) | 10 点到 11:10 在舞台表演 | 22:00-23:10 | 1103 |
| 1062001 | Webb(109) | 10 点在歌舞厅前排看表演 | 22:00-22:30 | 1103 |

### 4.2 时间点事件

| ID | NPC | 内容 | 时间点 | 场景 |
|----|-----|------|--------|------|
| 1051003 | Tommy(105) | 11:30 听到一声枪响 | 23:30 | 1103 |
| 1033002 | Rosa(103) | 11 点左右音乐异常响 | 23:00 | 1103 |
| 1062002 | Webb(109) | 10:30 回会客室独自喝酒 | 22:30 | 1103 |
| 1032001 | Rosa(103) | 约 12:30 躲进储藏室 | 24:30 | 1103 |
| 1061001 | Vivian(106) | Morrison 午夜 12 点到达现场 | 24:00 | 1103 |
| 1065001 | Vivian(106) | 约 23:05 去会客室被拒 | 23:05 | 1103 |
| 1034003 | Rosa(103) | 约 23 点看到 Vivian 携包走向会客室 | 23:00 | 1103 |
| 1054001 | Tommy(105) | 约 21 点看到 Jimmy 抱出古董水壶 | 21:00 | 1103 |

### 4.3 Morrison 到场时间矛盾（核心推理点）

```
Morrison 自称：23:30 接到线报立刻赶来  → 1041003 (104,1103,2330,2330)
Tommy 证词：开车最多 15 分钟           → 1052002 (104,1103,2330,2345)
Vivian 证词：Morrison 午夜 12 点到达    → 1061001 (104,1103,2400,2400)
Rosa 证词：凌晨 1 点 Morrison 空手进出  → 1032003 (104,1103,2500,2500)
```

将这四条证词全部放入时间线后，矛盾可视化：Morrison 声称 23:30 出发、15 分钟车程，但 Vivian 说他 00:00 才到，中间 30 分钟去向不明。

## 五、证词道具的 truth 字段

部分 TestimonyItem 包含 `truth` 和 `shortTruth` 字段，记录该证词背后的真相。这些在后续循环中揭示：

```json
{
  "id": "1031002",
  "testimony": ["Rosa从9点半开始在地下室打扫", "..."],
  "truth": ["Rosa目睹了Morrison栽赃Zack的过程，被Morrison以女儿安全威胁", "..."],
  "shortDesc": ["在地下室打扫", "Cleaning basement"],
  "shortTruth": ["实际晚上11点起在一楼走廊打扫", "Actually cleaning first floor corridor from 11 PM"]
}
```

→ Rosa 的证词在 Loop1 表面成立，但在后续循环中被揭穿为谎言。

## 六、与其他系统的关联

| 关联系统 | 方向 | 说明 |
|----------|------|------|
| **对话系统** | → 时间线 | 对话中 `script="get"` 获得时间线类型的证词道具 |
| **疑点系统** | 时间线 → | 疑点条件 `type=3`（Timeline）要求证词已放入时间线 |
| **指证系统** | 时间线 → | 时间线产出的证词道具可作为指证所需的 testimony 或 item |
| **关系网** | 并列 | 同属 INVESTIGATE 面板的两个整理工具，互不依赖 |
