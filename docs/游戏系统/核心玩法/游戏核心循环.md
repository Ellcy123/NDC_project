# 游戏核心循环

## 一、核心概念

游戏采用 **章节(Unit) > 循环(Loop) > 场景(Scene)** 的三级结构。每个章节对应一个完整案件，章节内分为多个循环，每个循环围绕一个嫌疑人展开调查和指证。第一章（EPI01）有 6 个循环，每轮指证不同的嫌疑人，逐步接近真相。

## 二、层级关系

```
GameFlowConfig（章节）
└── ChapterConfig（循环）× 6
    ├── 开场对话（initTalk → Talk 表）
    ├── 初始场景（initScene → SceneConfig 表）
    ├── 疑点列表（doubts → DoubtConfig）
    ├── 通关疑点（clearDoubts）
    └── 指证方案（exposes → ExposeData）
```

## 三、单循环玩法流程

每个循环遵循固定的游戏节奏：

```
开场对话 → 自由调查 → 收集证据/证词 → 整理分析 → 解锁疑点 → 指证
```

| 阶段 | 玩家行为 | 系统产出 |
|------|---------|---------|
| 开场对话 | 观看 Emma 引入对话 | 了解本轮调查目标 |
| 场景探索 | 点击场景中的物品 | 收集证据/线索（ItemStaticData） |
| NPC 对话 | 与场景中的 NPC 交谈 | 获取证词道具（TestimonyItem） |
| CASE BOARD | 分析物品、合并物品 | 获得分析后的新证据 |
| INVESTIGATE | 整理时间线、整理关系网 | 验证证词、发现矛盾 |
| 解锁疑点 | 满足 DoubtConfig 条件 | 疑点出现在面板中 |
| EXPOSE | 选择证词 + 出示物品指证 NPC | 推进剧情，结束本轮循环 |

### 指证解锁条件

每轮循环定义了 `clearDoubts` 列表，玩家必须解锁其中指定的疑点后才能进入指证环节。例如 Loop1 的 `clearDoubts` 为 `["1101","1102","1103"]`，需要解锁这 3 个疑点。

## 四、数据结构

### 4.1 GameFlowConfig -- 章节级配置

| 字段 | 含义 | 示例值 |
|------|------|--------|
| chapterId | 章节 ID | `"1"` |
| loopCount | 循环数量 | `"6"` |
| chapterName | 章节名称 [中, 英] | `["血色酒吧","Blue"]` |
| openingVideo | 开篇视频路径 | `"Art\MovieClip\epi01\opening0307"` |
| deceased | 死者 NPC（引用 NPCStaticData） | Webb (109) |

### 4.2 ChapterConfig -- 循环级配置

| 字段 | 含义 | 示例值 |
|------|------|--------|
| id | 循环索引（{unit}{loop}） | `"101"` |
| chapterTitle | 本轮标题 [中, 英] | `["目击者的沉默","The Witness's Silence"]` |
| chapterGoal | 本轮调查目标 [中, 英] | `["有人迷晕了我，是谁？","Someone knocked me out. Who?"]` |
| initTalk | 开场对话 ID（引用 Talk 表） | `"104001005"` |
| initScene | 初始场景 ID（引用 SceneConfig 表） | `"1122"` |
| doubts | 本轮疑点列表（引用 DoubtConfig） | 见下方示例 |
| clearDoubts | 通关所需疑点 ID 列表 | `["1101","1102","1103"]` |
| exposeNpcId | 指证目标 NPC ID | `"103"` (Rosa) |
| topBg / bottomBg | 指证场景背景图 | `"Art/Scene/Expose/epi01/top_rosa"` |
| exposes | 指证方案列表（引用 ExposeData） | 见下方示例 |
| map2Scenes | 地图点位与场景的映射 | `[{"mapId":"101","sceneId":"1110"}]` |


**解读**：
- 游戏从场景 1122（Webb 会客室开场）启动，播放对话 104001005

- 地图上蓝月亮酒吧（mapId=101）对应搜证场景 1110（酒吧大堂）

## 六、ID 编码规则

### ChapterConfig.id

格式：`{unit}{loop}`，如 `101` = 第一章 Loop1，`106` = 第一章 Loop6，`201` = 第二章 Loop1。

### 疑点 ID（DoubtConfig.id）

格式：`{unit}{loop}{序号}`，如 `1101` = 第一章 Loop1 第 1 个疑点。

## 七、与其他系统的关联

| 关联系统 | 关联方式 |
|---------|---------|
| 对话系统 | `initTalk` 指定每轮开场对话链 |
| 场景系统 | `initScene` 指定每轮初始场景；`map2Scenes` 映射地图与搜证场景 |
| 疑点系统 | `doubts` 定义本轮疑点及解锁条件；`clearDoubts` 定义通关所需疑点 |
| 指证系统 | `exposes` 定义指证方案；`exposeNpcId` 指定被指证 NPC |
| 搜证系统 | 疑点条件和指证方案均引用 ItemStaticData / TestimonyItem |
