# 搜证与物品系统

## 一、物品类型（itemType 枚举）

游戏中所有可交互对象定义在 `ItemStaticData` 表中，通过 `itemType` 区分。

**核心玩家可见的三大类**（绝大多数设计都落在这三类）：

| itemType | 名称 | 可收集 | 可分析 | 可合成 | 说明 |
|----------|------|--------|--------|--------|------|
| 1 | **clue** | ✓（拍照） | ✗ | ✗ | **现场线索**——现场留下的痕迹类，玩家通过拍照取证收集（如弹壳、玻璃碎片、留在原位的物品现场照），不能带走原物，也不可分析/合成 |
| 2 | **envir** | ✗ | ✗ | ✗ | **环境叙事**——可查看描述，不可收集；只承担氛围/背景/情感暗示功能（如合影、排班表、痛苦记录） |
| 3 | **item** | ✓（直接带走） | ✓（可选） | ✓（可选） | **物品**——可直接拾取带走的实物，包括武器、弹壳（已获得）、信件、协议书、账本、字条、日记等一切纸质文件与实物。`canAnalyzed/canCombined` 由该物品自身性质决定（如手枪可分析硝烟、毛巾可分析气味；信件等一般不可分析） |

**NPC 口述信息**走 `Testimony` 表（证词系统，见《对话与证词系统》），**不属于 `ItemStaticData`**。`testimony` 不是 itemType 的一种。

**其它辅助/基础设施类**（不作为推理证据，仅场景支持）：

| itemType | 名称 | 可收集 | 说明 |
|----------|------|--------|------|
| 0 | decoration | ✗ | 纯场景装饰，无法交互 |
| 5 | door | ✗ | 用于场景切换 |
| 6 | interactive_item | ✗ | 可交互但不可收集（如开关、机关） |
| 7 | embedded_item | ✗ | 嵌入场景中的特殊道具 |

> **已废弃类型**：原 `itemType=4 note`（"文件笔记"）已合并入 `item`——纸质文件（信件、账本、日记等）统一作为 item 处理，通过 `canAnalyzed=false` 表达"不可分析"的特性，无需独立类型。

## 二、物品获取途径

**场景探索**：点击场景中的物品 → 弹出 ItemPanel 显示详细描述 → 如果可收集则入背包

**对话获取**：NPC 对话中 `script="get"` 触发 → 自动获得指定物品/证词

## 三、物品加工

### 3.1 分析（Analysis）

当 `canAnalyzed=true` 时，玩家可在 CASE BOARD 的分析区对该物品进行化验/指纹/成分分析。

**实例**：
```
物品 1114「沾湿的毛巾」(itemType=3, canAnalyzed=true)
    ↓ 玩家拖入分析区，选择"化验分析"
    ↓ 播放分析动画
物品 1701「带有氯仿气味的毛巾」(analysedEvidence=1701)
    → 原始物品描述："一条沾湿的白色毛巾"
    → 分析后描述："经化验，毛巾上检测出氯仿残留，这是一种常见的麻醉剂"
```

分析后新物品通过 `beforeAnalysedEvidence` 反向指回原物品，形成双向引用。

### 3.2 合成（Combine）

当 `canCombined=true` 时，在 CASE BOARD 合成区将两个物品拖入槽位合并。

**交互流程**：
1. 将物品 A 拖入槽位 1
2. 将物品 B 拖入槽位 2
3. 系统检查：A 的 `combineParameter` 是否包含 B（或反之）
4. 匹配成功 → 产出新物品，原料标记为已使用
5. 匹配失败 → 提示 "物品不可以被组合"

**实例**：
```
物品 1121「走廊清洁记录」+ 物品 1701「带有氯仿气味的毛巾」
    ↓ 合成
物品 1702「分析报告：氯仿毛巾与清洁记录的时间矛盾」
```

## 四、物品 ID 编码规则

| ID 格式 | 含义 | 示例 |
|---------|------|------|
| `11xx` | Unit1 Loop1 的物品 | 1114 = Loop1 第14号物品 |
| `12xx` | Unit1 Loop2 的物品 | 1201 = Loop2 第1号物品 |
| `17xx` | 分析后产出的证据 | 1701 = 分析后的毛巾 |
| `18xx` | 门/道具类 | 1801 = 某扇门 |
| `110xxx` | 纯装饰（6位） | 110021 = 装饰物 |
| `21xx` | Unit2 Loop1 物品 | EPI02 |
| `22xx` | Unit2 Loop2 物品 | EPI02 |

## 五、CASE BOARD 面板

CASE BOARD 有两种模式：

1. **Note**（笔记栏）：展示已收集的物品/线索（分 item/clue 两列）
2. **Combine**（合成栏）：拖拽两个物品到槽位进行合成

分析/合成成功后可能触发疑点解锁（详见疑点系统文档）。
