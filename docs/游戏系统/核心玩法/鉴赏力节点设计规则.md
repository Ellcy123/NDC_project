# 鉴赏力节点设计规则

> 本文档是对 `鉴赏力系统.md`（系统机制层）的**设计层补充**，专门解决「如何让鉴赏力节点自然融入对话」的问题。
> 系统机制（三选一、加分不扣分、成就领域等）见 `鉴赏力系统.md`，此处不重复。

---

## 一、核心问题

Unit2实践中暴露的最大体验问题：

> **NPC突然像老师一样问出考试题目，打断对话节奏，玩家感觉在被考试而不是在对话。**

根本原因：quiz节点的设计思路是"NPC向Zack提问"——这让NPC变成了出题者，Zack变成了答题学生。

---

## 二、设计核心原则

### 原则1：鉴赏力的主语是Zack，不是NPC

**错误思路**：NPC问Zack一个问题 → Zack回答 → NPC评价对错

**正确思路**：对话进行到某个自然节点 → **Zack主动选择说出某个细节**来展现自己的了解程度 → NPC根据Zack说的话做出不同反应

区别在于：
- 错误：Tony问"我的招牌鸡尾酒叫什么？"（老师提问）
- 正确：Tony说"来一杯？"，Zack的选项是"来杯Southside" / "随便来一杯" / "来杯曼哈顿"（Zack主动展示）

**Zack是主动展示者，NPC是被动反应者。**

### 原则2：NPC必须有角色动机参与这个对话节点

每个鉴赏力节点中，NPC的反应必须符合其角色动机——不是因为"系统需要一个quiz"，而是因为**这个角色在这个情境下自然会在意Zack是否了解某件事**。

- Foster在意Zack是否理解法医的局限 → 因为他不想解释被误读
- Margaret在意Zack是否真正了解Mary的处境 → 因为她要决定是否信任儿子
- Bernard在意Zack是否懂保险 → 因为他想用专业术语打发外行

### 原则3：节点必须服务于对话的推进，而非打断

鉴赏力节点不是对话的"插入广告"，而是对话的**分水岭**——Zack展示的知识水平决定了NPC接下来说多少、说多深。

选对 → NPC认为Zack是内行，愿意多说（奖励信息）
选错 → NPC认为Zack是外行，一句带过（跳过细节）

两条路径都能继续主线对话，但**信息密度不同**。

## 三、六种自然触发模式

以下6种模式是鉴赏力节点融入对话的合法方式。每个quiz节点必须属于至少一种模式。

---

### 模式A：信任门槛（Trust Gate）

**逻辑**：NPC掌握敏感信息，需要确认Zack足够了解情况后才愿意深入交谈。不是"考试"，而是"你够格听这些吗？"

**触发方式**：NPC在即将透露重要信息前犹豫/试探，Zack的回应决定NPC是否打开话匣子。

**适用角色**：持有信息但有防备心的NPC（Bernard、Foster、Mickey）

**示例**：
```
Bernard: "Thomas的保单……您真的了解这种保险是怎么运作的吗？"
→ 分支：Zack选择说出保险运作方式的某个细节
→ 选对：Bernard叹气，"看来您做过功课。好吧，我告诉您……"
→ 选错：Bernard摇头，"这不是几句话能解释清楚的。总之，保单是合规的。"
```

**为什么自然**：信息持有者天然会筛选倾听者的水平。你不会对一个完全不懂行的人解释专业细节。

---

### 模式B：情感共鸣（Emotional Resonance）

**逻辑**：NPC情感上需要知道Zack是否真正理解他们在乎的事。不是测试知识，而是"你真的懂我的痛吗？"

**触发方式**：NPC谈到情感敏感话题时，Zack的回应方式决定NPC是否愿意进一步敞开心扉。

**适用角色**：情感驱动的NPC（Margaret、Liam、Helen、Mary）

**示例**：
```
Margaret: "你知道Mary过的是什么日子吗……"（停顿，看着Zack）
→ 分支：Zack选择如何回应——是说出具体细节（"她报过三次警"）还是泛泛而谈（"我知道她过得不好"）
→ 选对：Margaret眼眶红了，"你真的去了解了……好，那我告诉你那天晚上的事"
→ 选错：Margaret摇头，"你不懂。算了。"
```

**为什么自然**：人在分享痛苦经历前会本能地试探对方是否真正关心——不是考试，是情感确认。

---

### 模式C：专业对等（Professional Parity）

**逻辑**：专业人士（法医、银行家、律师）根据Zack展示的知识水平调整自己的沟通深度。

**触发方式**：专业NPC在解释技术问题时，Zack选择是否用专业术语/细节回应，NPC据此判断Zack的专业水平。

**适用角色**：专业身份NPC（Foster、Bernard）

**示例**：
```
Foster: "血液里的酒精浓度……以我们现在的技术手段……"（犹豫）
→ 分支：Zack的回应——"Widmark的方法还没传过来？"（展示了解） / "你是说你测不出具体数字？"（常识推断） / "那到底喝了多少？"（外行追问）
→ 选对：Foster眼睛一亮，"你知道Widmark？不错。实话说，我的数字是经验估算……"
→ 选错：Foster简短回答，"总之，他喝了很多。"
```

**为什么自然**：专家和内行说话的方式与对外行完全不同。这不是考试，是专业交流中的自然分层。

---

### 模式D：对话回声（Conversational Callback）

**逻辑**：当前对话中自然提到了某个之前出现过的信息点，Zack可以选择是否主动引用之前获得的具体信息。

**触发方式**：NPC提到一个话题，Zack的回应选项中包含一个引用前序信息的具体细节。不是NPC在问，是Zack在对话中自然地调用记忆。

**适用场景**：任何对话中涉及到前序Loop信息的时刻

**示例**：
```
Liam: "Mary那天晚上说她去教堂了……"
→ 分支：Zack选择回应——"教堂那个时间不可能还有弥撒"（引用KP-02蜡烛/弥撒知识） / "教堂离这儿不远"（无关回应） / "她经常去教堂吗？"（继续追问）
→ 选对：Liam挑眉，"嗯，你说得对。教堂晚上六七点就关门了，弥撒中午前就结束……那她去做什么？"
→ 选错：Liam继续说下去，不展开这个话题
```

**为什么自然**：人在对话中引用之前了解到的信息是最正常不过的事。Zack作为侦探，主动串联信息是他的本职工作。

---

### 模式E：防御反弹（Defensive Deflection）

**逻辑**：NPC在被追问时反过来试探Zack——"你到底知道多少？"不是出题，而是嫌疑人/证人的自我保护本能。

**触发方式**：在紧张的对话或指证前的对话中，NPC感到压力后反问Zack，试图评估对方掌握了多少信息。

**适用角色**：有防御心理的NPC（Helen、Bernard、Morrison）

**示例**：
```
Helen:（被追问楼顶的事后）"你调查了这么久，你觉得你了解这栋楼的人吗？"
→ 分支：Zack选择回应——具体说出某个关于社区/邻居的细节 / 泛泛回答 / 反问回去
→ 选对：Helen沉默片刻，"……你比我以为的知道得多。"
→ 选错：Helen冷笑，"看来你并不像你以为的那么了解。"
```

**为什么自然**：嫌疑人在被追问时反过来试探对方的信息量，是真实审讯中极常见的行为——不是考试，是心理博弈。

---

### 模式F：环境共振（Environmental Resonance）

**逻辑**：当前场景中的环境元素（物品、摆设、声音）自然引出一个话题，Zack可以选择是否展现对这个元素的了解。

**触发方式**：搜证或进入场景时，某个环境细节触发对话选项。

**适用场景**：搜证场景、首次进入新场景

**示例**：
```
（Zack在教堂搜证，看到蜡烛架）
旁白/Zack内心：蜡烛架上的蜡烛……
→ 分支：Zack选择观察方式——"这些蜡烛看起来烧了很久，至少一天了"（展示了解燃烧时间） / "有些蜡烛还在烧"（泛泛观察） / "挺多蜡烛的"（不关注）
→ 选对：（内心旁白）"5美分一根的祈愿蜡烛，玻璃杯的能烧24到48小时……也就是说，这些是昨天白天点的。昨晚没有人来点新蜡烛。"
→ 选错：不触发额外推理
```

**为什么自然**：侦探在现场观察细节时，展示专业素养是最自然的鉴赏力表现。

## 四、禁止模式（Anti-Patterns）

以下模式在设计鉴赏力节点时**严格禁止**：

### ❌ 1. 老师提问模式

```
NPC: "我考考你，XX是什么？"
```
NPC没有理由在对话中突然变成老师。如果NPC确实在"测试"Zack，必须有明确的角色动机（如模式A的信任门槛）。

### ❌ 2. 突兀插入模式

```
NPC:（正在说A话题）"对了，你还记得XXX吗？"（突然跳到B话题）
```
鉴赏力节点必须从当前话题自然生长出来，不能硬切话题。

### ❌ 3. 无动机测试模式

```
NPC:（没有任何理由地）"你知道验尸官是怎么选出来的吗？"
```
NPC必须有角色层面的理由关心Zack是否知道这件事。如果NPC本人都不关心答案，玩家会立刻感觉到这是"系统在考我"。

### ❌ 4. 单句弹窗模式

```
（对话中突然弹出三个选项，选完后NPC说一句话就结束）
```
鉴赏力节点前后必须有足够的对话上下文。选对后NPC多说1-3句的奖励内容应该自然融入后续对话，不是独立弹窗。

### ❌ 5. 重复测试模式

```
（同一条信息在不同NPC处被反复测试）
```
每条信息只考一次。重复测试会让玩家感觉被系统骚扰。

## 五、Source节点的设计规则

Source节点（信息埋伏）同样需要自然：

### 规则1：信息必须由NPC主动说出或由环境自然呈现

不能为了"埋source"而让NPC说出不符合角色的话。NPC说出某个信息必须有角色层面的理由。

### 规则2：信息密度适中

Source节点的信息不能太显眼（玩家觉得"这肯定要考"），也不能太隐蔽（玩家根本没注意到）。理想状态：**信息嵌入在一段自然对话中，不突出但也不隐藏——认真听的玩家能记住，走马观花的玩家会忽略。**

### 规则3：Source和Quiz的信息表述应有变化

Source处的原始信息和Quiz处的测试方式不应使用完全相同的措辞。玩家需要"理解"而不只是"复读"。

示例：
- Source（L2教堂搜证）：玩家在弥撒时间表上看到"最后一台弥撒11:00"
- Quiz（L3对话）：Zack选项中是"教堂晚上六七点就关门了，弥撒中午前就结束"——不是原文复述，而是理解后的转述

## 六、选对/选错的反应设计

### 选对反应

NPC的态度转变必须符合角色设定：

| 角色类型 | 选对反应风格 |
|---------|------------|
| 防备型（Bernard、Morrison） | 微妙的态度软化，"看来你不是随便来问问的" |
| 情感型（Margaret、Liam） | 情感打开，愿意说出更私密的内容 |
| 专业型（Foster） | 切换到专业对等模式，用术语和细节交流 |
| 敌对型（Helen被指证时） | 沉默或警惕，"你比我以为的知道得多" |

**奖励内容（多说1-3句）的定位**：这1-3句是**时代知识的教学载体**，但必须以NPC的口吻和立场说出，不是百科全书式的旁白。

### 选错反应

选错不是惩罚，而是NPC自然的信息过滤：

| 角色类型 | 选错反应风格 |
|---------|------------|
| 防备型 | 关上话题，"这不是几句话能说清的" |
| 情感型 | 失望但不愤怒，"……算了" |
| 专业型 | 简化解释，跳过细节 |
| 敌对型 | 轻蔑，"看来你什么都不知道" |

## 七、每个Loop的节点密度建议

| 循环阶段 | 建议节点数 | source:quiz比例 | 说明 |
|---------|----------|----------------|------|
| L1-L2（前期） | 2-3 | source为主 | 密集埋伏信息，少量quiz |
| L3-L4（中期） | 2-3 | quiz为主 | 收割前期埋伏，开始考查 |
| L5-L6（后期） | 1-2 | quiz为主 | 节点减少，聚焦终局 |

每章总计不超过15个节点（Unit2为19个，偏多）。

## 八、审查清单（在原有基础上扩展）

### 系统层审查（继承自鉴赏力系统.md）

- [ ] 信息来源是否在节点触发前玩家已接触过？
- [ ] 引用的证据ID是否在当前Loop范围内？
- [ ] 信息来源与测试点之间是否有足够距离？
- [ ] 是否以对话分支格式书写（不是考题）？
- [ ] 选对/选错的NPC反应是否有明显内容差异？

### 自然度审查（本文档新增）

- [ ] **主语检查**：是Zack主动展示知识，还是NPC在提问？→ 必须是Zack主动
- [ ] **动机检查**：NPC有角色层面的理由参与这个对话节点吗？→ 必须有
- [ ] **流程检查**：节点是从当前话题自然生长出来的吗？→ 不能硬切话题
- [ ] **模式检查**：属于六种合法触发模式中的哪一种？→ 必须明确标注
- [ ] **语气检查**：去掉鉴赏力标记后，这段对话读起来是否仍然通顺自然？→ 必须通顺
- [ ] **NPC反应检查**：选对后NPC多说的1-3句，是否以NPC自己的口吻和立场说出？→ 不能像百科旁白
