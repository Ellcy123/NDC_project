# 指证系统（Expose）

## 一、系统定位

指证是游戏的核心推理高潮环节。玩家通过选择正确证据揭穿 NPC 的谎言，NPC 多次狡辩，玩家需逐一反驳。

**前置条件**：收集完本轮全部主线疑点 → EXPOSE 按钮亮起

## 二、指证流程

```
收集完全部主线疑点 → EXPOSE 按钮亮起（红色）
    ↓
进入指证场景（ChapterConfig.topBg/bottomBg 提供背景）
    ↓
NPC 对话推进... → 遇到 Script="Lie" 的句子
    ↓
NPC 说出谎言 → 画面暂停 → 弹出证据选择界面
    ↓
玩家从背包选择证据放入槽位 → 点击"揭穿"
    ↓
系统验证：
  ├── ✓ 全部正确 → 印章动画 + 画面震动 → 跳转成功对话
  └── ✗ 有误 → 提示错误 → 重新选择
    ↓
NPC 被迫承认部分事实... 提出新狡辩（新 Lie）
    ↓
第 2 轮... 第 3 轮...
    ↓
所有轮次完成 → Script="end" → 本轮循环结束
```

## 三、验证逻辑

```
正确答案 = Talk.ParameterStr0（逗号分隔的物品 ID）
玩家提交 = 槽位中放置的物品

判定：
  1. 每件提交的证据都必须在正确列表中（不能有多余的错误证据）
  2. 正确列表中每件证据都必须被提交（不能缺少）
  → 精确匹配，不多不少
```

## 四、多轮指证数据结构

每轮的核心数据在 `ExposeData` 表中：

```json
{
  "id": "方案ID",
  "testimony": "所需证词道具 ID（TestimonyItem）",
  "item": ["需要出示的物品 ID（ItemStaticData）"],
  "talkId": "成功后跳转的对话 ID"
}
```

ChapterConfig (id=101): exposeNpcId=103(Rosa), 3 轮指证

## 六、揭穿按钮状态

| ParameterInt1 | 按钮 | 场景 |
|--------------|------|------|
| 1 | **揭穿** | 首次指证 |
| 2 | **再次揭穿** | 中间轮次 |
| 3 | **最终揭穿** | 最后一轮 |

## 七、谎言驱动原则（对话写作规则）

**核心：谎言由嫌疑人主动说出（止损），不是 Zack 喂话后被动否认。**

### 正确模式

```
Round N-1 揭穿 → 嫌疑人被迫承认部分事实
                → 嫌疑人主动缩小范围/淡化说辞（= 新谎言）
                → Zack 确认/记录嫌疑人的说法
                → 嫌疑人加码承诺（= Lie 步骤）
Round N    → Zack 出示证据打破新谎言
```

### 错误模式 ❌

```
Zack: "你和 Vinnie 之间有资金往来。"    ← Zack 喂话
NPC: "荒谬，没有。"                     ← 被动否认 = 弱
```

### 正确模式 ✓

```
NPC: "我承认我以前叫 Russo...但我们之间没有金钱关系。"  ← 主动撇清
Zack: "你说没有金钱关系。"                              ← 复述确认
NPC: "没有，一分钱都没有。"                              ← 自己加码 = Lie
→ Zack 用连号钞票证据打破
```

## 八、Expose 审查要点

- [ ] 谎言是嫌疑人**主动说出**的（止损），不是 Zack 喂话
- [ ] 每轮之间：嫌疑人被揭穿后**主动缩小范围**引出新谎言
- [ ] `Lie` 步骤的 `ParameterStr0` 中的证据 ID 真实存在
- [ ] Zack 引用证据时的描述与 `ItemStaticData` / `TestimonyItem` 一致
- [ ] 引用实物证据时，必须核对 `Describe` 和 `ArtRequirement` 字段，不可凭想象添加
