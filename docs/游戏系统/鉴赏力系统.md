# 鉴赏力系统（Connoisseur System）

## 一、系统定位

在对话和调查中插入**三选一节点**，测试玩家对之前获取信息的记忆和理解。

**核心原则：答案永远来自游戏内信息积累，不考验玩家现实知识。**

## 二、交互机制

触发时弹出三个选项：

| 结果 | NPC 反应 | 奖励 |
|------|---------|------|
| **选对** | 态度积极，对话更丰富（多说 1-3 句世界观细节） | 成就分值上涨 |
| **选错** | 态度冷淡/敷衍（1 句带过） | 不扣分但不加分 |
| **"我先想想"** | 允许暂时离开找答案再回来（高压场景除外） | — |

## 三、六大成就领域

| 徽章 | 领域 | 对应场景 |
|------|------|----------|
| 🥃 Mixologist | 调酒鉴赏 | 酒吧、地下酒馆 |
| 🎷 Jazz Cat | 爵士通 | 歌舞厅、爵士俱乐部 |
| 🔫 Arms Expert | 军火识别 | 枪店、犯罪现场 |
| 🃏 Card Sharp | 赌场老手 | 赌场、地下牌局 |
| 👔 High Society | 上流绅士 | 派对、豪宅 |
| 🕵️ Street Smart | 街头智慧 | 码头、贫民区 |

每个领域三档：铜 → 银 → 金

**结局关联**：
- 单领域金 → 解锁该领域彩蛋片段
- 多领域金 → 特殊结局变体
- 全领域金 → 终极彩蛋

## 四、信息来源限定

只有两种合法来源：
1. **对话** — 当前 Loop 或之前 Loop 中 NPC 已说过的内容
2. **证据/道具** — 已收集的 TestimonyItem 或 ItemStaticData 中的描述

### 证据 ID 与可引用时间（EPI02）

| ID 前缀 | 所属循环 | 最早可引用 |
|---------|---------|-----------|
| 21xx | Loop 1 | Loop 1 起 |
| 22xx | Loop 2 | Loop 2 起 |
| ... | ... | ... |
| 26xx | Loop 6 | Loop 6 起 |

写 Loop N 节点时，只能引用 21xx ~ 2Nxx 范围内的证据。

## 五、信息距离要求

- 信息来源与测试点之间应有**足够距离**（最好跨场景或跨 Loop）
- **禁止**同一段对话"刚说完就考"
- 理想：Loop N 测试 Loop 1~N-1 的旧信息，或当前 Loop 其他场景已给出的信息

## 六、对话格式（非考题格式）

鉴赏力节点必须写成**标准对话分支**，不是考题。

### 错误写法 ❌
```
[鉴赏力节点 #001]
问题：Frank拿什么做担保？
A. 退休金 B. 房产 ← 正确 C. 鞋坊经营权
```

### 正确写法 ✓
```markdown
<!-- 鉴赏力 #001 | 🕵️ Street Smart | 信息来源: L2 ohara_001 209001006 -->

### 206001050 `branches`
**扎克·布伦南** [盯着贷款协议]
> 一个退休码头工人，您觉得他能拿什么做抵押？
> - ❶ 他的退休金 → `206001051`
> - ❷ 他名下的房产 → `206001052`
> - ❸ 鞋坊的经营权 → `206001051`

### 206001052
**哈罗德·摩尔** [手指一顿]
> ……看来您对南区居民的情况很了解。是的，房产是唯一有价值的抵押物。
→ 汇合至 `206001008`

### 206001051
**哈罗德·摩尔** [微笑摇头]
> 具体的担保条款属于银行内部信息。
→ 汇合至 `206001008`
```

## 七、格式要点

| 元素 | 要求 |
|------|------|
| 元数据注释 | `<!-- 鉴赏力 #编号 \| 领域 \| 信息来源: 具体出处 -->` |
| 入口 | 标准 `branches` 分支 |
| 选对反应 | NPC 积极，**多说 1-3 句**世界观/细节（奖励性信息） |
| 选错反应 | NPC 冷淡，**1 句带过**，不给额外信息 |
| 汇合 | 必须汇合回主对话流，不影响主线 |

## 八、设计红线

- 选对加分，选错**不扣分**（维持"不终止不回档"调性）
- 不存在信任度数值系统，态度差异只是即时反馈
- 答案永远来自游戏内信息
- 高压场景取消"我先想想"
- 所有实质性内容奖励推到结局后

## 九、审查清单

- [ ] 信息来源是否在节点触发前**玩家已接触过**？
- [ ] 引用的证据 ID 是否在当前 Loop 范围内（21xx ~ 2Nxx）？
- [ ] 信息来源与测试点之间是否有足够距离？
- [ ] 是否以对话分支格式书写（不是考题）？
- [ ] 选对/选错的 NPC 反应是否有明显内容差异？
- [ ] 节点是否自然融入对话流？
