## 一、游戏结构

```
章节（Chapter）= 一个完整案件（EPI01/EPI02/EPI03）
└── 循环（Loop）= 子阶段，每章最多 6 轮
    └── 每轮有独立的调查目标、NPC 对话、证据池和指证对象
```

每轮循环对应一个 `ChapterConfig` 条目（如 `id=101` = Unit1 Loop1），定义：
- 开场对话（`initTalk`）→ Talk 表
- 初始场景（`initScene`）→ SceneConfig 表
- 本轮疑点（`doubts[]`）→ DoubtConfig 表
- 指证方案（`exposes[]`）→ ExposeData 表
- 指证对象（`exposeNpcId`）→ NPCStaticData 表

## 二、单循环核心流程

```
Emma 引入本轮目标
    ↓
场景探索 → 收集物证（点击场景物品）
    ↓
NPC 对话 → 获取证词（script="get"）
    ↓
CASE BOARD → 分析物品 / 合成物品 → 发现隐藏信息
    ↓
INVESTIGATE → 整理时间线 / 整理人物关系 → 发现矛盾
    ↓
疑点解锁（条件满足后自动弹出）
    ↓
全部主线疑点收集完毕 → 解锁 EXPOSE
    ↓
指证 NPC（多轮揭穿谎言）→ 循环结束 → 进入下一轮
```

## 三、一条推理链的完整数据流（Loop1 实例）

```
玩家进入场景 1103（Tommy 办公室）
  → SceneConfig.NPCInfos 加载 NPCLoopData 1001（Tommy 实例）
    → TalkInfo 指向 Talk 105001001（首次对话）

对话推进到 Talk 103001004（Rosa 证词句）
  → script="get", ParameterInt=1031002
    → 获得 TestimonyItem 1031002「Rosa 从 9 点半在地下室打扫」

INVESTIGATE 面板操作
  → 将 1031002 拖入时间线 → triggerType=1 条件满足
  → 获取物品 1115（排班表）→ 背包中拥有

DoubtConfig 1101 的两个条件满足
  → type=3(时间线) param=1031002 ✓
  → type=1(道具) param=1115 ✓
  → 疑点解锁："Rosa 当晚究竟在哪工作？"

4 个疑点全部解锁 → EXPOSE 按钮亮起

进入指证
  → ExposeData: testimony=1031002, item=["1115"]
  → Rosa 说谎 (Script="Lie") → 玩家出示 1115
  → 验证通过 → Round 2... → Round 3... → Script="end" → Loop1 结束
```

## 四、数据表关联总览

```
ChapterConfig (章节主配置)
├── initTalk ────→ Talk (开场对话链)
├── initScene ───→ SceneConfig (初始场景)
├── exposeNpcId ─→ NPCStaticData (指证对象)
├── doubts[] ────→ DoubtConfig → condition[].param → Item / TestimonyItem
└── exposes[] ───→ ExposeData → testimony → TestimonyItem
                              → item[] → ItemStaticData
                              → talkId → Talk

SceneConfig (场景)
├── NPCInfos[] ──→ NPCLoopData → NPC / TalkInfo / LoopTalkInfo
└── ItemIDs[] ───→ ItemStaticData

Talk (对话链)
├── Speaker → NPCStaticData
├── next → Talk (链式)
├── branches → Parameters[].ParameterInt → Talk (分支跳转)
├── get → ParameterInt → Item / TestimonyItem
└── change_scene → ParameterStr → SceneConfig

Testimony → evidenceItem[] → TestimonyItem → triggerType (时间线/关系网)
```

## 五、关键设计理念

| 原则 | 说明 |
|------|------|
| **操作驱动的低门槛推理** | 系统自动检测矛盾，玩家通过拖拽/分析/合成"参与推理" |
| **不终止不回档** | 选错不扣分，指证错误可重试，鉴赏力选错只是少获信息 |
| **疑点引导指证** | 收集完毕后自然进入指证，玩家不会卡关 |
| **谎言由嫌疑人驱动** | 谎言是嫌疑人的"止损"行为，Zack 出示证据而非指控 |
| **信息来自游戏内** | 所有测试答案源自已接触的对话/证据 |
| **提前收集与好奇心** | 可在当前 Loop 获得后续 Loop 的疑点 |
