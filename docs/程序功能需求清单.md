# 程序功能需求清单

> Unit9 设计过程中识别出的 3 个功能需求。本文档只写**我们需要什么**，不写怎么实现。

---

## 需求 1：插入剧情

**需求**：在满足特定条件时，自动播放一段玩家无法跳过的剧情段（Talk 链，可能包含视频、切场景、发证据）。播完自动推进到下一个游戏状态或回到自由探索。

**与普通对白的区别**：不是玩家点 NPC 触发，玩家不能选分支。

**触发条件分两类**：

### 1.1 现有已支持的触发条件

| 触发时机 | Unit9 用到的地方 |
|---|---|
| Loop 开场 | L5 灵魂黑夜 |
| 指证完成后 | L1 72h 交涉 / L4 Morrison 强押 / L5 James 自杀 + 感情戏 / L6 结局 |
| 首次进入某场景（无前置条件） | L3 首次进 James 厨房 → James 销毁节目单 |

### 1.2 本次需要新增的触发条件

| 触发时机 | 前置条件 | Unit9 用到的地方 |
|---|---|---|
| 进入某场景 | 背包里已有指定道具 | L6 进 Morrison 家客厅（9013）+ 背包里有借款记录 → Whale 电话响 |

**约束**：
- 单道具前置即可（不需要多道具 AND、也不需要 OR）
- 不需要"访问过某场景 / 完成过某对话"这类条件
- 每段剧情只触发一次

## 需求 2：道具拾取触发对白

**需求**：玩家在场景里捡到某件指定道具的瞬间，自动触发一段短对白（Zack 独白 / Zack 和 Emma 的对话 / Zack 心理活动）。

**为什么不能用现有方式代替**：
- 道具描述（Describe）只是静态文字
- 对话里 `get` 发道具是"对话中发给你"，而不是"你捡到它之后反向触发对话"

**Unit9 要用的地方**：

| 道具 | Loop | 拾取后对白 |
|---|---|---|
| James 给 Whale 的回信（9503） | L5 | Emma：这字迹和纸条 9302 是同一个人 |
| Berlitz 英语练习书（9504） | L5 | Zack：J 的笔锋倒着写——James 真名不是 J 开头 |
| Anna 移民通知书（9505） | L5 | Zack：非法入境……这就是 Whale 的把柄 |
| 赌债欠条（9603） | L6 | Emma：Morrison 欠 Whale 的——这就是他的动机 |
| 找回的 .45 弹壳（9401） | L6 | Zack：它居然还在，Morrison 没敢销毁 |

**约束**：一件道具只在首次拾取时触发一次。

---

## 需求 3：多人指证场景

**需求**：指证时现场除了被指证的 NPC，还有其他 NPC 在场——他们的立绘要能显示出来，关键时刻能插话，关键时刻能变表情。

**现状**：现在的指证界面只有上下两个立绘位（被指证者 + Zack），没有第三人位。

**Unit9 要用的地方**：

| Loop | 指证目标 | 同场 NPC | 作用 |
|---|---|---|---|
| L4 | Vivian | Rosa | Rosa 旁观反应强化"共犯"叙事 |
| L6 终局 | Morrison | Vivian + Emma | Vivian 给出最终证词 / Emma 推理介入 |

**需要支持的三种表现**：
- 静默立绘（只站着不说话，比如 L4 Rosa）
- 插话（指证某轮关键节点插 1-2 句，比如 L6 Vivian）
- 表情反应（指证成功/失败时旁观 NPC 换表情）

## 三个需求的 Unit9 使用总量

| 需求 | Unit9 必须 | 出现次数 |
|---|---|---|
| 1 插入剧情 | ✓ | 7 段过场 + 1 处突发事件（Whale 电话） |
| 2 拾取对白 | ✓ | 5 处 |
| 3 多人指证 | ✓ | 2 场（L4、L6） |

## 需求 4：对话触发动态漫画（小方块图）

**需求**：对话过程中，在画面左侧或右侧弹出一张小方块图片（可能带轻微动态效果），类似《逆转裁判》"异议！"选中物证后出现的物证小图。

**通用规则**：不是 Unit9 专属需求，是一个可在任意对话节点调用的通用表现手段。

**配置项**：
- 图片资源
- 出现位置（左 / 右，可能需要可配置坐标）
- 出现时机（绑定到对话某一句）
- 消失时机（下一句 / 指定句 / 手动关闭）

**与现有"发证据弹窗"的区别**：发证据是全屏或居中的弹窗，玩家需要关掉才能继续；动态漫画小方块是**画面侧边出现、不打断对话流**的表现层元素。

## 待确认：对话触发道具获取

> 不确定现在是否已有该功能，如果已支持请忽略此条。

**需求**：在对话流程中（非拾取场景道具）直接把一件道具塞进玩家背包。

**与现有 `get` 的关系**：如果对话里的 `get` 字段已经能实现"对话中发给你"，那这条就是已有功能，无需新增。请程序侧确认。
