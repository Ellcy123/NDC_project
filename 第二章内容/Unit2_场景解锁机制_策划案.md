# Unit 2 场景解锁机制 - 策划案

## 核心逻辑

**解锁流程**：前置条件满足 → 触发解锁事件 → 地点解锁 → Map显示动效 → 玩家可进入

---

## 场景分类

游戏中的场景分为两大类：

**硬切场景**
- 包括：开篇、结尾、指证、突发事件
- 特点：剧情强制切换，不经过Map，无需满足条件

**Map探索场景**
- 包括：自由探索阶段的所有场景
- 特点：需要满足前置条件，通过Map选择进入

---

## 前置条件类型

| 条件类型 | 说明 |
|:---|:---|
| 剧情阶段 | 当前处于特定剧情阶段 |
| 道具持有 | 玩家拥有特定道具 |
| 证据获取 | 玩家已获取特定证据 |
| 对话完成 | 玩家已完成特定对话 |
| 场景访问 | 玩家已访问过特定场景 |
| 跨循环继承 | 前循环已解锁的场景 |

---

## 解锁类型

| 解锁类型 | 说明 |
|:---|:---|
| 硬切 | 开篇/结尾/指证/突发事件，剧情强制切换，无需条件 |
| 自动解锁 | 同一建筑或相邻区域，进入自由探索时自动可用 |
| 地图解锁 | 通过地图道具批量解锁一组地点 |
| 证据解锁 | 获取特定证据后解锁新地点 |
| 道具解锁 | 拥有特定道具（如钥匙）后解锁 |
| 剧情解锁 | 完成特定剧情事件后解锁 |
| 已解锁 | 前循环已访问，本循环继续可用 |

---

## Map交互流程

1. 系统检测到条件满足
2. 触发对应地点的解锁事件
3. Map界面显示该地点图标亮起 + 解锁动画
4. 玩家点击图标进入该场景

---

## Map动效类型

| 动效类型 | 使用场景 |
|:---|:---|
| 淡入显示 | 普通解锁 |
| 脉冲高亮 | 重要地点首次解锁 |
| 连锁解锁 | 批量解锁（如地图解锁多个地点） |

---

## 设计要点

1. **硬切场景不经过Map**：开篇/结尾/指证/突发事件直接黑屏切换，玩家无法选择
2. **自由探索场景通过Map选择**：玩家在Map上点击已解锁的地点进入
3. **解锁状态跨循环继承**：一旦解锁的场景，在后续循环中保持可用
4. **条件驱动解锁**：通过道具、证据、对话、剧情事件等控制玩家探索节奏
