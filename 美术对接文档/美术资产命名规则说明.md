# NDC Episode 1 - 美术资产命名规则说明

## 📋 规则概述

本文档定义了NDC第一章所有美术资产的统一命名规范，确保资产管理的一致性和可追溯性。

---

## 🏗️ 核心命名结构

### 1. 场景ID编号规则

**格式**：`SC` + `循环编号` + `场景固定编号`

**说明**：
- `SC`：Scene的缩写，固定前缀
- `循环编号`：1-6，代表第几个循环
- `场景固定编号`：01-99，每个独立场景的全局唯一编号

**示例**：
```
SC101 = 循环1 + Rosa储藏室（场景固定编号：01）
SC102 = 循环1 + 歌舞厅一楼走廊（场景固定编号：02）
SC103 = 循环1 + Tommy办公室（场景固定编号：03）

SC201 = 循环2 + Rosa储藏室（同一场景，固定编号保持：01）
SC202 = 循环2 + 歌舞厅一楼走廊（固定编号保持：02）
```

**重要原则**：
- 同一物理场景在不同循环中，场景固定编号保持一致
- 场景固定编号全局唯一，便于跨循环资产复用

## 🎨 资产分类命名规则

### 2. 线索/证据资产

**格式**：`SC[循环编号][场景编号]_clue_[证据编号]`

**示例**：
```
SC101_clue_111  → 循环1-Rosa储藏室-证据111（芝加哥警局通缉令）
SC101_clue_114  → 循环1-Rosa储藏室-证据114（沾有氯仿的毛巾）
SC102_clue_121  → 循环1-歌舞厅走廊-证据121（氯仿瓶）
```

**编号说明**：
- 使用文档中已有的证据编号（如证据111、证据112...）
- 证据编号通常为三位数字

---

### 3. 物品资产

**格式**：`SC[循环编号][场景编号]_item_[物品编号]`

**示例**：
```
SC101_item_001  → 循环1-Rosa储藏室-普通物品001
SC102_item_001  → 循环1-歌舞厅走廊-普通物品001
```

**编号说明**：
- 物品编号从001开始递增
- 区分于证据，用于非关键交互物品

---

### 4. 背景资产（特殊规则）

**格式**：`SC00[场景固定编号]_bg_[场景描述]`

**示例**：
```
SC001_bg_储藏室      → Rosa储藏室的背景（对应场景固定编号01）
SC002_bg_走廊        → 歌舞厅一楼走廊的背景（对应场景固定编号02）
SC003_bg_办公室      → Tommy办公室的背景（对应场景固定编号03）
```

**重要原则**：
- 背景资产使用独立编号系统，从SC001开始
- 背景编号的**最后一位数字 = 场景固定编号**
- 背景资产可跨循环复用（同一场景在不同循环中使用同一背景）

**对应关系表**：
| 场景固定编号 | 场景名称 | 背景编号 |
|---------|---------|---------|
| 01 | Rosa储藏室 | SC001_bg_储藏室 |
| 02 | 歌舞厅一楼走廊 | SC002_bg_走廊 |
| 03 | Tommy办公室 | SC003_bg_办公室 |
| ... | ... | ... |

---

### 5. NPC角色资产

**格式**：`SC[循环编号][场景编号]_npc_[角色标识]`

**示例**：
```
SC101_npc_Rosa      → 循环1-Rosa储藏室-NPC Rosa
SC103_npc_Tommy     → 循环1-Tommy办公室-NPC Tommy
SC104_npc_Rosa      → 循环1-酒吧大堂-NPC Rosa
```

**编号说明**：
- 角色标识可使用角色名称或编号
- 同一NPC在不同场景出现时，场景编号不同

---

### 6. 环境叙事资产

**格式**：`SC[循环编号][场景编号]_environment_[编号]`

**示例**：
```
SC101_environment_001  → 循环1-Rosa储藏室-环境叙事001
SC102_environment_001  → 循环1-歌舞厅走廊-环境叙事001
```

**编号说明**：
- 环境叙事用于氛围营造、世界观展示等非核心证据
- 编号从001开始递增

## 🔄 跨循环资产复用示例

### 同一场景在不同循环中的命名

**Rosa储藏室**在不同循环中：
- 循环1：`SC101`（场景） + `SC001_bg_储藏室`（背景）
- 循环2：`SC201`（场景） + `SC001_bg_储藏室`（背景 - 复用）
- 循环3：`SC301`（场景） + `SC001_bg_储藏室`（背景 - 复用）

**证据资产**因循环不同而不同：
- 循环1中的证据：`SC101_clue_111`
- 循环2中的新证据：`SC201_clue_211`

## ✅ 命名规则优势

1. **唯一性**：每个资产都有全局唯一标识
2. **可追溯**：通过编号快速定位循环、场景、资产类型
3. **可复用**：背景资产通过固定编号实现跨循环复用
4. **可扩展**：编号系统支持后续章节扩展
5. **易管理**：清晰的分类便于资产管理和版本控制

---

## 📝 注意事项

1. **证据编号**：严格使用文档中已定义的证据编号（111、112、121...）
2. **场景固定编号**：同一物理场景必须使用相同的固定编号
3. **背景编号对应**：背景编号的最后一位必须等于场景固定编号
4. **中文描述**：编号后的中文描述仅用于可读性，实际资产文件名使用编号
5. **大小写**：建议统一使用大写SC，小写类别标识（clue, item, npc等）
