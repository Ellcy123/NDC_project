## 📋 规则概述

本文档定义了NDC第一章所有美术资产的统一命名规范，确保资产管理的一致性和可追溯性。

**v2.0 更新内容：**

## 🏗️ 核心命名结构

### 1. 场景ID编号规则

**格式**：`SC` + `循环编号` + `场景固定编号`

**说明**：
- `SC`：Scene的缩写，固定前缀
- `循环编号`：1-6，代表第几个循环
- `场景固定编号`：01-99，每个独立场景的全局唯一编号

**示例**：
```
SC101 = 循环1 + Rosa储藏室（场景固定编号：01）
SC102 = 循环1 + 歌舞厅一楼走廊（场景固定编号：02）
SC103 = 循环1 + Tommy办公室（场景固定编号：03）

SC201 = 循环2 + Rosa储藏室（同一场景，固定编号保持：01）
SC202 = 循环2 + 歌舞厅一楼走廊（固定编号保持：02）
```

**重要原则**：
- 同一物理场景在不同循环中，场景固定编号保持一致
- 场景固定编号全局唯一，便于跨循环资产复用

## 🎨 资产分类命名规则

### 2. 背景资产（特殊规则）

#### 2.1 基础背景命名

**格式**：`SC00[场景固定编号]_bg_[场景英文名]`

**示例**：
```
SC001_bg_storageroom     → Rosa储藏室背景
SC002_bg_corridor        → 歌舞厅一楼走廊背景
SC003_bg_office          → Tommy办公室背景
SC004_bg_barhall         → 酒吧大堂背景
SC005_bg_crimescene      → Webb办公室（案发现场）背景
```

**命名原则**：
- 使用**英文小写**描述场景
- 多单词连写（storageroom，不使用storage_room）
- 背景编号从 SC001 开始
- 背景编号最后一位 = 场景固定编号

#### 2.2 同场景多角度图

**格式**：`SC00[场景固定编号]_bg_[场景英文名][序号]`

**示例**：
```
SC001_bg_storageroom1    → 储藏室视角1
SC001_bg_storageroom2    → 储藏室视角2
SC001_bg_storageroom3    → 储藏室视角3
```

**使用场景**：
- 同一场景需要多个不同角度/区域的背景图时使用
- 序号从1开始递增

#### 2.3 场景局部放大图

**格式**：`SC00[场景固定编号]_bg_[场景英文名]_[局部位置英文]`

**示例**：
```
SC001_bg_storageroom_bed           → 储藏室-床的特写
SC001_bg_storageroom_toolbox       → 储藏室-工具箱特写
SC003_bg_office_desk               → 办公室-书桌特写
SC004_bg_barhall_bar               → 酒吧大堂-吧台特写
SC005_bg_crimescene_safe           → 案发现场-保险柜特写
```

**使用场景**：
- 玩家点击场景中特定区域触发局部特写时使用
- 用于解谜、证据发现等互动场景

### 3. 证据资产（Clue）

#### 3.1 基础命名格式

**格式**：`SC[循环][场景]_clue_[编号]`

**编号规则**：
- 使用**两位数字**：01-99
- 按发现顺序或重要性递增
- 每个循环-场景组合独立编号

**示例**：
```
SC101_clue_01   → 循环1-储藏室-证据01（通缉令）
SC101_clue_02   → 循环1-储藏室-证据02（女儿照片）
SC101_clue_03   → 循环1-储藏室-证据03（医院清单）
SC102_clue_01   → 循环1-走廊-证据01（氯仿瓶）
```

#### 3.2 三种尺寸规则

**证据资产需要提供三种尺寸：**

| 尺寸类型 | 命名格式 | 用途 | 示例 |
|---------|---------|------|------|
| 小图（原始） | `SC[循环][场景]_clue_[编号]` | 背包/列表显示 | `SC101_clue_01` |
| 大图 | `SC[循环][场景]_clue_[编号]_big` | 详细查看界面 | `SC101_clue_01_big` |
| UI图标 | `SC[循环][场景]_clue_[编号]_icon` | 界面图标/缩略图 | `SC101_clue_01_icon` |

**完整示例**：
```
循环1储藏室的通缉令证据：
- SC101_clue_01         （小图：背包中显示）
- SC101_clue_01_big     （大图：点击放大查看）
- SC101_clue_01_icon    （UI图标：在推理面板上显示）
```

**尺寸说明**：
- **小图**：用于背包界面、证据列表等需要缩略显示的地方
- **大图**：玩家点击证据后的全屏详细查看，可能包含可交互热区
- **UI图标**：用于案情面板、推理连线等UI界面的小图标

### 4. 物品资产（Item）

#### 4.1 基础命名格式

**格式**：`SC[循环][场景]_item_[编号]`

**与证据的区别**：
- **clue（证据）**：关键线索，推动剧情，参与推理
- **item（物品）**：环境交互物品，解谜道具，非核心证据

**示例**：
```
SC101_item_01   → 循环1-储藏室-清洁工具箱（解谜物品）
SC102_item_01   → 循环1-走廊-垃圾桶（环境交互）
SC103_item_01   → 循环1-办公室-台灯（环境物品）
```

#### 4.2 三种尺寸规则

**物品同样需要三种尺寸：**
```
SC101_item_01          （小图）
SC101_item_01_big      （大图）
SC101_item_01_icon     （UI图标）
```

---

### 5. 证词资产（Note）

#### 5.1 基础命名格式

**格式**：`SC[循环][场景]_note_[编号]`

**用途**：
- 记录NPC证词、对话记录
- 可能**没有实际美术资源**（纯文本）
- 但需要统一编号便于策划管理

**示例**：
```
SC103_note_01   → 循环1-办公室-Tommy的第一轮证词
SC104_note_01   → 循环1-酒吧大堂-Rosa的第一轮证词
SC201_note_01   → 循环2-储藏室-Rosa的第二轮证词
```

**特殊说明**：
- note 类型通常**不需要三种尺寸**
- 如果有对应的美术资源（如信件、文件），则使用 clue 类型
- note 主要用于策划记录和系统引用

---

### 6. 环境叙事资产（Environment）

#### 6.1 基础命名格式

**格式**：`SC[循环][场景]_environment_[编号]`

**用途**：
- 氛围营造、世界观展示
- 非核心证据，但可以点击查看
- 增强沉浸感和叙事深度

**示例**：
```
SC101_environment_01   → 循环1-储藏室-墙上的家庭合影
SC102_environment_01   → 循环1-走廊-禁酒令宣传海报
SC104_environment_01   → 循环1-酒吧大堂-留声机
```

#### 6.2 三种尺寸规则

**环境叙事物品如果可交互，需要三种尺寸：**
```
SC101_environment_01          （小图）
SC101_environment_01_big      （大图）
SC101_environment_01_icon     （UI图标）
```

**如果仅作为背景装饰，可以不需要三种尺寸**

---

### 7. NPC角色资产

#### 7.1 场景内NPC命名

**格式**：`SC[循环][场景]_npc_[角色名]`

**示例**：
```
SC101_npc_Rosa          → 循环1-储藏室-NPC Rosa
SC103_npc_Tommy         → 循环1-办公室-NPC Tommy
SC104_npc_Rosa          → 循环1-酒吧大堂-NPC Rosa（不同场景）
SC201_npc_Rosa          → 循环2-储藏室-NPC Rosa（不同循环）
```

**说明**：
- NPC角色资产指的是**场景内的站位姿态图**
- 不包括AVG对话立绘（立绘有独立的命名系统）
- 同一NPC在不同场景/循环出现时，编号不同

#### 7.2 AVG立绘命名（独立系统）

**格式**：`[角色名]_[角度]_[表情]`

**示例**：
```
Jimmy_front_neutral     → Jimmy正面中性表情
Jimmy_front_angry       → Jimmy正面愤怒表情
Jimmy_side_worried      → Jimmy侧面担忧表情
Vivian_front_sad        → Vivian正面悲伤表情
```

**角度选项**：
- `front` - 正面
- `side` - 侧面
- `back` - 背面（少用）

**表情建议**（基于Episode 1统计）：
- 主角Zack：5个表情
- 搭档Emma：6个表情
- 真凶Jimmy：12个表情
- 其他主要NPC：8个表情
- 次要NPC：2-3个表情

## ✅ 命名规则优势

1. **唯一性**：每个资产都有全局唯一标识
2. **可追溯**：通过编号快速定位循环、场景、资产类型
3. **可复用**：背景资产通过固定编号实现跨循环复用
4. **可扩展**：编号系统支持后续章节扩展（Episode 2, 3...）
5. **易管理**：清晰的分类便于资产管理和版本控制
6. **国际化友好**：全英文命名便于多语言版本管理
7. **三种尺寸规范**：明确小图/大图/UI图的命名，避免混乱

## 📝 重要注意事项

### 1. 编号规则
- ✅ 证据编号使用**两位数**：01-99
- ✅ 场景固定编号使用**两位数**：01-99
- ❌ 避免使用三位数编号（旧版规则已废弃）

### 2. 英文命名
- ✅ 所有场景和位置使用**英文小写**
- ✅ 多单词**连写**（storageroom，不使用storage_room）
- ❌ 不使用中文或拼音

### 3. 三种尺寸
- ✅ 证据（clue）、物品（item）、环境叙事（environment）**必须**提供三种尺寸
- ✅ 证词（note）通常**不需要**三种尺寸
- ✅ 背景（bg）只需一个尺寸

### 4. 资产类型
- **clue**：核心证据，推动剧情
- **item**：解谜物品，环境交互
- **note**：证词记录，可能无美术资源
- **environment**：环境叙事，氛围营造
- **npc**：场景内NPC姿态

### 5. 场景编号一致性
- ✅ 同一物理场景在不同循环中**必须**使用相同的场景固定编号
- ✅ 背景编号的最后一位**必须**等于场景固定编号
- 例如：Rosa储藏室固定编号01，背景编号SC001

### 6. 文件命名实践
- ✅ 实际资产文件名使用完整编号（如 `SC101_clue_01_big.png`）
- ✅ 建议统一使用**小写**字母（sc, bg, clue等）
- ✅ 图片格式建议：PNG（透明背景）或JPG（不透明背景）
