# 循环叙事设计

## 5.1 "知识银河战士"模型（Knowledge Metroidvania）

学术论文 *"Play Again: A Decalogue to Designing Time Loops in Video Games"* 提出了一个核心概念：

> 时间循环游戏本质上是"知识银河战士（Knowledge Metroidvania）"——玩家获得的不是新能力，而是新知识，这些知识解锁新的路径和区域。

**银河战士的能力 → 循环游戏的知识：**

| 银河战士 | 循环叙事 |
|---------|---------|
| 获得"双重跳跃"能力 | 获得"Morrison 受贿"的知识 |
| 能力让之前无法到达的平台变得可达 | 知识让之前无法理解的证据变得有意义 |
| 回到旧区域发现新路径 | 回到旧证据发现新含义 |

**NDC 的知识门控链：**

```
L1 知识 → 解锁 L2
  "Rosa 在走廊目击"
  → 解锁"调查 Vivian"路径

L2 知识 → 解锁 L3
  "Vivian 有强烈杀人动机"
  → 解锁"直面 Vivian"路径

L3 知识 → 解锁 L4  ← 关键门控点！
  "凶器不匹配 + Morrison 反常"
  → 解锁"调查 Morrison"路径
  （如果没有 L3 的凶器发现，玩家没有理由转向 Morrison）

L4 知识 → 解锁 L5
  "Morrison 是 Whale 的棋子 + 还有真正的执行者"
  → 解锁"调查 Jimmy"路径

L5 知识 → 解锁 L6
  "完整时间线 + 录音带"
  → 解锁最终叙事
```

每个 Loop 的知识收获既是**奖励**（玩家感到进步），也是**钥匙**（打开下一扇门）。如果某个 Loop 的知识收获不能同时充当下一个 Loop 的钥匙，那么叙事链就会断裂。

## 5.2 Outer Wilds 的设计原则

Alex Beachum 在 GDC 2021 演讲 *"Sparking Curiosity-Driven Exploration Through Narrative"* 中分享了 Outer Wilds 的核心设计哲学：

**原则一：唯一的进度货币是玩家的知识**

Outer Wilds 没有经验值、没有装备升级、没有技能树。玩家唯一的成长是"知道了什么"。这意味着：
- 每个循环必须让玩家学到实质性的新东西
- "我已经知道这个了"的循环会让人厌烦
- 知识的获取本身就是游戏体验

**原则二：叙事网络 > 线性叙事**

Outer Wilds 的信息不是线性排列的，而是一张网络——任何信息都可能与其他信息产生新的联系。Beachum 的核心洞见：

> "叙事网络成为核心组件：玩家做什么变得重要，是因为它与玩家做的其他事情之间产生了联系。"

这对 NDC 的启示：不同 Loop 获得的证据之间应该能产生"交叉验证"——L2 的一条证据和 L4 的一条证据组合后，能得出在任何单一 Loop 中都无法得出的结论。

**原则三：较短的循环减少挫败感**

Outer Wilds 的循环时长约 22 分钟。Beachum 发现更短的循环能：
- 减少"丢失进度"的挫败感
- 鼓励玩家尝试不同的探索路线
- 让"失败→重来"的循环感觉像"快速迭代"而非"浪费时间"

NDC 每个 Loop 约 15-20 分钟的设计与此一致。

## 5.3 Zero Escape 的"知识锁"系统

系列创作者打越钢太郎（Kotaro Uchikoshi）在设计 Zero Escape 系列时使用了"知识锁"（Knowledge Lock）机制：

**核心设计：**
- 某些结局或分支在玩家经历特定事件/学到特定信息之前是**字面意义上不可进入的**
- 流程图（Flowchart）不只是界面元素，它**就是**游戏的时间旅行机制
- 一条分支学到的密码可以用来解锁另一条分支的门

**打越的设计方法论：**
- 使用表格列出所有场景的结果，与主流程图交叉匹配
- 先设计信息流向，再填充故事细节
- 这是一种工程化、系统化的叙事设计方法

**对 NDC 疑点（Doubt）系统的启示：**

NDC 的疑点就是"知识锁"的实现：
- `unlock_condition: "item:1513 + testimony:2051001"` = 需要特定知识组合才能解锁
- `clear_loop: 4` = 这把锁在 Loop 4 才能打开
- 疑点系统让玩家能看到"我还有什么不知道的"，提供了方向感

## 5.4 The Forgotten City 的"二级叙事"技法

Nick Pearce（The Forgotten City 开发者）在 Game Developer 访谈中分享了"二级叙事"（Secondary Narrative）技法：

> "当主叙事正在进行时，在背景中建立一条二级叙事线。这条线最终会浮出水面，产生出人意料的转折。"

**关键操作：**
- 主叙事吸引玩家的注意力（"谁杀了 Webb？"）
- 二级叙事在背景中悄悄铺垫（Whale 的存在、Margaret 的下落）
- 当主叙事解决后，二级叙事浮出水面成为新的主线

这完美描述了 NDC 的 Unit 1 → Unit 2 过渡：
- Unit 1 的主叙事：谁杀了 Webb → Jimmy
- Unit 1 的二级叙事：Whale 是谁？为什么要杀 Webb？Zack 的母亲 Margaret 在哪？
- Unit 2 的主叙事：追踪 Whale，营救 Margaret

## 5.5 Return of the Obra Dinn 的"分区灾难"结构

Lucas Pope 在设计 Return of the Obra Dinn 时使用了"分区灾难"（Compartmentalized Disaster）结构：

**核心思路：**
- 将一个巨大的谜题（全船 60 人的命运）拆分为多个独立的"灾难事件"（船上发生的 10 次事故）
- 每次灾难涉及 3-6 个角色，形成可管理的子问题
- 子问题之间有联系但可以独立解决

**矩阵式推理（Matrix-Based Deduction）：**
- 玩家填写一张"姓名 x 命运 x 死因"的矩阵
- 刚性的机械框架支持自由形式的推理
- 玩家知道"终点是什么"（填满矩阵），但不知道"怎么到达"

**对 NDC 的启示：**
- NDC 的每个 Loop 就是一个"分区灾难"——聚焦于特定 NPC 的谎言，形成可管理的子问题
- 疑点面板就是玩家的"矩阵"——列出所有未解决的问题，让玩家知道"还有什么没搞清楚"
- 关键：子问题（每个 Loop 的指证）必须有独立的满足感，同时推进整体谜题
