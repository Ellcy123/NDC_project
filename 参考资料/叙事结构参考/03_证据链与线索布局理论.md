# 证据链与线索布局理论

## 3.1 三线索法则（The Three Clue Rule）

由 Justin Alexander 在其博客 The Alexandrian 中提出的推理游戏设计核心法则。

**法则内容：**

> 对于你希望玩家得出的每一个结论，至少放置三条独立的线索指向它。

**为什么是三条：**

Alexander 的原话：

> "玩家大概率会错过第一条线索，忽略第二条，曲解第三条，然后做出某个不可思议的逻辑跳跃到达你想让他们到的地方。"

这不是夸张——在交互式媒体中，玩家的信息获取路径是不可预测的。有些玩家会仔细检查每个角落，有些玩家会跳过大段对话，有些玩家会关注完全出乎意料的细节。三条线索不是"冗余"，而是"保障"。

**三线索法则的反面——"逆三线索"：**

> 如果玩家能接触到任意三条线索，他们至少会得出一个结论。

这个反面同样重要：它意味着你可以在游戏世界中自由散布线索，让不同的玩家通过不同的路径得出相同的结论。这对非线性探索（如 NDC 的自由搜证阶段）至关重要。

**线索类型应该多样化：**

三条线索不应该是同一种类型。好的线索布局应该覆盖不同的信息渠道：

| 线索类型 | 例 | 适合的玩家 |
|---------|---|----------|
| 物证（Physical） | 地上的拖痕、氯仿瓶 | 喜欢仔细搜索场景的玩家 |
| 证词（Testimonial） | Rosa 的供述、Tommy 的谎言 | 喜欢和 NPC 交谈的玩家 |
| 环境细节（Environmental） | 音乐的时间、走廊的布局 | 观察力强的玩家 |
| 行为观察（Behavioral） | Morrison 急着结案、Jimmy 的反常镇定 | 注重角色心理的玩家 |
| 文书/记录（Documentary） | 工作记录卡、账本、日记 | 喜欢阅读文档的玩家 |

**NDC 的实例——"Vivian 不是凶手"这个结论需要的三条线索：**

| # | 线索 | 类型 | 在哪获取 |
|---|------|------|---------|
| 1 | Webb 的致命伤是刀伤，Vivian 带的是枪 | 物证 x 物证 | L3 道具箱 + 法医报告 |
| 2 | Vivian 敲门时有人回应"今晚不要再来"——但 Webb 可能那时已经死了 | 证词 x 推理 | L3 Vivian 供述 |
| 3 | 现场压痕与枪型不匹配 | 物证 | L3 会客室搜证 |

三条线索，三种获取方式，都指向同一个结论。即使玩家只注意到其中两条，也足够支撑"Vivian 不是凶手"的判断。

## 3.2 节点式场景设计（Node-Based Scenario Design）

同样出自 Justin Alexander，是三线索法则的系统化延伸。

**核心概念：**

将游戏世界视为一个节点网络。每个"节点"是一个场景/NPC/证据，每个节点包含指向其他节点的线索。

```
[Rosa 储藏室] ──线索──→ [Tommy 办公室]
      │                        │
      线索                     线索
      ↓                        ↓
[走廊现场] ──线索──→ [Webb 会客室]
```

**两种线索方向（The Two Prongs）：**

Alexander 区分了两种线索的功能：

1. **前进型线索（Proactive）**：引导玩家走向新的调查方向
   - "Rosa 提到看见 Vivian 当晚出现" → 引导玩家去找 Tommy 了解 Vivian

2. **验证型线索（Reactive）**：帮助玩家确认或否定已有的假设
   - "凶器是刀伤" → 帮助玩家否定"Vivian（带枪）是凶手"的假设

好的场景设计应该同时包含两种线索——既给玩家新的方向，又帮玩家检验已有的想法。

## 3.3 公平游戏原则（Fair-Play Rules）

推理小说的"黄金时代"（1920s-1940s）留下了两套影响至今的规则，它们定义了"公平的悬疑"：

**Knox 十诫（Ronald Knox，1929）——最适用于游戏设计的五条：**

| # | 规则 | 对游戏设计的启示 |
|---|------|----------------|
| 1 | 凶手必须在故事早期出场，且不能是读者能读到内心想法的人 | 凶手 NPC 必须在玩家可互动的范围内，但其内心不能被直接展示 |
| 5 | 不得有意外帮助侦探，侦探不得拥有无法解释的直觉 | 玩家不能靠"系统提示"解谜，所有推理必须基于已掌握的证据 |
| 7 | 侦探本人不得是凶手 | （基本准则） |
| 8 | 侦探不得使用未向读者展示的线索 | 破案证据必须是玩家已收集的，不能凭空出现 |
| 10 | 不允许存在未事先声明的双胞胎或模仿者 | Jimmy 模仿 Webb 声音必须有线索铺垫（如 Jimmy 的模仿能力被其他 NPC 提及） |

**Van Dine 二十则（S.S. Van Dine，1928）——最核心的四条：**

1. **读者必须与侦探拥有平等的解谜机会**——所有用于破案的线索必须明确呈现给玩家。
   - NDC 应用：每一轮 Expose 使用的证据必须是玩家已经收集到背包里的，不能是"系统给的"。

2. **不得有意对读者施加的诡计**——唯一允许的诡计是凶手对侦探施加的。
   - NDC 应用：NPC 的谎言是合法的迷惑；但系统不能用模糊的证据描述来故意误导玩家。

3. **解答揭示后必须是不可避免的——读者应该能回溯发现所有线索早已存在。**
   - NDC 应用：L5 指证 Jimmy 时，使用的所有证据必须是玩家在 L1-L5 中已经获取的。玩家回看时应该感叹"原来那个细节是这个意思"。

4. **凶手的动机必须是个人的**——不能是超自然力量或组织命令。
   - NDC 应用：虽然 Jimmy 受 Whale 雇佣，但他的动机还有个人层面（为妻子和孩子、自身的野心），这让他的行为更可信。

## 3.4 证据维度与指证设计

这一节特别针对 NDC 的 Expose 系统。

**一个好的指证轮次（Round）必须满足的条件：**

1. **谎言措辞精确匹配证据维度**——每轮的谎言必须精确到只有对应维度的证据能反驳。
   - 好的设计：谎言说"当晚9:30起在地下室打扫" → 用工作卡证明"你实际在走廊"。维度是"地点"。
   - 差的设计：谎言说"我什么都不知道" → 太笼统，任何证据都能反驳，没有推理感。

2. **同一维度的证据不跨轮拆分**——每轮用不同维度的证据打不同论点。
   - R1 打"地点"维度（工作记录 vs 声称位置）
   - R2 打"行为"维度（拖痕 vs 声称没看到）
   - R3 打"动机"维度（氯仿 vs 声称不知情）

3. **陷阱证据有逻辑**——错误选项不是随机的，而是"看起来相关但实际不构成反驳"的。
   - 好的陷阱：用"11:30的枪声"来反驳"当晚在地下室"——虽然时间相关，但枪声不能证明位置。
   - 差的陷阱：放一个完全无关的证据——玩家不会被迷惑，也学不到东西。

**逐轮升级（Escalation）的设计原则：**

好的 Expose 应该逐轮升级谎言的"层次"：

```
R1: 表面谎言（事实层面的否认）
    ↓ 被击穿
R2: 退守谎言（承认部分事实但改口）
    ↓ 被击穿
R3: 核心谎言（逼近真相时的最后防线）
    ↓ 被击穿
→ 真相揭示 + 新信息 + 疑点更新
```

每一轮击穿都应该让 NPC 退到一个更深层的谎言——像剥洋葱一样，从外层谎言逐渐逼近核心秘密。
