# NDC 项目具体应用

## 6.1 Unit 1 重构版的理论验证

用上述框架逐项检验 Unit 1 重构版的设计决策：

**起承转合结构验证：**

| 段落 | 对应 Loop | 理论要求 | 重构版实现 | 评估 |
|------|----------|---------|-----------|------|
| 起 | L1 | 建立框架 + 初始问题 | 玩家在犯罪现场醒来，被当作嫌疑人 | 有效——清晰的困境建立 |
| 承 | L2-L3前半 | 在框架内深化，让玩家相信 | Vivian 的动机被层层剥开，玩家被引导向"情杀"结论 | 有效——误导是"挣来的" |
| 转 | L3后半 | 框架崩塌，认知重构 | 凶器不匹配 + Morrison 反常行为 | 有效——不是新信息，而是已有信息的新含义 |
| 合 | L4-L6 | 在新框架下收束 | 阴谋揭露 → 真凶指证 → 情感收尾 | 有效——逐层收束 |

**三线索法则验证——以"Morrison 有问题"为例：**

玩家在 L3 结束时应该得出"Morrison 有问题"的结论。检查是否有三条独立线索：

| # | 线索 | 类型 | 获取方式 |
|---|------|------|---------|
| 1 | Morrison 在 Vivian "认罪"后立即以"情杀"结案，不做进一步调查 | 行为观察 | L3 剧情事件 |
| 2 | Morrison 现场检查时未使用任何测量工具（Rosa L3 证词） | 证词 | L3 与 Rosa 对话 |
| 3 | Morrison 之前在 L1 就急于逮捕 Zack，行为一直反常 | 跨循环积累 | L1-L3 行为模式 |

三条线索，三种类型。通过。

**情感先于事实验证：**

| 情感铺垫 (先) | 事实揭示 (后) | 间隔 | 效果 |
|-------------|-------------|------|------|
| L2：Webb 对 Vivian 的欺骗 | L3：Vivian 的行动与克制 | 1 Loop | 玩家理解 Vivian 的痛苦 |
| L1-L3：Morrison 的反常行为积累 | L4：Morrison 被收买的证据 | 2-3 Loops | 积累的怀疑被坐实 |
| L4：Jimmy 是最亲近 Webb 的人 | L5：Jimmy 是真凶 | 1 Loop | "近在咫尺"的冲击感 |

## 6.2 每个 Loop 的叙事功能清单

设计新 Loop 时，用这个清单验证叙事完整性：

```
□ 本 Loop 有明确的核心问题（玩家带着什么问题进来？）
□ 核心问题在 Loop 结尾得到解答（不能悬而未决地离开）
□ 解答的同时产生 2-3 个新问题（指向后续 Loop）
□ 至少有一条前序 Loop 的"种子"在本 Loop 发芽（呼应之前的铺垫）
□ 情感铺垫领先事实揭示约 1 个 Loop（先理解动机再知道行为）
□ 假线索有独立的内在逻辑（不是为了误导而误导）
□ 所有用于指证的证据在指证前已可获取（公平游戏原则）
□ 指证击穿后 NPC 的反应符合其谎言动机类型
□ 本 Loop 的知识收获能同时充当下一 Loop 的钥匙
□ 玩家回看时能发现"原来那个细节是这个意思"（重构认知体验）
```

## 6.3 常见叙事设计陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|---------|
| 信息倾泻 | 一个 Loop 揭示太多信息，后续 Loop 无话可说 | 严格执行"每 Loop 一层真相"，多余信息推迟到后续 Loop |
| 假线索没有回报 | 玩家追踪了错误方向但什么都没得到 | 假线索消除时必须揭示新信息（侧面信息也算） |
| 角色工具化 | NPC 只是"提供线索的机器"，没有自己的故事 | 每个 NPC 至少有一个与案件无关的个人维度 |
| 万能侦探 | Zack 替玩家说出结论 | Zack 只出示证据和指出矛盾，结论由玩家自己补完 |
| 证据巧合 | 关键证据靠"刚好路过发现" | 每条证据的获取应有逻辑动机（为什么玩家会去那里？） |
| 节奏断裂 | 连续两个 Loop 的情感基调相同 | 交替使用不同的情感色彩（紧张→温暖→震惊→沉重） |
| 知识门控断裂 | 某个 Loop 的知识不指向任何后续方向 | 每个 Loop 的最终揭示必须包含"接下来查什么"的明确指向 |
