## 附录 A：术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 起承转合 | Kishotenketsu | 四段式叙事结构：引入→展开→翻转→收束 |
| 红鲱鱼 | Red Herring | 故意误导的线索或嫌疑人 |
| 三线索法则 | Three Clue Rule | 每个结论至少需要三条独立线索支撑 |
| 知识门控 | Knowledge Gating | 用玩家已获得的知识控制可访问的内容 |
| 知识银河战士 | Knowledge Metroidvania | 用知识（而非能力）解锁新路径的游戏结构 |
| 知识锁 | Knowledge Lock | 需要特定信息才能解锁的内容节点 |
| 公平游戏 | Fair Play | 读者/玩家拥有与侦探相同的解谜信息 |
| 二级叙事 | Secondary Narrative | 在主叙事背景中铺设的辅助故事线 |
| 分区灾难 | Compartmentalized Disaster | 将大谜题拆分为独立可解的子问题 |
| 节点式设计 | Node-Based Design | 将游戏世界视为互相连接的信息节点网络 |
| 戏剧反讽 | Dramatic Irony | 观众/玩家比角色知道更多信息产生的张力 |

## 附录 B：快速决策参考卡

**"这个 Loop 的核心揭示放在这里合适吗？"决策树：**

```
玩家在之前的 Loop 中是否已经积累了对这个揭示的情感基础？
├── 是 → 可以揭示（"坐实"的满足感）
└── 否 → 这个 Loop 应该先铺垫情感基础
         └── 核心揭示推迟到下一个 Loop

这个揭示是否同时打开了"接下来调查什么"的方向？
├── 是 → 继续
└── 否 → 揭示中需要包含一个"但是..."或"然而..."的转折
         指向新的调查方向

玩家有没有至少三条独立线索已经指向这个结论？
├── 是 → 揭示会让玩家觉得"我早该想到的"（最佳体验）
└── 否 → 需要在本 Loop 的搜证阶段补充更多线索
         确保不同类型的线索覆盖

这个揭示是否让之前某条"看似无关"的细节突然有了新含义？
├── 是 → 恭喜，这是"恍然大悟"时刻，极佳设计
└── 否 → 可以的话，在前序 Loop 埋一个微小细节
         在这个揭示的瞬间被重新解读
```
