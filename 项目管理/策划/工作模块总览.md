## 全局管线总览

策划工作按**严格的先后依赖**分为 6 个阶段，每章（Unit/Episode）完整走一遍。

```
 Phase 1          Phase 2        Phase 3           Phase 4          Phase 5          Phase 6
 策划创作          State生成       对话工程           数字人制片         配置表            数据发布
 ─────────        ─────────      ─────────         ─────────        ─────────        ─────────
 故事提炼          State YAML     对话配置(MD→JSON)  声音设计          配置表填写         预览验证
     ↓                ↓          分支编辑            ↓               (14张XLSX)        数据同步
 大纲生成              ↓          重复对话           TTS语音                            Git发布
     ↓                ↓          证词凝缩            ↓
 人物设定              ↓          疑点配置           表情图片
 场景设计              ↓                              ↓
 证据设计              ↓                            口播视频
 指证谜题              ↓
 疑点设计              ↓
 小玩法/突发事件       ↓
 美术参考文档          ↓
```

| 阶段 | 子模块数 | 性质 | 仓库 |
|------|---------|------|------|
| Phase 1. 策划创作 | 10 | 核心创意设计 | NDC_project |
| Phase 2. State 生成 | 1 | 数据架构枢纽 | NDC_project |
| Phase 3. 对话工程 | 4 | 文本→可运行JSON | NDC (D:\NDC) |
| Phase 4. 数字人制片 | 4 | AI批量资产生产 | NDC (D:\NDC) |
| Phase 5. 配置表 | 1（14张表） | 数据录入 | NDC (D:\NDC) |
| Phase 6. 数据发布 | 3 | 验证与同步 | 双仓库 |

## Phase 1. 策划创作（NDC_project · 设计文档）

> 管线：故事提炼 → 大纲生成 → 人物/场景/证据/指证/疑点 → 

### 1.1 故事提炼

| 项目 | 说明 |
|------|------|
| 工作流入口 | `/episode-story-extractor` |
| 输入 | 原始故事文档 |
| 产出 | 三大核心要素（核心事件、立意、情感共鸣点） |
| 路径 | `剧情设计/Unit{N}/剧情/` |
| 内容 | 从原始故事中提炼核心事件、立意、情感共鸣点。6章宏观视角，明确本章在整体叙事中的位置 |
| 依赖 | 世界观文档（`剧情设计/00_世界观与角色/`）、五章大纲 |
| 难度 | ★★★★★（最核心的创意工作） |

## Phase 2. State 生成（NDC_project · 数据架构枢纽）

|------|------|

## Phase 3. 对话工程（NDC · 草稿→可运行JSON）

> 管线：对话配置(MD→JSON) → 分支编辑 → 重复对话 → 证词凝缩 → 疑点配置

### 3.1 对话配置（Talk + Expose 草稿 → JSON）

| 项目 | 说明 |
|------|------|
| 工作流入口 | `/avg-dialogue`（Phase 2 同步用） |
| 草稿生成 | `/generate-dialogue {N}` 或 `@dialogue-writer` |
| 审查 | `@dialogue-reviewer` |
| 产出 Phase 1 | `Loop{1-6}_对话草稿.md`（MD草稿），按NPC/场景拆分为60-80个独立MD文件 |
| 产出 Phase 2 | Talk.json + ExposeTalk.json（通过 `sync_to_json.py` 同步） |
| Talk路径 | `AVG/EPI{N}/Talk/loop{1-6}/` |
| Expose路径 | `AVG/EPI{N}/Expose/` |
| 草稿路径 | `AVG/对话配置工作及草稿/` |
| EPI01参考 | 59个Talk JSON + 6个Expose JSON |
| EPI02参考 | 61个Talk JSON + 6个Expose JSON |
| EPI03参考 | 60+个MD草稿（Phase 1进行中） |
| 核心约束 | 17条对话规则（见`AVG对话配置规则.md`）；Phase 1只改MD，Phase 2用户确认后才同步JSON |
| 依赖 | 2.1 State文件、1.3 人物设定 |
| 难度 | Talk ★★★★☆（量大，需角色声音一致性）；Expose ★★★★★（逻辑最密集） |
