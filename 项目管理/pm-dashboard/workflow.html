二、数据流向

正向：项目状态 → 仪表盘
PM 运行扫描
python scan_and_upload.py 或 /pm-dashboard
3 个扫描器并行执行
design_scanner (设计文档) + dev_scanner (配置表/代码) + art_scanner (美术资源)
生成 project_state.json
保存到 data/project_state.json，包含文件计数、覆盖率、同步状态
上传到 Supabase scan_snapshots 表
完整 JSON 存入云端，供 dashboard 读取
Dashboard 刷新显示最新数据
总览 Tab 显示项目扫描概况 + 各部门进度
反向：团队编辑 → 数据持久化
团队成员打开 Dashboard
浏览器直接打开 HTML 文件（或未来的 GitHub Pages URL）
修改任务状态/进度/备注
所有编辑操作实时写入 Supabase，无需手动保存
其他人刷新即可看到更新
数据在云端，不存在"改在本地别人看不到"的问题

文件说明类型

五、日常操作

PM 每日操作

运行扫描（每天一次）
在 Claude Code 中：/pm-dashboard
或命令行：cd 项目管理/pm-dashboard && python scan_and_upload.py
打开仪表盘
双击 dashboard.html，查看总览 → 检查各部门进度 → 关注逾期里程碑和 P0 任务
调整排期
修改优先级、调整截止日期、添加备注 → 数据自动保存到 Supabase
风险分析（可选）
在 Claude Code 中：/pm-dashboard analyze → AI 自动分析风险并给出建议

程序 / 美术 操作

打开仪表盘
打开 PM 分享的 Dashboard 链接（或本地 HTML 文件）
找到自己的任务
程序 → 程序 Tab（看板视图）；美术 → 美术 Tab（表格视图）
更新状态
修改状态（未开始→进行中→已完成） / 填写实际工时 / 更新进度百分比

六、命令参考

# ===== Claude Code Skill =====
/pm-dashboard            # 完整流程：扫描 → 上传 → 提示打开
/pm-dashboard scan       # 仅扫描，保存本地 JSON（不上传）
/pm-dashboard analyze    # AI 风险分析
/pm-dashboard init       # 重新导入种子数据（会清空！需确认）
/pm-dashboard open       # 显示仪表盘地址

# ===== 命令行 =====
cd 项目管理/pm-dashboard
python scan_and_upload.py            # 扫描 + 上传
python scan_and_upload.py --local    # 仅扫描，不上传
python scan_and_upload.py --auto     # 扫描 + 上传 + 自动更新 auto_link 任务进度
python seed_data.py                  # 重新导入种子数据

AI 分析Claude Code pm-analyst agent按使用计费

Supabase 注意事项

  • 免费版 500MB 数据库，6 人团队足够
  • 7 天无人访问会自动暂停 — 在 Supabase Dashboard 点 Restore 即可恢复（~1分钟）
  • anon key 是公开的（设计如此），安全性通过 RLS 策略控制
  • 当前 RLS 策略为"全员可读写"，适合内部小团队
