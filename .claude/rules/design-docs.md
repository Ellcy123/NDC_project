# 设计文档规则

## 场景描述
每个场景描述必须包含：场景名、解锁条件、NPC 列表、可获取证据、场景类型（🎬🔓🔒⏸️⚔️⚡🎭）

## 指证设计
每个指证设计必须包含：谎言层级（3-4 层）、每轮证据需求、击穿逻辑、指证后戏剧

## 证据设计
每个证据条目必须包含：ID、中英文名称、类型（itemType）、获取方式、中英文描述、note 字段

## 信息节奏
- 每 Loop 只揭示一层真相
- 后续 Loop 的信息在当前 Loop 必须完全隐藏或伪装成无关细节

## 下游影响
修改设计文档后必须检查下游影响：
- state 文件是否需要更新？
- 对话草稿是否需要修改？
- 配置表是否需要同步？
- 预览数据是否需要重新生成？
