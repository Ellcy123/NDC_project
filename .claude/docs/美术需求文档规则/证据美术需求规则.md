# 证据美术需求规则

适用于 NDC 项目所有 Unit 的证据美术需求文档（`剧情设计/Unit{N}/美术需求/Unit{N}_证据美术需求分类.md`）。

写证据美术需求和审稿时按以下规则检查。

---

## 规则一：每件证据分两段——"重点（必不可少）" + "美术参考（不影响推理）"

证据美术需求**必须分两段**：

### 重点（信息表达必不可少）

玩家必须从画面读到的关键信息——直接关系到推理正确性。例如：
- 关键数字（金额、利率、年份、序列号）
- 关键文字（姓名、地址、签名、印章、抬头）
- 关键状态（已抵押/未抵押、新钞/旧钞、连号缺口位置）
- 跨证据视觉关系（同批次、同抬头、同版式）

**写法准则**：每条重点都要能写成一句"美术做不到这个，玩家就无法完成 X 推理"的可验证陈述。

### 美术参考（不影响推理）

证据视觉的氛围、年代、风格、构图建议。例如：
- 1920s 真实物件的考据参考（材质、字体、配色）
- 背景搭配（拾取场景的环境融合）
- 道具的新旧、磨损、折痕程度

**写法准则**：本段错画了不影响游戏推理逻辑，只影响沉浸感。

---

## 规则二：跨证据视觉一致性必须显式声明

如果两件证据要让玩家"一眼看出关联"（同批次、同份合同的不同页、同款道具的不同状态），**必须在两件证据的"重点"段都标注**。

**示例**：
- 10202 钞票 + 10206 钞票 = 同 5$ 联邦储备券版式（连号缺口可视化）
- 10303 合订本第 5/7/12 条 = 同款排版 + 同款奶白合同纸 + 装订线贯穿
- 10301 Frank 真迹 + 10302 贷款档案签名 = 用同一支钢笔写但笔迹质感完全相反（颤抖 vs 工整）

文末"跨证据视觉一致性"表格汇总，避免美术散落处理。

---

## 规则三：派生证据要标注产出机制

派生证据（如 10701 由小玩法产出）的美术需求段必须开头声明：
- 来源（哪几件证据合成 / 哪个玩法产出）
- 形态（纸带 / 报告 / 拼合件 / 派生照片）
- 触发方式（"分析"按钮 / CASE BOARD 合成 / 玩法完成后自动产出）

让美术理解派生件的产出语境，避免画成"凭空出现的文档"。

---

## 规则四：ID 编码 + 抬头分类汇总

文档末尾必含两张汇总表：

### Loop 件数表

| Loop | ID 范围 | 件数 | 备注 |
|---|---|---|---|

### 视觉资源复用表

| 复用元素 | 涉及证据 ID |
|---|---|
| `XYZ` 统一抬头与 logo | 10201 / 10203 / ... |
| 同款字体/配色/版式 | ... |

---

## 规则五：待裁决项独立列出

文档末尾"待策划/审核员确认的设计点"列表收录：
- ID 编号尚未确定的
- 跨证据冲突待统一（如品牌名、年份、地址）
- 涉及历史考据需查证的（如法律编号、机构名）

**已裁决的项**用 `~~原条目~~ **已裁决**：{结论}` 格式标注，**保留原条目** 不直接删除——方便回溯设计决策。

---

## 配套审查清单

- [ ] 每件证据是否都有"重点"和"美术参考"两段？
- [ ] 重点段每条是否可以写成"美术做不到 X 玩家就无法推理 Y"的可验证陈述？
- [ ] 跨证据视觉一致性是否在双方"重点"都标注 + 文末汇总？
- [ ] 派生证据是否标注产出机制（来源 + 形态 + 触发）？
- [ ] 文档末尾有 Loop 件数表 + 视觉资源复用表 + 待裁决项？
- [ ] 待裁决项的"已裁决"标注是否保留原条目（不直接删除）？
