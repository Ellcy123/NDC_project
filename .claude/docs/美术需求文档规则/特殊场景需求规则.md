# 特殊场景需求规则

适用于 NDC 项目所有 Unit 的特殊场景文档（`剧情设计/Unit{N}/美术需求/Unit{N}_AVG特殊场景需求.md`）。

写特殊场景和审稿时按以下规则检查。

---

## 规则一：游戏结构三段——只覆盖"开篇"和"指证后特殊剧情"和"其他新加入的 AVG"

游戏核心循环：**开篇 → 自由探索 → 指证**。

本文档只覆盖：

### A 开篇（每 Loop 必有）

每个 Loop 的 opening_cutscene——即使内容大量复用既有立绘/场景，仍要单独列段，给美术对 Loop 入口的完整概念。

### B 指证后特殊剧情（有特殊美术需求才入）

post_expose 演出段——如 L3 Moore 钥匙交付（房产证 + 钥匙特写）/ L6 ENDING（多段独立特写）。

**指证轮次本身（R1-R4）不入本文档**——复用本场景背景 + NPC 大头，无新美术。

### C 其他新加入的 AVG（有特殊美术需求才入）

- Loop 间的过渡 cutscene（如 L5→L6 转场 Vinnie 上警车 + 电话亭 Foster 双消息）
- 系统级分支选择段（如 L5 律师/侦探分支选择 UI）
- 全 Unit ENDING 序列（每个 ENDING 段独立成节）

---

## 规则二：判断准则——是否需要"新建美术资源"

入本文档的核心标准：**需要新建背景图 / 新建立绘 / 新建道具特写 / 新建系统 UI**。

| ✅ 该入文档 | ❌ 不入文档 |
|---|---|
| 火灾后鞋坊新场景背景 | 自由探索内嵌的标准过场（电话亭/警局/银行大厅）|
| Mickey 首次出场的新立绘 | 复用既有立绘的 NPC 互动 |
| 房产证 + 钥匙道具特写 | 普通证据拾取（已在《证据美术需求》分类）|
| 分支选择系统 UI | Talk 内常规对白 |
| 字幕卡黑底白字版式 | NPC 标准 R1-R4 指证回合 |

**典型误判**：把 state 里 `type: cutscene` 自动当作"需要特殊美术"——其实 cutscene 只是触发模式（自动播放），美术上跟标准对白一样用复用资源。

---

## 规则三：每个特殊场景按 Unit9 v2.0 简洁模板

每段按以下结构写（**不要**节拍调度表 / 详细分镜 / 音效层 / 台词参考——那些归动态漫画/对白草稿）：

```
## §X · L{N} {简短标题}（{场景 ID}）

**类型**：opening_cutscene / post_expose_cutscene / transition_cutscene / ending_segment / ending_title_card
**触发**：{什么完成后自动触发 / 玩家做什么之后触发}
**衔接**：{本段结束后跳转到哪 / 衔接哪个文档段}

**叙事**：{1 段紧凑叙事，5-10 行，覆盖关键剧情节拍 + 角色动作}

**在场角色**：{角色列表}
**中途出场角色**：{...}（若有）
**中途退场角色**：{...}（若有）

**美术资产**：{新建项 bullet list}
```

---

## 规则四：复用资源要明确标注

每段"美术资产"中，**复用既有资源的部分要明确标注**——避免美术误以为要全部新建。

**示例**：
> 美术资产：病房 / Mickey 立绘 / Margaret 复用 §2；**Emma 端两杯咖啡进门的轻松姿态新立绘**（L2 §2 Emma 是关切站姿，本场是端咖啡进门，动作不同）

---

## 规则五：ENDING 序列每段独立成节

全 Unit ENDING 通常是**多段连续演出**（字幕卡 / 多个场景切换 / 关键道具特写）。

**每段独立成 §**：ENDING-1 字幕卡 + ENDING-2 第一段 + ENDING-3 第二段 + ENDING-4 最终段。

不要把多段塞进一个 § 内——美术对接时每段是独立需求，分开列才能各自评估工作量。

---

## 规则六：总览表 + 变更记录

文档开头必有总览表：

| 编号 | Loop | 场景 | 类型 |
|---|---|---|---|

文档结尾必有变更记录：

```
**变更记录**：
- {日期} v{X.Y}：{本次变动概要}
```

每次大改（增删段 / 重新编号 / 结构调整）都要更新版本号 + 留记录。

---

## 规则七：与其他美术需求文档的分工

| 内容 | 文档归属 |
|---|---|
| 证据本身的视觉需求 | 《证据美术需求分类》|
| 整段场景调度（开篇/post_expose/转场/ENDING） | **本文档** |
| 瞬间方格动态漫画 | 《突发事件动态漫画》|
| 自由探索阶段 NPC 立绘特殊姿势 | 《人物立绘需求》|

跨文档衔接时必须双向标注（如本文档 §X 衔接《突发事件》#N 时，两个文档都写引用）。

- [ ] ENDING 是否每段独立成 §？
- [ ] 是否有总览表 + 变更记录？
- [ ] 跨文档衔接是否双向标注？
