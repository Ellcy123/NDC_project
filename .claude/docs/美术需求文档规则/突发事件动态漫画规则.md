# 突发事件（动态漫画）规则

适用于 NDC 项目所有 Unit 的突发事件文档（`剧情设计/Unit{N}/美术需求/Unit{N}_AVG突发事件动态漫画.md`）。

写突发事件和审稿时按以下规则检查。

---

## 规则一：突发事件 = "瞬间方格"动态漫画，不是完整场景

突发事件文档**只收录**以方格动态漫画形式呈现的关键瞬间，不收录整段场景调度。区分如下：

### 该入本文档（动态漫画"瞬间方格"）

- ⚡ 突发事件：自由探索期玩家动作 / 自动触发 / 走位触发的突然中断点
- 🔵 插入剧情：NPC 突然介入 / 信息突然显形 / 不可预期的揭示瞬间
- turn_cutscene 内的高潮瞬间（如 Lula 隔窗举证那一帧）
- Expose 击穿后的某个特定戏剧化反应帧

### 不入本文档（归《特殊场景需求》）

- Loop 开篇整段场景调度
- post_expose 整段戏剧（含多句对白 + 多帧调度）
- ENDING 整段叙事

---

## 规则二：每件突发事件必含 7 段

每件突发事件按以下结构写：

```
## 事件 X · L{N} {简短标题}

**Loop / 场景**：L{N} / {场景 ID 与名称}
**类型**：⚡ 突发事件 / 🔵 插入剧情 / turn_cutscene 段
**触发条件**：{具体触发条件，玩家做什么 / 自动什么时间触发 / 哪个对白节点之后}

**叙事作用**：
- {1-3 条 bullet 说明本事件在推理/叙事/情感链中的作用}
- {要让美术理解"为什么要画这一帧"}

**在场角色**：{角色列表，主体 + 边缘可见的}

**美术资产**：待美术设计
- 关键帧 1：{机位 + 画面元素 + 角色姿势}
- 关键帧 2：{...}
- 关键帧 3：{...}
- 视觉重点：{这段动态漫画的"啊哈"帧是哪一帧、要让玩家看清什么}
- 画面外：{环境/背景音对应的视觉氛围支撑}

**音效**：
- {音效层 bullet list，节拍化}

**台词参考**（最终以对白草稿为准）：
- {角色}（动作）："{台词}"
- {...}
```

---

## 规则三：关键帧不超过 4 帧

动态漫画的本质是**节拍化方格**——一件事件 2-4 帧为宜。

- 单帧：纯瞬间反应（如表情骤变）
- 2-3 帧：动作 + 反应（递信→拒绝→Zack 收回手）
- 4 帧：复杂多角色互动（如 Tony 问 → Vinnie 否认 → Vinnie 失态 → Tony 收尾）

**超过 4 帧的事件应该拆分成两件事件**，或者归《特殊场景需求》而非突发事件。

---

## 规则四：触发条件要可验证

触发条件必须明确写出**程序能识别的状态**，不能含糊：

| ✅ 合格 | ❌ 不合格 |
|---|---|
| 进入 10015 Silver Moon 场景即触发 | 玩家进入赌场后某个时机 |
| Zack 回事务所时桌上电报浮现 | 调查告一段落时 |
| R3 Danny 攀咬"Lula 骗签遗嘱"后强制触发 | Lula 出场时 |

---

## 规则五：与《特殊场景需求》的衔接要双向标注

如果一件突发事件嵌在一个特殊场景里（如 L4 R3 Lula 隔窗举证嵌在厕所指证 turn_cutscene 内），**双方文档都要标注衔接**：

- 突发事件文档：注明"嵌在 §X 内"
- 特殊场景文档：注明"高潮瞬间见《突发事件》#N"

避免美术理解时两端各画一份重复内容。

---

## 规则六：台词参考标注"最终以对白草稿为准"

突发事件的台词参考是**给美术理解情境用的**，不是最终对白。文档末尾必须显式标注：

> **台词参考**（最终以对白草稿为准）

避免对白草稿落地后台词被改，美术按本文档台词画了反而对不上。

---

## 配套审查清单

- [ ] 是否真的是"瞬间方格"动态漫画？还是其实是整段场景？
- [ ] 每件事件是否有完整 7 段结构？
- [ ] 关键帧不超过 4 帧？
- [ ] 触发条件是否可验证（程序能识别）？
- [ ] 与《特殊场景需求》是否有衔接？衔接是否双向标注？
- [ ] 台词参考是否标注"最终以对白草稿为准"？
