# 人物立绘需求规则

适用于 NDC 项目所有 Unit 的人物立绘需求文档（`剧情设计/Unit{N}/美术需求/Unit{N}_AVG人物立绘需求.md`）。

写人物立绘需求和审稿时按以下规则检查。

---

## 规则一：只收录"特殊立绘需求"，不写基础立绘清单

本文档**不**是 NPC 立绘资产 inventory（基础立绘清单由美术规范另行管理）。

本文档**只收录**：

### 该入本文档（特殊立绘需求）

- 特定**姿势**（如"蹲马桶上按肚子"、"持枪呆立"、"倚靠门框"）
- 特定**动作**（如"擦杯子的持续背景动作"、"递文件给门卫"、"望向楼梯的转身"）
- 特定**表情**（如"眼神涣散"、"嘴角往上扯一下"、"突然身体一紧"）
- 特定**位置关系**（如"画面边缘旁听"、"半身裁切出现在窗格"、"画面最低点的窘态轮廓"）

### 不入本文档（基础立绘）

- 标准站姿 / 坐姿 / 走路 / 表情中性
- 常规对话立绘（已由美术基础规范覆盖）
- 突发事件文档 + 特殊场景需求文档已记录的立绘需求

---

## 规则二：按 Loop × 场景结构，每条立绘一行

文档结构：

```
## Loop {N}

### 场景 {ID} {场景名}

- {NPC 名}：{特定立绘需求一句话}
- {NPC 名}：{特定立绘需求一句话}

### 场景 {ID} {场景名}

- {...}
```

每条立绘一行就够，不要长段解释——美术需要的是简洁列表，不是叙事描述。

---

## 规则三：参考 Unit9 v2.0 立绘描述风格

Unit9 v2.0 范例（高质量参考）：

| ✅ 合格 | ❌ 不合格 |
|---|---|
| "Vivian 持枪呆立、眼神涣散" | "Vivian 在房间正中，看起来很慌张" |
| "Morrison 蹲姿勘验" | "Morrison 在尸体旁工作" |
| "James 侧背对门站灶台，刀起刀落，不正面转身" | "James 在厨房做饭" |
| "Rosa 缩肩坐姿、腰背不靠墙" | "Rosa 坐在角落里" |
| "Anna 手放腹部、门缝半遮的脸、背光剪影" | "Anna 在门口紧张地站着" |

**写法准则**：
- 用具体身体部位 + 具体动作（不是抽象情绪）
- 用空间位置（"画面右侧"、"门缝里"、"小窗外"）
- 用一句话——超过一句就太碎

---

## 规则三-B：抽象描述 = 没有立绘需求，直接不写

如果某 NPC 在某场景**写不出具体身体部位 + 具体动作**，只能写出抽象的情绪/心理状态描述（"赎罪感"、"裂缝感"、"专业冷静"、"紧张转变"），**说明没有真正的立绘需求**——美术只能凭感觉发挥，但本质上是用标准 NPC 立绘 + 标准对话表情就够了。

**禁止写法举例**（这些都该删，不该列）：

| ❌ 删 | 删除理由 |
|---|---|
| "Morrison：看完法医报告后的长停顿中抬头的眼神接触、拿笔签搜查令时的沉默动作（与 L1 立绘的'疲惫'质感延续，但本场赎罪感更强）" | 全部抽象——"长停顿"、"赎罪感"、"质感延续"。具体动作只有"拿笔签字"，但这是标准动作不需美术专门画 |
| "Leonard：新配眼镜的调整细节（摘下/推上的习惯动作），Zack 靠近时身体坐直的紧张转变（与 L2/L3 立绘的'专业冷静'对比，开始裂缝）" | "调整细节"模糊，"紧张转变"抽象，"开始裂缝"是叙事质感不是动作 |
| "Tony：与 Zack 交互时的自然吧台动作 + 递出钞票的随性手势" | "自然"、"随性"是抽象形容词，缺具体身体动作 |

**通过判断**（这些才可以列）：

| ✅ 列 | 通过理由 |
|---|---|
| "Vinnie 上身用力前压、声音大半度" | 身体部位（上身）+ 具体动作（前压）+ 声音变化（大半度） |
| "Danny 一只手按肚子、一只手抓裤腰，蹲马桶上" | 多个身体部位 + 具体动作 + 空间位置 |
| "Lula 在窗格内侧举起协议给 Danny 看" | 具体动作（举起）+ 道具（协议）+ 视角（隔窗内侧）|
| "Morrison 站客厅中央，夹烟手垂在身侧" | 具体位置 + 具体身体部位（夹烟手）+ 具体姿势（垂在身侧）|

**核心判断**：闭眼想象，美术按你的描述能画出**完全相同的姿势**吗？如果靠美术自由发挥才能呈现，就是没具体到位——删掉，让美术用标准立绘渲染。

## 规则四：指证轮次的立绘需求——取决于是否有 post-expose 特殊 AVG

指证流程（R1-R4）本身**复用本场景背景 + NPC 大头**，原则上**不产生新立绘需求**。

**例外条件**——以下情况才在本文档列指证场内的 NPC 立绘：

| 条件 | 例子 | 是否列立绘 |
|---|---|---|
| 指证场环境本身不常规（如 Danny 蹲马桶 / Vivian 化妆室 / James 灶台前背身） | L4 §10020 厕所 Danny 蹲马桶 | ✅ 列（场景特殊性，NPC 大头表达不了） |
| 指证后接特殊 AVG / ENDING 序列（如 Leonard 招供后接 ENDING / Moore 钥匙交付） | L6 §10017 Leonard 4 轮防守递进 → ENDING-3 法庭假释 | ✅ 列（与 post_expose 立绘连续） |
| 指证后 NPC 直接被带走（警察押送 / 离场 / 无后续戏） | L5 §10015 Vinnie 招供后直接上警车 | ❌ 不列（NPC 大头足够） |
| 指证场是标准 AVG 渲染（NPC 站工位 / 坐桌后） | L2 §10011 Leonard 在大厅工位 | ❌ 不列 |

**判断核心问题**：这场指证之后，该 NPC 还要"活下去"做有立绘需求的戏吗？
- 是 → 指证场内的立绘演变可以列（防守递进 / 表情裂缝 → 与 post-expose 立绘形成连续）
- 否 → 指证场内 NPC 立绘归零，大头表情足够

---

## 规则五：与其他美术文档的分工

| 立绘类型 | 归属文档 |
|---|---|
| Loop 开篇 / post_expose / 转场 / ENDING 内的 NPC 立绘 | 《特殊场景需求》|
| 突发事件动态漫画内的 NPC 立绘 | 《突发事件动态漫画》|
| **自由探索阶段 + 其他文档未覆盖的 NPC 特殊立绘** | **本文档** |
| 基础 NPC 立绘 inventory | 美术规范，不入设计文档 |

**核心判断**：写立绘需求前先问——这条需求是不是已经被《特殊场景需求》或《突发事件》某段覆盖了？是的话不重复，只在本文档加 NPC 名 + "（见 §X / 突发事件 #N）"指引即可。

---

## 规则六：场景下无特殊立绘需求时显式标注

如果某 Loop 某场景的 NPC 都用标准立绘，**显式写**："无特殊立绘需求"——避免美术猜测是否漏写。

**示例**：
```
### Loop 3
**无特殊立绘需求**（L3 场景 10011/10013/10014 的 Leonard、Moore 为标准站位）
```

---

## 规则七：跨场景一致性 NPC 立绘要汇总

某 NPC 在多个场景有立绘需求时，文档末尾用"NPC 立绘汇总"表格汇总各立绘版本，便于美术理解总量：

```
## NPC 立绘汇总

| NPC | 立绘版本 | 出现位置 | 详见 |
|---|---|---|---|
| Vinnie | 吧台坐姿（紧绷） | L2 自由探索 | 本文档 |
| Vinnie | 否认 Russo 上身前压 | L2 突发 #2 | 《突发事件》|
| Vinnie | 招供爆发表情 | L5 Expose | state/loop5 |
| Vinnie | 戴铐涣散 | L5→L6 §7 | 《特殊场景》|
```

## 配套审查清单

- [ ] 是否只收录"特殊立绘需求"（不写基础立绘）？
- [ ] 每条立绘是否一句话简洁描述（具体身体部位 + 动作）？
- [ ] **是否避免抽象描述**（规则三-B）：闭眼想象，美术按描述能画出**完全相同的姿势**吗？模糊的就删
- [ ] 是否避免重复《特殊场景需求》《突发事件》已记录的立绘？
- [ ] **指证场内的立绘是否符合规则四**（场景特殊 / 接 post-expose 特殊 AVG 才列；NPC 直接被带走则不列）？
- [ ] 无特殊立绘需求的场景是否显式标注？
- [ ] 跨场景多版本立绘的 NPC 是否在末尾汇总？
- [ ] 立绘描述是否用具体名词 + 动作（避免抽象情绪形容词）？
