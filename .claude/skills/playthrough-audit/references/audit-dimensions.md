# 审计维度定义与评分标准

7 个独立审计维度，每个维度由专属 agent 执行，输出独立报告。

---

## D1: 玩家流程模拟 (Player Experience Simulation)

**执行 Agent**: player-simulator
**核心问题**: 一个普通玩家能否自然地走完推理路径并获得满足感？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 推理路径连贯性 | 20 | 每步推理都有信息支撑=20；存在1处断层=10；多处断层=0 |
| "啊哈"时刻质量 | 20 | 铺垫充分+惊喜感强=20；铺垫不足=10；无明显"啊哈"时刻=0 |
| 信息获取节奏 | 20 | 密度均匀、节奏舒适=20；局部过密/过稀=10；严重失衡=0 |
| 玩家动力持续性 | 20 | 全程有悬念驱动=20；中段动力下降=10；多处失去方向=0 |
| 路径鲁棒性 | 20 | 任何探索顺序体验一致=20；部分顺序体验差=10；存在必须特定顺序=0 |

### 问题严重度判定

| 严重度 | 判定条件 |
|--------|---------|
| Critical | 玩家可能卡关（推理断层无替代路径）；核心谜题无法合理推出 |
| Major | 推理断层存在但有弱替代路径；某 Loop 信息严重失衡 |
| Minor | 单点信息过密/过稀但不影响整体；分支吸引力不均但不致命 |
| Suggestion | 可增加额外暗示提升体验；可调整信息顺序优化节奏 |

## D2: 证据链路完整性 (Evidence Chain Integrity)

**执行 Agent**: consistency-checker
**核心问题**: 每条证据是否有完整的生命周期？是否存在断链或冗余？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 链路完整率 | 20 | 100%证据有完整链路=20；>90%=15；>80%=10；<80%=0 |
| 孤立证据检测 | 20 | 无孤立证据=20；1-2个=10；>2个=0 |
| 证据使用效率 | 20 | 所有证据至少被引用1次=20；>90%=15；<90%=10 |
| 陷阱证据质量 | 20 | 每个陷阱有"看似相关实则不构成反驳"的逻辑=20；部分牵强=10 |
| 跨 Loop 属性一致 | 20 | 同一证据跨 Loop 无属性矛盾=20；有轻微不一致=10；有明显矛盾=0 |

### 问题严重度判定

| 严重度 | 判定条件 |
|--------|---------|
| Critical | 指证所需证据不可获取；疑点解锁条件引用不存在的证据 |
| Major | 证据链断裂（获取了但无处使用）；陷阱证据逻辑不成立 |
| Minor | 证据描述跨 Loop 有微小不一致；使用效率偏低但不影响游戏 |
| Suggestion | 可增加证据间的交叉引用；可优化陷阱证据的迷惑性 |

---

## D3: 人物一致性 (Character Consistency)

**执行 Agent**: narrative-designer
**核心问题**: NPC 的知识、行为、性格是否跨 Loop 自洽？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 知识边界严守 | 20 | 无泄露=20；1处轻微暗示=10；明确泄露后续 Loop 信息=0 |
| 说谎逻辑自洽 | 20 | 谎言有内在逻辑且前后一致=20；存在自相矛盾=10；谎言漏洞明显=0 |
| 性格语气稳定 | 20 | 跨 Loop 语气一致（合理变化除外）=20；偶有偏离=10；判若两人=0 |
| 态度转变合理 | 20 | 态度变化有剧情驱动=20；部分突兀=10；完全无铺垫=0 |
| 关系网络一致 | 20 | NPC 间关系描述一致=20；有矛盾描述=10；严重冲突=0 |

### 问题严重度判定

| 严重度 | 判定条件 |
|--------|---------|
| Critical | NPC 泄露了后续 Loop 的核心真相；谎言自相矛盾导致推理崩溃 |
| Major | 知识边界有灰色地带可能被玩家察觉；性格突变无铺垫 |
| Minor | 语气细微偏差；关系描述用词不完全统一 |
| Suggestion | 可增加性格细节强化一致性；可优化态度转变的过渡 |

---

## D4: 对话质量 (Dialogue Quality)

**执行 Agent**: dialogue-reviewer
**核心问题**: 对话是否符合 12 项审查标准？是否有跨 Loop 的模式问题？

### 评分维度（12 项审查清单，每项 PASS=满分/WARN=半分/FAIL=0）

| # | 审查项 | 满分 | PASS 条件 |
|---|--------|------|----------|
| 1 | 知识边界 | 10 | NPC 未泄露 blind_spots 或 withheld_topics |
| 2 | 信息分配 | 8 | NPC 间无信息重复 |
| 3 | 证词覆盖 | 10 | 所有 testimony_id 有对应 get 标记 |
| 4 | 物品描述准确性 | 8 | 对话中证据描述与 ItemStaticData 一致 |
| 5 | 角色语气一致性 | 8 | NPC 说话方式符合人设 |
| 6 | 对话链完整性 | 10 | 无断链、死节点、不可达分支 |
| 7 | 证词获取节奏 | 6 | 两次 get 间隔 ≥3-5 句 |
| 8 | ID 编码正确性 | 10 | 9 位格式、无重复 |
| 9 | 指证谎言机制 | 10 | Lie 由嫌疑人主动说出（止损式） |
| 10 | source 区分 |
| 11 | 玩家视角测试 | 8 | Zack 只引用直接感知；无 As-You-Know-Bob |
| 12 | 平行场景隔离 | 6 | 未引用同 Loop 其他🔓场景信息 |

**总分**: 100

### 跨 Loop 模式问题（额外检查）

- 同一类 FAIL/WARN 在 ≥3 个 Loop 出现 → 模式问题
- 模式问题比单点问题优先级更高（说明系统性缺陷）

## D5: 时序一致性 (Timeline Consistency)

**执行 Agent**: timeline-auditor
**核心问题**: 信息是否严格按循环节奏揭示？跨 Loop 数据是否一致？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 信息泄露零容忍 | 20 | 无跨 Loop 泄露=20；1处微泄露=5；明确泄露=0 |
| known_facts 流转 | 20 | 所有 Loop 流转正确=20；1处遗漏/多余=10；多处错误=0 |
| 证据属性一致 | 20 | 同一证据跨 Loop 属性完全一致=20；有差异=0 |
| 单层真相验证 | 20 | 每 Loop 严格只揭示一层=20；有轻微越界=10；明显越界=0 |
| NPC 知识时序 | 20 | active/withheld_topics 与 Loop 进度匹配=20；有错配=0 |

### 问题严重度判定

| 严重度 | 判定条件 |
|--------|---------|
| Critical | 后续 Loop 核心真相在当前 Loop 被明确说出 |
| Major | 证据属性矛盾可能让玩家困惑；known_facts 遗漏导致对话不合理 |
| Minor | 极轻微的暗示但不影响推理；known_facts 多了一条无害信息 |
| Suggestion | 可收紧 withheld_topics 范围；可优化信息揭示的精确度 |

## D6: 难度曲线 (Difficulty Curve)

**执行 Agent**: player-advocate
**核心问题**: 难度是否合理递进？玩家是否在正确的位置获得正确难度的挑战？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 整体递进合理 | 20 | L1简单→L6复杂的渐进曲线=20；有倒退=10；大起大落=0 |
| 信息密度控制 | 20 | 每场景≤3信息点+每Loop≤10信息点=20；偶有超标=10；严重超标=0 |
| 推理断层检测 | 20 | 无断层=20；1处=10；多处=0 |
| 指证难度匹配 | 20 | 指证难度与证据铺垫匹配=20；部分证据不足=10；需要碰运气=0 |
| 挫败感控制 | 20 | 无高风险挫败点=20；1处=10；多处=0 |

## D7: 叙事连贯性 (Narrative Coherence)

**执行 Agent**: narrative-designer
**核心问题**: 故事是否有完整的情感弧线？伏笔是否回收？主题是否递进？

### 评分维度（每项 0-20 分，满分 100）

| 子维度 | 满分 | 评分标准 |
|--------|------|---------|
| 情绪弧线匹配 | 20 | 实际情绪与设计情绪弧高度匹配=20；部分偏离=10；严重偏离=0 |
| 主题递进 | 20 | 个案→体制→道德灰色的递进清晰=20；有断层=10；混乱=0 |
| 伏笔回收率 | 20 | >90%伏笔被回收=20；>70%=15；>50%=10；<50%=0 |
| 场景氛围一致 | 20 | 场景氛围与剧情阶段匹配=20；偶有不搭=10；严重不搭=0 |
| L4 转折质量 | 20 | 成功建立道德灰色地带=20；尝试但不充分=10；未达成=0 |

## Critical 发现验证规则（所有维度通用）

**任何 agent 在提出 Critical 级别问题前，必须执行源文件验证：**

1. **不得仅凭 brief 摘要定 Critical** — brief 是从 state 文件生成的二手信息，可能与实际对话 JSON 内容不一致
2. **必须读取相关源文件的关键段落**（对话 JSON 的前 5-10 条 cnWords、ItemStaticData 中的对应条目等），确认问题确实存在
3. **区分"state 描述"与"实际内容"** — 如果 brief 说某对话是"开篇引导"但实际 JSON 内容是 post-expose 场景，以实际 JSON 为准
4. 验证后在报告中标注 `[已验证: 读取了 {文件名}]` 或 `[未验证: 基于 brief 推断]`

**反面案例**：brief 标注 emma_004 为"L2开篇"，多个 agent 据此判定"开篇破梗=Critical"。但 emma_004 的实际 JSON 内容是 Expose 后的讨论场景，信息来自 post_expose_knowledge，完全合规。如果任一 agent 读了 JSON 前 3 行就能发现这是误报。

## 跨维度交叉规则

以下情况需要在交叉分析阶段特别关注：

| 交叉维度 | 典型关联 |
|---------|---------|
| D1 + D2 | 玩家推理断层 ↔ 证据链断裂（同一根因） |
| D1 + D6 | 玩家挫败点 ↔ 难度曲线异常（同一位置） |
| D3 + D4 | NPC 性格漂移 ↔ 对话语气不一致（同一 NPC） |
| D3 + D5 | NPC 知识泄露 ↔ 时序违规（同一 Loop） |
| D4 + D5 | 对话内容泄露 ↔ 信息泄露（同一对话文件） |
| D6 + D7 | 难度异常 ↔ 叙事节奏失衡（同一 Loop） |
| D1 + D7 | 玩家动力下降 ↔ 情绪弧线断裂（同一阶段） |
