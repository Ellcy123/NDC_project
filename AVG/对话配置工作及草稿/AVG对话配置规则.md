# AVG 对话配置规则

> 基于第一章（EPI01）6轮循环的完整配置数据总结的信息披露与对话设计规则。
> 配置新对话、新循环、新章节时应遵守以下规则。

---

## 一、分支对话规则

### 1. 分支必须指向不同的信息维度

每组分支选项（通常2-3个）必须分别指向不同类别的信息点，不能让两个分支揭示同一维度的内容。

常见信息维度：

| 维度 | 说明 | 举例 |
|------|------|------|
| 身份/关系 | 人物背景、职业、与其他角色的关联 | "Webb是做什么生意的？" |
| 时间线 | 具体时间点、在场证明、行动轨迹 | "今晚你在哪儿？" |
| 事件/线索 | 发生的异常事件、可获取的物证 | "昨晚有什么异常？" |
| 情绪/态度 | 角色的心理状态、威胁暗示、动机 | "你在害怕什么？" |
| 背景/环境 | 人物来历、场所信息、时代背景 | "Foster什么来头？" |

**典型示例** — Tommy首次对话 (105001005)：
- ❶ "Webb是做什么生意的？" → **身份/关系**（Webb的生意本质）
- ❷ "今晚你在哪儿？" → **时间线**（Tommy 22:00在办公室，获取TIM009）
- ❸ "昨晚有什么异常？" → **事件**（23:30枪声，获取TIM001）

### 2. 分支内可独占获取关键信息

不同分支路径内允许有各自独占的 `get` 获取（证据/证词/时间线标记），玩家单次对话只能走一条路径。这是鼓励玩家回访NPC的核心机制。

**示例** — Emma与Foster电话 (202101010)：
- ❶ 骨骼分析 → 获取"死者骨骼不像女性"信息
- ❷ 起火原因 → 获取证据 2021002（消防队长结论）
- ❸ Foster背景 → 获取证据 2021003（1912港口事故）

### 3. Repeat对话负责补完未走过的分支

每个NPC的 `_repeat.json` 中的分支选项应直接指回首次对话中的分支路径，确保：
- 玩家不会永久错过任何信息
- 需要主动回访NPC才能获取完整信息
- 重访对话更短更直接，无需重复铺垫演出

### 4. 分支选项的语气差异影响演出但不影响核心信息

当分支是"态度选择"（如"你刚才说漏嘴了" vs "没事慢慢说"）时，两条路径最终获得的关键信息（keyInfoType/keyInfoContent）应当相同，但NPC的反应、情绪表现、台词措辞可以不同。

**目的**：让玩家感受到选择的重量（角色互动差异），但不因态度选择而错过关键线索。

### 5. 所有分支最终汇合到同一节点

无论玩家选择哪条分支，对话链最终必须收束到同一个汇合点继续推进。禁止出现分支导致对话永久分叉或遗漏后续内容的情况。

## 二、信息获取层级规则

### 6. 证据有「直接获取」和「派生获取」两层

| 层级 | 获取方式 | 举例 |
|------|---------|------|
| 直接获取 | 对话中 `script:"get"` 或场景物品点击 | 1114(沾湿毛巾), TIM001(枪声时间) |
| 分析派生 | 对直接证据执行"分析"操作 | 1114→1701(氯仿毛巾) |
| 合成派生 | 两个证据组合 | 1320+1425→1429(综合鉴定报告) |

### 7. 每个场景/NPC承载1-3个核心信息点

单个场景或单次NPC对话不应承载超过3个核心信息点（不含纯装饰/氛围信息）。过多会让玩家信息过载，过少会让场景感觉空洞。

| 信息点数 | 适用场景 |
|----------|---------|
| 1 | 过渡场景、补充信息场景 |
| 2 | 标准调查场景 |
| 3 | 核心剧情场景（首次对话、关键NPC） |

### 8. 对话中的 keyInfoType 标记必须准确分类

每句包含关键信息的对话通过 `keyInfoType` 标记其信息类型：
- `timeline`：时间线相关（时间点、行动轨迹）
- `statement`：陈述类（声明、否认、自述）
- `identity`：身份/关系类（职业、人物关联）

标记的准确性直接影响玩家收集信息时的分类体验。

## 三、跨循环信息递进规则

### 9. 每轮循环只揭示一层真相

每个Loop有一个核心揭示主题，不应在同一Loop内解决多个层面的疑问：

| Loop | 揭示层 | 核心发现 |
|------|--------|---------|
| 1 | 犯罪事实确认 | 有人被下药、被拖拽、被枪杀 |
| 2 | 执法者腐败 | Morrison时间线矛盾、提前准备工具、赌债+Whale |
| 3 | 经济犯罪网络 | Webb的勒索帝国、假古董交易 |
| 4 | 嫌疑人浮现 | Jimmy偷盗、收Whale的钱、无不在场证明 |
| 5 | 情感动机补全 | Vivian被背叛、曲目掩护、复仇日记 |
| 6 | 真相拼图 | 弹道+音量+门锁+排班+Whale雇凶 |

### 10. 疑点(Doubt)推荐跨类型信息交叉验证（不强制）

解锁一个疑点**推荐**两种不同来源或类型的信息组合，以增强推理厚度；单件强证据也允许直接解锁疑点，由设计师基于谜题语境判断：

| 交叉模式 | 举例 |
|----------|------|
| 证词 × 物证 | 时间线证词(1031002) + 物品(1115) → 疑点1101 |
| 物证 × 物证 | 分析后毛巾(1701) + 氯仿瓶(1121) → 疑点1102 |
| 多方证词交叉 | 三人证词(1052002+1041003+1061001) → 疑点1201 |
| 证词 × 财务证据 | 关系网(1071001) + 额外收入(1424) → 疑点1404 |

### 11. 指证(Expose)遵循「递进击穿」结构

每轮指证针对同一条核心证词，用不同维度的证据逐轮反驳，层层递进：

**示例** — Loop1指证Rosa（证词: "我从九点半开始在地下室走廊打扫"）：
1. Round 1: 工作卡显示"后台走廊" → **地点矛盾**
2. Round 2: 氯仿毛巾+氯仿瓶 → **Rosa被下药，非主动行为**
3. Round 3: 拖拽痕迹匹配男性 → **体能排除**

从「事实矛盾 → 物证逻辑 → 物理排除」逐步深入。

### 12. 后续Loop可复用前序Loop的证据

跨Loop证据复用是设计的核心机制，尤其最终Loop应从几乎所有前序Loop调取证据形成完整拼图：

**示例** — Loop6最终指证的5轮证据来源：
| Round | 证据来源 |
|-------|---------|
| 1 | Loop 6 — 弹道(1706) |
| 2 | Loop 6 — 节目单(1631) |
| 3 | Loop 5 — Vivian证词(1065001) |
| 4 | Loop 4 — 厨房排班(1414) |
| 5 | Loop 4+3 — 额外收入(1424)+Webb收入(1311) |

## 四、证词配置规则

### 13. 提取的证词道具信息量应与原文匹配

Testimony.words（证词原文）和 TestimonyItem.testimony（提取的证词道具摘要）之间的信息密度应当对等，不膨胀也不缩水。

**反面示例 — 缩水：**
- 原文："我从晚上<special=2>九点半开始就在地下室走廊打扫</special>，Webb先生月底要求彻底清洁"
- 提取道具写成："Rosa在打扫"
- 问题：丢了时间(九点半)、地点(地下室走廊)、原因(月底清洁)三个信息点，玩家会困惑"我明明读到了更多细节"

**反面示例 — 膨胀：**
- 原文："我一直在这儿"
- 提取道具写成："Tommy全程在办公室未离开，期间无人来访，排除了一切外出可能"
- 问题：原文只是笼统一句话，提取却扩写了大量推断，玩家会质疑"这些结论我从哪读到的？"

**正确做法：**
- 原文的 `<special>` 标记内容应与提取道具的核心陈述高度一致
- 提取道具可以精简措辞，但不应丢失原文中的关键限定词（时间、地点、人物、条件）
- 提取道具不应包含原文中未明确表述的推断或结论

### 14. 不要在证词中写出「看起来能提取但实际不能提取」的信息

证词原文中如果出现了具体的时间、地点、人名、数字等细节，玩家会本能地认为"这条应该能提取为证据"。如果实际没有对应的 TestimonyItem，玩家会反复尝试后感到困惑和挫败。

**规则：**
- 每段证词中，凡是包含**具体时间点、具体地点、具体人名关联、明显矛盾陈述**的句子，要么配上对应的 TestimonyItem 使其可提取，要么改写措辞使其不那么"像线索"
- `<special>` 标签标记的内容必须 100% 可提取——标记了就一定要有对应道具
- 纯氛围/情绪描写可以不可提取，但要注意不要在其中夹带过于具体的事实细节

**反面示例：**
- 原文："那天晚上<special=1>十一点十五分</special>我听到走廊有脚步声，像是从厨房方向过来的"
- 实际没有对应的 TestimonyItem
- 问题：special标签 + 具体时间 + 具体方位，三重"线索信号"让玩家确信这能提取，但提取不了

**正确做法（二选一）：**
- 方案A：配上 TestimonyItem — `{testimony: "11:15走廊有脚步声，来自厨房方向", triggerType: ...}`
- 方案B：改写为不可提取的氛围文 — "那天晚上走廊时不时有动静，也说不清是什么时候的事"（去掉精确时间和方位）

**核心原则：信息的"可提取感"和"实际可提取性"必须一致。**

## 五、补充规范

### 15. 证词、谎言与指证的关系

#### 证词 = 谎言的容器

Talk 阶段玩家通过 `get`（七位 ID）收集的证词（TestimonyItem）包含嫌疑人的陈述。Zack 在调查中发现这条证词可疑，因此发起 Expose 指证。

#### Expose 的 Lie 结构

- **Round 1 的 `Lie`**：对应 Talk 中已收集的那条证词——这是 Expose 的起点
- **后续 Round 的 `Lie`**：嫌疑人被揭穿后为了止损主动说出的新谎言（狡辩/缩小范围/淡化说辞），不是 Talk 中预先收集的证词，而是 Expose 过程中嫌疑人的临场反应

#### 流程

```
Talk: 收集证词（get → TestimonyItem）
  → Zack 发现证词可疑
  → 进入 Expose

Expose Round 1: Lie = Talk 中收集的证词（嫌疑人重申）
  → 玩家出示证据揭穿
  → 嫌疑人被迫承认部分事实，但主动缩小范围（新谎言）

Expose Round 2: Lie = 嫌疑人的新狡辩（非预先收集）
  → 玩家出示新证据揭穿
  → 嫌疑人再次止损......

依此递进，直到无法再辩解
```

#### 规则

- 每个 Expose 的第一个 `Lie` 必须对应一条 Talk 中已收集的证词（ExposeData.testimony 引用同一 TestimonyItem ID）
- 后续 `Lie` 是嫌疑人在 Expose 中被逼出的新谎言，由玩家用新证据揭穿
- 谎言的揭穿必须由玩家通过出示证据完成，不能在 Talk 对话中由 Zack 直接戳穿
- Talk 中 NPC 说的可疑信息如果不在 Expose 中被指证，不称其为"谎言"

#### 注意区分

- `后面要考` = 鉴赏力测试的信息来源，与 Expose 无关
- `Lie` = Expose 中的谎言步骤，必须有证据可揭穿

### 16. 对话中获取证据的节奏控制

连续获取证据的间隔不应过密。两次 `script:"get"` 之间至少隔3-5句普通对话，给玩家消化信息的空间。如果一段对话需要获取多个证据，应通过分支机制分散到不同路径中。

### 17. 视频资源路径一致性

每句对话的 `videoEpisode/videoLoop/videoScene/videoId` 必须与其所属的对话文件保持一致：
- `videoEpisode` = 章节标识（EPI01/EPI02）
- `videoLoop` = 循环标识（loop1-loop6）
- `videoScene` = 对话文件名（不含.json）
- `videoId` = 对话ID（与 `id` 字段一致）
