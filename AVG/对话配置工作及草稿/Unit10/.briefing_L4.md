## 二、破梗红线（CLAUDE.md 原则一·悬疑感优先·全局最高·违反=FAIL）

- **Mickey=Whale 绝不暴雷**：opening 中 Mickey 把控局势、对案件了解深，但只留"太准/太及时"的轻微余味，绝不写成"他有问题/可疑"。玩家 Unit10 内不能怀疑 Mickey 是反派。
- **Emma–Miller 伏笔潜伏**：state/大纲要 Emma 透露"医院是 Miller 集团资助的，所以 Margaret 医药费大大减轻"。按 emma.md 红线落地为**纯中性事实陈述**：Emma 不知 Miller 真实意义，说这句时无任何暗示语气、无"我发现了秘密"的表演痕迹、无停顿加重——就是顺口交代医药费为什么轻。玩家 Unit11+ 回看才懂。绝不由 Emma 口说出任何指向 Miller 阴暗面的暗示。
- **骗中骗不揭**：Danny 自以为交房产证给银行能"停催债+房子处置后分一大笔钱"稳赚；实际银行根本不会兑现给钱（Leonard 只是工具人）。L4 击破 Danny 时**不点破**这层，Danny 仍以为自己得了便宜，玩家此时不知道他也被骗，留后续 Loop 自己拼出"叔侄被同一台机器吃掉"。
- **Leonard 真凶(L6)不泄**；**纵火者身份 / Frank 真实死因(钝器) / 眼镜碎片** L4 锁定不揭。
- **Danny post_expose 目击证词**只描述"大块头、深色衣服、走路像意大利人、打晕了 Margaret、戒指反光顺手捡走"——**不点名 Vinnie**，Danny 本人不知道是谁（天黑没看清），这让证词更可信；L5 才用于指证 Vinnie。
- **Frank 小铁盒内容（举报材料+情书）L5 才揭**：Lula 知道有备用钥匙(10408)、知道"小铁盒以后会用到"，但**不知道/不说**里面具体是什么（withheld）。
- **Zack 不替玩家下结论**：可出证、指出矛盾、追问、回呛、带情绪、用私人重量顶，但不像旁白说"所以这说明…"；玩家自己补逻辑链。NPC 谎言/退守是 NPC 主动自然反应，非 Zack 喂话后被动否认。

## 四、文本规则 R1–R10（原样，必须遵守，违反=FAIL）

### R1 · 一来一回的对白，不是演讲
对白是 dialogue 不是 monologue。任何一方都不能被写成"一方播报/演讲 + 另一方点头/转述"。该追问就追问、该回呛就回呛，可打断、可半句、可语气词、可带情绪。沉默用动作括注承载，不写空台词。一段读下来要像两个人在说话，不像一个人在做报告。

### R2 · 先懂大纲，再写本场
动笔前必须读 Unit 大纲，弄清这段对话在整章里的叙事功能：揭示哪一层真相、为后面哪个 Loop 埋什么伏笔、玩家此刻该有的情绪、它在角色弧光的哪一步、与上下场如何衔接。对白为这个功能服务，不孤立写一场戏。Phase 2a 讨论与 2b prompt 必须显式写出"本场叙事功能"。

### R3 · 人设是角色声音的唯一依据
角色怎么说话，只看 `{npc}.md` 的"声音风格 / 禁忌 / 设计备注 / 关键关系"。照人设写，不照任何外部语言风格模板、不堆刻板族裔/阶级标签。每个 NPC 首句要让人不看档案也认得出是谁。人设里的口头禅、反讽标记、禁忌、设计核心是绑定约束。**严禁引用 `参考资料/对白语言风格资料/` 任何文件（已弃用）**。

### R4 · 人设质感与连接戏（正向强制）
state 信息点是骨架不是全部。每个单元必须有服务 ①人物声音 ②人物关系 ③世界/时代质感 的台词；碎嘴型人设的废话本身就是人设，要写足。判据：一句非 state 信息点的台词，服务人物/关系/世界任一→要；与剧情人物世界全无关的纯空镜渲染→禁。自检：删掉所有 state 信息点后，剩下台词仍能让人认出"这是谁/什么关系/什么世界"——认不出=血肉不足，打回。

### R5 ·（已删除，由 CLAUDE.md 原则一全局管辖，见上"破梗红线"）
Zack 声音说明：Zack 可激烈地一来一回、追问、回呛、带情绪、用私人重量（"我妈"）顶上去——他逼的是线索，不是替玩家说出答案。不设"Zack 钝/句数上限/零感叹号"。

### R6 · 信息严密 · 游戏设计原则二
证据物理属性/数值/时间前后一致、跨 Loop 引用不矛盾；证词摘要与原文信息密度对等（不丢关键限定词，不膨胀出未述结论）；每个 NPC 在其知识边界内自洽——即使说谎，谎言本身也有内在逻辑。

### R7 · 旁白克制不空 + 无大段独白
环境/动作旁白少写；世界质感由人物台词承载，不靠旁白渲染。任何角色不写大段内心独白/演讲。纯画面/转场/NPC 静默反应用 `<!-- -->` 注释或并进相邻台词的动作括号 `[...]`，**不写成空 `**角色名**` 发言块**（承载 @get/@branch/@lie/@goto 机制处除外，用 @指令显式标记）。

### R8 · 格式与命名
- 节点边界=一个 `**角色名**` 发言块（其下连续 `>` 行属同一节点）；不写 9 位对话 ID。
- 分支/提取/指证一律 @指令：`@branch 名 / @opt "文本" -> 标签 #维度 / @path 标签 / @goto 标签 / @label 标签`；`@get 证词|证据 {配置ID} "{摘要}" #{keyInfoType}`；`@round N / @lie anchor={testimony_id|null} / @evidence "名" -> 标签 #correct|#trap "理由"`。
- 证词 ID 7 位 `{loop}{npc_code}{seq}`；Unit10 证据 ID 5 位（10xxx）。
- 角色发言名全程统一中文译名：扎克·布伦南 / 米奇 / 艾玛 / 露拉 / 奥哈拉太太 / 丹尼 / 莫里森 等，不中英混用、不中途改名。
- 文件名取自 state 字段（见三）。

### R9 · 承上启下：开场锚定 + 结尾留勾（都不机械）
- 开场锚定（≠重新进场）：每段非硬切对话，开头一两句自然台词/动作让玩家落地"我此刻在哪、上一步刚发生什么、现在紧张点是什么"。靠人物之口/动作带出，禁旁白 info-dump、禁重复进场。
- 结尾留勾（≠任务提示）：结尾由人物自然带出"下一步往哪使劲/还有什么没解决"。禁 Zack 总结式"我们现在该去X做Y"、禁任务播报、禁破梗。
- 硬切 cutscene 不强制开场锚定，但结尾仍要有自然前推/悬念。
- 段与段衔接：上一段的"勾"≈下一段开场要落地的"处境"。

### R10 · 地位跷跷板：场景必须转弯（不机械、不破梗）
每场对手戏（审讯/对质/谈判/Expose/有张力的搭档戏），"谁占上风"必须在场内**至少易手一次**，且每次易手是被一句台词或一个动作**挣来的**（一个转弯），不是设定好的、不是旁白宣布的。低位/惊惶/说谎一方必须有至少一拍用示弱/装傻/绕开/踩痛处真的把强势方挡回去一次（强势方可见地"得换招"）。Expose 可引入在场第三方作支点。易手靠态势不靠说破谜底/不靠"我赢了"自陈/不靠旁白标注。判据：画"上风曲线"——有无被挣得的翻折？低位有无赢过一拍？一条直线=FAIL。豁免：硬切 cutscene、纯过场无对手关系。
