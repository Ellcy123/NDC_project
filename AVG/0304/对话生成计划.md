# EPI02 全6循环对话生成计划

## 概述

### 目标
从 `AVG/0304/state/loop{1-6}_state.yaml` 出发，为 Unit2 全6循环生成完整的对话草稿 MD 文件。

### 输入 → 输出

```
输入:
  AVG/0304/state/loop{N}_state.yaml          ← 对话蓝图（核心驱动）
  + 知识池 / 角色人设 / 主角人设 / 设计文档   ← 补充数据

输出:
  AVG/0304/对话草稿/Loop{N}/
    ├── emma_00{N}.md                         ← 开篇
    ├── {npc}_00{X}.md                        ← NPC Talk
    ├── {npc}_00{X}_repeat.md                 ← NPC Repeat
    ├── loop{N}_{target}_expose.md            ← Expose
    └── (突发事件写在相关文件内，不单独成文件)
```

### 生成规则引用
- 对话格式与审查规则：`.claude/commands/generate-dialogue.md`
- State 结构规则：`.claude/commands/generate-state.md`
- 对话配置17条规则：`AVG/对话配置工作及草稿/AVG对话配置规则.md`

### 绝对禁止
- 参考任何已有对话草稿（`AVG/对话配置工作及草稿/Loop{N}_对话草稿.md`）
- 凭记忆编写剧情细节——所有事实必须从输入文件中提取

## 双 Agent 工作流

每个对话文件的生成采用 **Writer + Reviewer 双 Agent 并行** 模式：

### Writer Agent（生成者）
- 读取 state + 知识池 + 角色人设 + 设计文档
- 按 generate-dialogue skill 规则逐段生成对话
- 生成前先建立该 NPC 的**禁区清单**（blind_spots + withheld_topics 合并）

### Reviewer Agent（审查者）
- 与 Writer 同步工作，逐句/逐段审查
- 对每句台词执行以下检查并打分（10分满分）：

| 检查项 | 权重 | 说明 |
|--------|------|------|
| 禁区合规 | 致命 | NPC 是否说了/暗示了 blind_spots 或 withheld_topics 中的信息？违反 = 直接 0 分 |
| 信息可达性 | 致命 | Zack 的分支选项是否引用了他不可能知道的信息？违反 = 直接 0 分 |
| 角色语气 | 2分 | 是否符合 characters/*.md 中的说话方式？有无"串味"？ |
| 信息密度 | 2分 | 是否啰嗦/重复？是否在 state 要求的句数范围内？ |
| 叙事功能 | 2分 | 这句话是否推进了对话目标（player_inquiry）？还是废话？ |
| 上下文连贯 | 2分 | 与前后句的逻辑衔接是否自然？ |
| 情绪弧线 | 2分 | NPC 的情绪变化是否符合 mindset 描述的弧线？ |

### 评分规则
- **8-10 分**：通过，继续下一句
- **6-7 分**：标记警告，Writer 决定是否微调
- **< 6 分**：**必须重写**。Reviewer 给出具体原因 + 上下文分析，Writer 重新生成
- **0 分（致命）**：整段回退，重新审视该 NPC 的禁区清单和信息可达范围

### 工作流程图

```
Writer 生成第 1-5 句
  → Reviewer 逐句打分 + 标注
    → 低于 8 分的句子返回 Writer 重写
      → 重写后 Reviewer 二次打分
        → 通过 → Writer 继续生成第 6-10 句
        → 仍不通过 → Reviewer 给出替代方案，Writer 采纳或讨论
```

### 最终输出格式
每个对话文件末尾附带 Reviewer 的质量报告：

```markdown
<!-- 质量报告
总句数: XX
平均分: X.X / 10
重写次数: X
致命问题: X（已修复）
最低分句: #{ID} — {原因}
-->
```

## 生成顺序

**L1 → L2 → L3 → L4 → L5 → L6**（严格顺序，不可并行跨循环）

原因：
- 后续循环开篇需承接前序对话的语气基调和叙事节奏
- 鉴赏力 source 在前序循环埋入，quiz 在后续循环考验——需要确认 source 的具体措辞后才能设计 quiz 的迷惑选项
- Expose 结论影响下一循环 Opening 中 Emma 的 knows 清单

## 单文件生成流程（每个对话文件都执行）

```
Step 1: 准备
  ├── Writer 读取: state 对应段落 + 知识池 + 角色人设
  ├── Writer 建立: 该 NPC 的禁区清单（blind_spots + withheld_topics）
  └── Reviewer 读取: 同样的输入 + 前序已生成的对话（确保连贯）

Step 2: 生成 + 实时审查
  ├── Writer 按 5 句一批生成
  ├── Reviewer 逐句打分（10分制）
  │     ├── 8-10分 → 通过
  │     ├── 6-7分 → 标记警告，Writer 决定微调
  │     └── <6分 → 必须重写（Reviewer 给原因 + 上下文分析）
  │     └── 0分（致命：禁区/信息可达） → 整段回退
  └── 循环直到全文完成

Step 3: 配套生成 Repeat 文件
  └── 基于首次对话的分支结构生成（简短寒暄 + 分支回指）

Step 4: 文件级自检
  ├── 证据获取覆盖：testimony_ids 每个 ID 都有 get 句
  ├── 获取节奏：两次 get 之间 ≥ 3-5 句
  ├── 鉴赏力节点：source 有注释标注 / quiz 有3选项+正确+错误路径+汇合
  ├── ID 编码：9位数字无重复
  └── 对话链完整：无断链、分支汇合、末尾有 end
```
