# NDC 质量标准手册

> 本文档是所有内容设计的评审基准。评审角色按模块逐项检查，每个模块独立评分。
>
> **使用方式**：根据被审内容类型选择适用模块，不必每次全跑。
>
> | 被审内容 | 必查模块 | 建议加查 |
> |---------|---------|---------|
> | 指证(Expose)设计 | A + B + D | C + F |
> | 对话(Talk)草稿 | A + E | F + H |
> | 证据/谜题设计 | A + B + C | G |
> | 跨循环联动设计 | A + G | B |
> | 
> | 完整场景设计 | 全部模块 | — |

---

## 模块 A：三大设计原则（红线）

> 最高优先级。任何一条红线被触发 = 该内容不合格，必须返工。

### A1. 悬疑感优先——绝不提前破梗

- [ ] **玩家自主推理**：Zack 可以出示证据、指出矛盾，但不能替玩家说出结论
- [ ] **信息按循环节奏揭示**：后续 Loop 才揭晓的信息，在当前 Loop 中完全隐藏或伪装成无关细节
- [ ] **藏梗 > 明示**：能藏的线索就藏，让玩家回头看时恍然大悟
- [ ] **NPC 主动反应**：谎言、退守、狡辩是 NPC 主动说出的自然反应，不是 Zack 喂话后的被动否认

**红线**：
- ❌ Zack 在对话中直接解释"所以这说明……"
- ❌ 当前 Loop 泄露了后续 Loop 才应揭示的信息
- ❌ NPC 的谎言是被 Zack 引导后才说出的（"你是不是在说谎？""不，我没有！"）

### A2. 信息严密性——零矛盾、零漏洞

- [ ] **物理属性一致**：证据的尺寸、时间、距离、品牌、价格等跨循环引用无矛盾
- [ ] **信息密度对等**：证词原文与提取摘要不丢失关键限定词（时间、地点、人物、条件）
- [ ] **NPC 自洽**：每个 NPC 的陈述在其知识范围内自洽——即使说谎，谎言本身也有内在逻辑
- [ ] **时序严格**：证据获取不早于其存在时间；角色不引用尚未发生的事件

**红线**：
- ❌ 同一证据在不同文件中描述矛盾（如时间、数量、品牌不一致）
- ❌ 提取摘要比原文多出了原文未说的推断
- ❌ NPC 在 Loop2 提到了 Loop4 才发生的事件

### A3. 逻辑闭环——严密指向 + 多方暗示

- [ ] **交叉验证优先**：关键结论优先使用两种不同来源/类型的信息交叉验证（优秀标准）；单一证据指证也允许，但需确保该证据指向性足够强
- [ ] **多方暗示**：同一真相通过不同渠道给出暗示，单独看都不足以确定结论，组合后指向唯一答案
- [ ] **谎言精确匹配**：每轮指证的谎言只有对应维度的证据能反驳，不跨维度拆分
- [ ] **陷阱有逻辑**：错误选项有明确的"看起来相关但实际不构成反驳"的逻辑原因

**红线**：
- ❌ 单一弱证据直接定案——如果只用一个证据，必须指向性极强且无歧义
- ❌ 错误选项完全无关，玩家一眼排除
- ❌ 多个证据都能反驳同一条谎言（缺乏精确匹配）

## 模块 B：逻辑严密性检查

> 适用于：谜题设计、证据设计、指证设计

### 检查项（加权评分，及格线 ≥ 3.5 / 5 分）

| # | 检查项 | 权重 | 评分要点 |
|---|--------|------|---------|
| B1 | 人物智商/行为合理性 | 30% | 角色行为符合正常智商，"聪明但有疏漏"而非"蠢到留破绽" |
| B2 | 证据获取合理性 | 25% | Zack 的权限和时序合理，不超越当前 Loop 的调查权限 |
| B3 | 辩解空间堵死程度 | 20% | 嫌疑人难以轻易反驳证据组合，但单一证据仍有辩解余地 |
| B4 | 时序符合性 | 15% | 符合当前循环的时间线和角色登场顺序 |
| B5 | 细节自洽性 | 10% | 各细节相互印证，无内部矛盾 |

### 常见问题检查

- [ ] **"擦一半忘一半"问题**：犯罪者毁灭证据是否有合理解释（时间紧迫、慌乱等）？
- [ ] **"主动留证据"问题**：证据是无意遗留还是刻意安排？应为前者
- [ ] **概率依赖**：推理链是否依赖极小概率事件？
- [ ] **权限越界**：Zack 在当前 Loop 是否有合理渠道获取该证据？

### 评分参考

| 分数 | 含义 |
|------|------|
| 5 | 无懈可击，角色聪明但恰好有这个盲点 |
| 4 | 逻辑合理，需少量脑补但可接受 |
| 3.5 | 及格线——能说通但略有牵强 |
| 3 | 有明显薄弱环节，需修改 |
| ≤2 | 逻辑硬伤，必须重做 |

## 模块 C：谜题趣味性评分

> 适用于：证据设计、指证谜题设计

### 检查项（加权评分，及格线 ≥ 3 / 5 分）

| # | 维度 | 权重 | 评分要点 |
|---|------|------|---------|
| C1 | 精巧性/惊喜感 | 40% | 是否有视觉陷阱、表面证据反转、多层推理？ |
| C2 | 逻辑链完整性 | 25% | 推理链至少 2-3 步，环环相扣 |
| C3 | 身份/职业契合度 | 15% | 证据是否与嫌疑人身份深度绑定？换个人就不成立？ |
| C4 | 游戏可视化适配性 | 15% | 关键证据是否能在游戏 UI 中直观呈现？ |
| C5 | 人物行为合理性 | 5% | 犯罪/失误行为是否自然合理？ |

### 高分特征（4-5 分）

- 证据本身有"表里两层"——第一眼看到的和深入分析后发现的不同
- 推理需要组合多条信息，任何单条都不足以得出结论
- 证据与角色身份/职业深度绑定（如连号钞票与银行职员）
- 游戏画面可以直观展示关键差异（如镜像反转、尺寸对比）

### 低分特征（1-2 分）

- 仅仅是"A 有而 B 没有"的简单排除——缺乏精巧性
- 推理只需一步，无思考空间
- 依赖生活常识而非游戏内线索——玩家可能不认同
- 需要 NPC "降智" 才能成立

## 模块 D：指证质量检查

> 适用于：Expose 设计、指证谜题配置

### D1. 结构完整性

- [ ] 至少 3 轮谎言递进（事实矛盾 → 物证逻辑 → 物理排除）
- [ ] 第 1 轮 Lie = Talk 中已收集的证词（来源可追溯）
- [ ] 第 2 轮起的 Lie = 嫌疑人被逼出的新谎言（非预先收集）
- [ ] 谎言由嫌疑人主动说出（止损），不是 Zack 喂话

### D2. 证据配置

| 轮次 | 证据数量 | 作用 | 矛盾类型 |
|------|---------|------|---------|
| 第 1 轮 | 1-2 个 | 戳破表层谎言 | 直接矛盾 |
| 第 2 轮 | 2-3 个 | 攻击逻辑漏洞 | 间接矛盾 |
| 第 3 轮 | 1 个 | 彻底击穿 | 致命矛盾 |

- [ ] 每轮证据精确匹配该轮谎言的维度，不多不少
- [ ] 同一维度的证据不跨轮拆分
- [ ] 每轮使用不同维度的证据打不同论点

### D3. 陷阱证据

- [ ] 每轮至少 1 个陷阱选项（看起来相关但不构成反驳）
- [ ] 陷阱有明确的"为什么选错"的逻辑原因
- [ ] 选错后有合理的失败反馈（不是简单的"再试一次"）

### D4. 情绪递进

- [ ] 嫌疑人情绪随轮次递进（自信 → 动摇 → 慌张 → 崩溃）
- [ ] 每次退守/狡辩在嫌疑人立场上是合理的止损策略
- [ ] 最终击穿后的反应符合角色人设

### 红线

- ❌ **一击必杀**：一个证据直接让嫌疑人认罪
- ❌ **逻辑跳跃**：某轮结论不基于上一轮信息递进
- ❌ **无效防御**：嫌疑人的新谎言完全无意义或毫无说服力
- ❌ **证据堆砌**：单轮要求选择 4 个以上证据

## 模块 E：对话规范检查

> 适用于：Talk 草稿、Repeat 对话、分支对话

### E1. 分支设计

- [ ] 每组分支选项指向不同信息维度（身份/时间线/事件/情绪/背景）
- [ ] 不同分支允许各自独占 `get` 获取
- [ ] 态度分支：核心信息相同，措辞/NPC反应不同
- [ ] 所有分支最终汇合到同一节点，不永久分叉
- [ ] Repeat 对话补完首次对话中未走过的分支

### E2. 信息获取

- [ ] 每个场景/NPC 承载 1-3 个核心信息点（不超过 3 个）
- [ ] 两次 `script:"get"` 之间至少隔 3-5 句普通对话
- [ ] `keyInfoType` 标记准确（timeline / statement / identity）
- [ ] 多证据获取通过分支机制分散，不集中轰炸

### E3. 证词质量

- [ ] `<special>` 标记内容与 TestimonyItem 提取道具高度一致
- [ ] 提取摘要不丢失关键限定词，不添加原文未说的推断
- [ ] 信息的"可提取感"和"实际可提取性"一致——看起来像线索的就必须能提取

### E4. 格式与路径

- [ ] `videoEpisode` / `videoLoop` / `videoScene` / `videoId` 与所属对话文件一致
- [ ] 对话 ID 编码格式正确（9 位：`{loop}{scene}{sequence}`）
- [ ] 证词 ID 编码格式正确（7 位：`{loop}{npc}{sequence}`）

### 红线

- ❌ 分支永久分叉，玩家永久错过内容且无法通过 Repeat 补回
- ❌ 连续 get 无间隔——玩家来不及消化
- ❌ 有 `<special>` 标记但实际不可提取

## 模块 F：玩家体验评分

> 适用于：完整场景/流程审查、指证体验审查

### 三维度评分（每维度及格线 ≥ 3.5 / 5 分）

| # | 维度 | 定义 | 3.5 分（及格线） | 5 分（优秀） |
|---|------|------|----------------|------------|
| F1 | 趣味性 | 推理过程是否有乐趣 | 不无聊，能玩下去 | 想截图分享给朋友 |
| F2 | 合理性 | 逻辑是否让人信服 | 需额外解释但勉强接受 | 完全不出戏 |
| F3 | 情感共鸣 | 是否引发情感波动 | 有轻微触动 | 放下手机沉思 |

### 玩家体验流程检查

1. **初见阶段**：玩家第一次接触谜题/场景时，是否有好奇心？信息量是否适中？
2. **搜证阶段**：收集证据的过程是否流畅？是否有"啊哈时刻"（突然发现线索）？
3. **推理阶段**：逻辑链是否清晰可循？玩家是否能"自己"得出结论而非被强制引导？
4. **指证阶段**：嫌疑人反应是否合理？击穿谎言是否有情感冲击？
5. **结束阶段**：谜题解开后是否有满足感？是否想继续下一个循环？

### 常见体验问题

- [ ] **信息过载**：单场景/单对话承载过多新信息，玩家消化不了
- [ ] **引导缺失**：玩家完全不知道下一步该做什么
- [ ] **引导过度**：Zack 把所有推理说完了，玩家只是点"确认"
- [ ] **情感断裂**：紧张场景后突然插入无关对话，破坏节奏
- [ ] **挫败感**：陷阱证据过于接近正确答案，玩家反复试错但学不到为什么错

## 模块 G：信息时序与跨循环一致性

> 适用于：跨 Loop 设计、证据链设计、State 文件审查

### G1. 循环信息节奏

| Loop | 揭示层 | 核心发现 | 检查要点 |
|------|--------|---------|---------|
| 1 | 犯罪事实确认 | 下药、拖拽、枪杀 | 只确认"发生了什么"，不暴露"谁做的" |
| 2 | 执法者腐败 | Morrison 矛盾、赌债 | 打破玩家对警方的信任，不暴露幕后 |
| 3 | 经济犯罪网络 | 勒索帝国、假古董 | 揭示金钱动机，不暴露真凶 |
| 4 | 嫌疑人浮现 | 偷盗、收钱、无不在场证明 | 浮现嫌疑但不定案 |
| 5 | 情感动机补全 | 背叛、复仇日记 | 补完动机，不做最终裁定 |
| 6 | 真相拼图 | 弹道+音量+门锁+排班 | 所有前序证据组合，指向唯一答案 |

- [ ] 当前 Loop 严格只揭示对应层的信息
- [ ] 后续 Loop 的关键信息在当前 Loop 不以任何形式泄露
- [ ] Loop 6 从几乎所有前序 Loop 调取证据形成完整拼图

### G2. 证据获取权限

| 证据类型 | 获取难度 | 所需条件 |
|---------|---------|---------|
| 公开证据 | 低 | 侦探身份即可（现场物证、公开证词） |
| 半公开证据 | 中 | 调查权限或已建立的人际关系 |
| 机密证据 | 高 | 搜查令、内线、特殊渠道 |
| 非法证据 | 极高 | 灰色手段或内线协助 |

- [ ] 每个证据的获取渠道与 Zack 在当前 Loop 的权限匹配
- [ ] 证据获取时间不早于其物理存在时间
- [ ] 角色关系满足获取条件（如需要内线的证据，内线角色已登场）

### G3. 疑点解锁交叉验证

- [ ] 疑点(Doubt)优先要求两种不同来源/类型的信息组合解锁（优秀标准）
- [ ] 交叉模式明确（证词×物证 / 物证×物证 / 多方证词 / 证词×财务）
- [ ] 若单一证据解锁疑点，需确保该证据指向性足够强且无歧义

### G4. 跨 Loop 证据复用

- [ ] 后续 Loop 复用前序 Loop 证据时，证据描述完全一致
- [ ] 复用证据的 ID 引用正确
- [ ] 证据在新 Loop 语境下的使用方式合理（不是生搬硬套）

## 模块 H：节点检查

> 适用于：包含

### H1. 触发模式（必须属于以下六种之一）

| 模式 | 描述 | 核心要求 |
|------|------|---------|
| 信任门槛 | NPC 掌握敏感信息，需确认 Zack 了解情况 | NPC 有分享的动机 |
| 情感共鸣 | NPC 试探 Zack 是否真正理解 | 涉及 NPC 的情感核心 |
| 专业对等 | 专业人士调整沟通深度 | Zack 需展示专业知识 |
| 对话回声 | Zack 主动引用前序信息 | 引用的信息玩家之前确实获取过 |
| 防御反弹 | NPC 反过来试探 Zack | NPC 有防御动机 |
| 环境共振 | 环境细节自然引出话题 | 场景中有可见的触发物 |

- [ ] 节点属于上述六种模式之一
- [ ] 主语是 Zack 主动展示，不是 NPC 提问考试
- [ ] NPC 有角色动机参与（不是"系统需要"）
- [ ] 节点服务于对话推进，不是打断

### H2. 自然度检查

- [ ] 删掉该节点后对话仍然通顺（说明节点是"加分项"而非"必经路"）
- [ ] 成功/失败两条分支的 NPC 反应都符合人设
- [ ] 同一 NPC 的）

### 红线

- ❌ **老师提问模式**："我考考你，XX 是什么？"
- ❌ **突兀插入**：对话正常进行时硬切到
- ❌ **无动机测试**：NPC 没有理由关心 Zack 是否知道
- ❌ **单句弹窗**：前后无上下文，突然弹出选择
- ❌ **重复测试**：同一信息在多处被测试
