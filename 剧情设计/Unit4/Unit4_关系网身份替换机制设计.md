## 一、核心机制概述

### 一句话定位

**身份推理的最终交互动作发生在玩家身上**——Zack 不替玩家说出 "Whale = Mickey"，玩家自己在关系网中输入这个名字。

### 设计哲学

身份推理的最高形态是 **玩家亲手指认凶手**。NDC 的关系网 UI 让 Whale 一直作为"匿名条目"存在 —— 玩家通过键盘输入完成最后一步。

这种"亲手"的体验比剧情告白强 100 倍——它把"侦探"从一个角色变成玩家自己。

### 与 Unit 4 主题的契合

Unit 4 主题：**被操控的善意是否还是善意**。

玩家在 L1-L4 经历记忆场景，投资了 Mickey 这个温暖兄弟。L5 玩家自己在键盘上敲下"Mickey"——**亲手把那个温暖的兄弟移到凶手位置**。

这是 Unit 4 主题的最终具象——**玩家自己也是那个操控的一部分**。

## 二、UI 状态流转

### 状态 1：Unit 1-3（关系网中 Whale 一直存在但锁定）

- 关系网中有一个独立条目：「**❓ Whale**」
- 占位图：剪影 + 问号
- 状态：**不可点击**（灰色 / 半透明）
- 玩家鼠标悬停提示："**身份不明**"
- Mickey 在伙伴位置（和 Emma、Foster 等并列）：**正常照片 + 名字 "Mickey Donnelly"**

### 状态 2：Unit 4 L1-L3（Whale 仍然锁定，但开始有信号）

- L1-L3 期间 Whale 条目依然不可点击
- 但 Whale 卡片下方累积"侧写信息"——每收集到一片拼图，自动添加到卡片描述中
- 例如 L1 后："抽古巴雪茄"
- 例如 L3 后："能调度警局武器库 / 11 年首次亲自动手"
- 玩家可以查看侧写，但不能点击

### 状态 3：L4 阶段一锁定后（Whale 条目解锁）

⭐ **触发条件**：L4 侦探板对接段落完成（Zack 嘴上说出"就是那个 Whale"）。

- Whale 卡片从灰色变为彩色
- 卡片下方出现新的提示："**点击输入身份**"
- 玩家可以点击 Whale 条目

### 状态 4：玩家点击 Whale → 输入框出现

- 卡片中央 "Whale" 字样下方出现可输入文本框
- 占位文字："输入你认为的真实身份"
- 玩家在键盘上输入字符——**字符实时替换 "Whale" 字符**
- 如果输入 "Morrison"，卡片显示 "Morrison"（但还是匿名剪影 + 问号）
- 如果输入任何错误名字，卡片字样变化但**没有照片出现**——保持匿名状态

### 状态 5：玩家输入 "Mickey" → 匹配触发

⭐ **核心瞬间**——玩家完成阶段二锁定。

视觉序列（约 2-3 秒）：

1. 玩家敲完最后一个字母 "y"
2. **暂停一秒**——文本框边缘开始发光
3. 关系网伙伴位置的 Mickey 卡片**慢慢褪色**（饱和度降低）
4. Mickey 照片**从伙伴位置飞向** Whale 位置（约 1 秒）
5. 飞行过程中，照片轨迹是一道**冷色调的线**——不是温暖的过场
6. Mickey 照片落到 Whale 位置——伙伴位置变成空白卡片
7. Whale 条目下方的剪影 + 问号被照片完全替换
8. 卡片标题正式从 "Whale" 变为 "**Mickey Donnelly / Whale**"
9. 卡片底部出现 "**已锁定**" 标记——不可再点击

### 状态 6：锁定后

- Mickey 永久在凶手位置
- 伙伴位置永久空缺（视觉上提醒玩家"你失去了一个伙伴"）
- 玩家无法撤销

## 三、输入容错规则

### 接受的输入（都算"对"）

宽松容错——只要满足以下任一条件就算输入正确：

- 包含 "Mickey"（不分大小写）
- 包含 "Donnelly"（不分大小写）
- "Mickey Donnelly" / "mickey donnelly" / "MICKEY" / "Donnelly, Michael" 都接受
- 全名 "Michael Donnelly" / "Michael Francis Donnelly" 也接受

### 不接受的输入（不触发匹配）

- 任何不含 "Mickey" 或 "Donnelly" 的字符串
- "M.D."（首字母）—— **不接受**（要求玩家明确说出名字，而不是 Emma 笔记本里画的圆圈）
- 部分字母 "Mick" / "Don" —— 不接受（必须完整）

### 边界情况

- 玩家输入 "Mickey Mouse" —— 接受（包含 "Mickey"，触发匹配）
- 玩家输入 "Mickey Brennan" —— 接受
- 这种宽松容错是为了**降低玩家挫败感**——避免"我答对了但拼写错了一个字母"的体验

---

## 四、错误输入的反馈

### 设计原则：**纯粹留白**

错误输入时**完全没有反馈**——只是字符替换 + 没有照片出现。

### 为什么是这种设计

- 玩家自己感受到"还差什么"——这是身份推理"自己想出来"的核心精神
- 系统提示 "错误" 会破坏玩家的探索感
- 留白让玩家有 **"啊，我猜错了，但游戏没告诉我答案"** 的体验

### 玩家典型行为路径

1. L4 阶段一锁定后玩家可能尝试输入
2. 第一次可能输入 "Morrison"（最像凶手的人）→ 字符替换 + 没照片
3. 然后可能输入 "Watts" / "Whitfield" / "Bernard"（前作其他律师/法律相关）→ 仍然没照片
4. 玩家心里浮现："那这些都不是？谁是？"
5. L5 篝火记忆场景 + Margaret 揭 Patrick 真相 + 思维 DNA → 玩家心里开始想 Mickey
6. 玩家鼓起勇气输入 "Mickey" → ⭐ 触发匹配

### 一种例外（可选）

如果玩家输入次数 >= 5 次都没有匹配，可以触发 Emma 一句话：

> Emma（路过）：「Brennan 先生，您是不是在猜？」
> Zack：「……」
> Emma：「猜不出来时，回到证据。」

但这是**可选**——不要主动救援玩家。

## 五、与两阶段锁定的整合

| 阶段 | Whale 条目状态 | 关系网交互 |
|---|---|---|
| Unit 1-3 + Unit 4 L1-L3 | 锁定（不可点击） | 玩家观察侧写信息累积 |
| L4 阶段一锁定后 | 解锁（可点击） | 玩家可以输入名字尝试 |
| L5 阶段二锁定瞬间 | 输入 "Mickey" 触发匹配 | ⭐ Mickey 照片飞过去 |
| L6 法人文件 | 已锁定 | 法律层面物证确认 |

### 阶段一 vs 阶段二 的"锁定"对应

| 阶段一锁定 | 关系网状态 |
|---|---|
| Zack 嘴上说"就是那个 Whale"（L4 侦探板）| Whale 条目从锁定 → 可点击 |

| 阶段二锁定 | 关系网状态 |
|---|---|
| 玩家自己输入"Mickey"（L5）| Whale → Mickey 身份替换 |
| Zack 内心说"Whale = Mickey"（L5 末尾）| 与玩家输入同时发生 |

**关键**：玩家的输入动作和 Zack 的内心独白**同步发生** —— 玩家就是 Zack 的内心。这是身份推理类型最高级的"玩家即角色"体验。

## 六、与记忆场景机制的呼应

记忆场景让玩家**投资 Mickey**：
- L1 糖罐场景：13 岁 Mickey 第一次接糖
- L2 葬礼场景：24 岁 Mickey 哭着说"我会做对的事"
- L4 四十二层初访场景：30 岁 Mickey 说"从这里看每个人都很小"
- L5 篝火场景：20 岁 Mickey 在 Patrick 篝火旁记笔记
- L6 童年信物：Mickey 真心做对事的瞬间

玩家投资了 30 年的 Mickey 形象 → L5 在键盘上敲 "Mickey" → **亲手把这个 30 年的伙伴移到凶手位置**。

**记忆场景投资得越深，输入"Mickey"的瞬间越心碎**。

---

## 七、L6 法人文件的角色

L5 玩家输入 Mickey + 照片切换 = **玩家心里的锁定**
L6 法人文件 = **法律层面的物证锁定**

两者递进 —— L5 是情感和直觉的承诺，L6 是 Zack 在四十二层对质前的最后一片证据。

L6 法人文件揭晓时，玩家会想起"我 L5 就敲过这个名字"——这是**玩家给自己的回响**。

## 八、程序需求清单

### 1. 关系网新条目类型

`NPC` 表加字段：
- `nameLockType`: `LOCKED` / `UNLOCKABLE` / `UNLOCKED` / `IDENTIFIED`
- `displayPlaceholder`: 占位图（剪影 + 问号）
- `realPortrait`: 真实照片（解锁后显示）
- `playerInputName`: 玩家输入的名字

### 2. 解锁时机触发

- L4 阶段一锁定段落完成（侦探板对接结束）→ 触发 Whale 条目解锁
- 这个触发需要主线进度系统支持

### 3. 输入框 UI

- 玩家点击 Whale 条目 → 弹出 / 内嵌输入框
- 实时字符替换（替换的是显示的"Whale"字样，不是数据库里的字段）
- 容错匹配逻辑（包含 "Mickey" 或 "Donnelly" 即匹配）

### 4. 匹配动画序列

- 文本框边缘发光（约 1 秒）
- Mickey 照片从伙伴位置飞到 Whale 位置（约 1-2 秒）
- 伙伴位置变空（同步）
- Whale 条目变 Mickey + 锁定状态

### 5. 锁定状态

- 输入正确后，`nameLockType` 变为 `IDENTIFIED`
- 不可再点击编辑
- 不可重置

### 6. 反馈差异化

- 错误输入：只字符替换，无任何反馈
- 正确输入：触发完整动画序列

## 九、美术需求清单

### 1. Whale 占位图

- 男性剪影 + 大问号
- 偏冷色调（蓝灰 / 黑色）
- 暗示"危险但身份未明"

### 2. Mickey 照片切换动画

- Mickey 现有照片从伙伴位置 → Whale 位置
- 飞行过程加冷色调滤镜（区别于伙伴位置的暖色）
- 落到 Whale 位置时**完全冷色**

### 3. 伙伴位置空缺

- Mickey 离开后伙伴位置变成什么样？
- 选项 A：空白卡片 + "已离队" 文字（推荐）
- 选项 B：完全消失（伙伴位置数量 -1）

推荐 **A** —— 空白卡片有视觉重量，提醒玩家"你失去了一个伙伴"。

### 4. 输入框 UI

- 简洁的文本框设计
- 占位文字："输入你认为的真实身份"
- 输入字符与"Whale"字样平滑替换的视觉过渡

### 5. 锁定标记

- 玩家锁定后，卡片角落出现小图标（锁 / 勾 / 印章）
- 表示"已确认，不可更改"

## 十、设计师注意事项

### 1. Whale 一直在关系网中存在
⚠️ 从 Unit 1 起 Whale 就在关系网中——锁定状态。玩家**一直能看到他但不能动他**。这种"看得见摸不着"的张力贯穿前 3 章。

### 2. Mickey 在 Unit 1-3 + Unit 4 L1-L4 一直在伙伴位置
⚠️ 玩家看到 Mickey 的位置是"伙伴"——和 Emma、Foster 并列。**他从来不在嫌疑人池里**。

### 3. 阶段一锁定才解锁输入
⚠️ L4 之前 Whale 条目不可点击。这避免玩家在还没看完主线时就猜出来。

### 4. 输入"Mickey"是唯一答案
⚠️ 不接受任何其他名字 —— 即使玩家逻辑上可能猜到（比如玩家发现签字小钩）。**必须输入完整名字**。

### 5. 错误输入完全无反馈
⚠️ 不要主动救援玩家。错误输入只字符替换，没有"试试别的名字"提示。让玩家自己感受。

### 6. Mickey 照片切换的瞬间是 Unit 4 高潮
⚠️ 这是**整章的视觉高潮**——所有美术 / 程序资源应该投入这一刻。一个"普通"的动画做不出这种效果。需要**冷暖色对比 + 空间位移 + 时间停顿** 三件事都做对。

### 7. 锁定不可逆
⚠️ 玩家输入正确后不能撤回——这是"承诺"。如果玩家想"重玩"必须重开存档。
