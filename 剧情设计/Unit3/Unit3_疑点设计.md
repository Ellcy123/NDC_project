# Unit3 疑点系统设计

> 基于第三章（Unit3 / EPI03 杀夫反击）6个循环的指证设计和证据设计，为每个循环定义疑点(DoubtConfig)条目。
>
> **设计原则**：
> - 疑点ID编码：`{unit}{loop}{seq}`，如 3101 = Unit3 Loop1 第1个疑点
> - 每循环3-5个疑点（参照第一章经验总结）
> - 所有指证使用的证据必须出现在至少一个疑点的condition中
> - condition类型：type=1(道具) / type=2(关系网) / type=3(时间线)
> - 证据标注：★ = 指证使用的证据 / ☆ = 非指证证据（伏笔/氛围）
>
> 日期：2026-02-18

---
