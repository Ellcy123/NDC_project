# 玩法设计文档：油样本化学比对 (Oil Sample Chemical Comparison)

**文档版本**: v1.0
**隶属章节**: Unit 3 - 循环3
**设计目标**: 通过色谱峰位对齐（Chromatography Peak Alignment）交互，让玩家亲手验证楼顶地面油迹与Helen家轮椅维护油的化学成分完全一致，将抽象的"成分相同"转化为直观的波形重合体验。

## 0. 前因后果 (Context & Impact)

### 前因——证据从哪来

| 证据 | ID | 获取场景 | 获取方式 |
|------|-----|---------|---------|
| 地面油迹采样 | 3311 (33-E-01) | L3 SC3002 楼顶二次搜证 | 楼顶护栏附近地面采集的油迹样本（石油基润滑剂），中间产物 |

**触发动机**: 楼顶出现油迹本身就极度可疑——油的分布范围仅1.5米，精确对应Thomas坠落点。Helen家恰好存有同类型润滑油。两样东西摆在一起，任何侦探都会想：是同一种油吗？

### 后果——证据用在哪

- **直接用途**: **L3 Expose R2** — Helen谎称"天台我自己从来没上去过，楼顶发生的事跟我没关系"
  - 出示 3312（油样本化学比对报告）→ 地面油 = Helen家轮椅油
  - 品牌 Standard Oil / 化学指纹 / 添加剂比例14.2% **完全一致**
  - 油为人为涂抹（非自然滴落），范围仅1.5米精确对应坠落点，氧化程度 = 案发前1-2天涂抹
  - **Helen去过天台并提前涂了油 → 预谋性行为**
  - Helen崩溃承认涂油，辩称Thomas指导她涂哪一段

- **跨循环复用 (L4)**: 3312 作为 L4 派生输入 — 3312 + 3401(门把手外侧油迹) → 3405(油迹二方比对报告)
  - 用于 L4 Expose R1 击穿 Helen "那晚在家" 最后防线
  - 门把手外侧有油 + 内侧干净 = 单向进入（楼梯间→楼顶），且油与 Helen 家轮椅油同源
  - 详见 `L4_油迹二方比对.md`

- **剧情意义**: 首次将Helen与犯罪现场建立**物理性连接**，打破Helen "只是帮忙配钥匙" 的无害形象。更重要的是，油被人为涂抹在坠落点——这不是巧合，是预谋。

## 1. 玩法概述 (Overview)

*   **玩法类型**: chromatography_alignment（色谱峰位对齐）
*   **核心体验**: **观察两条化学成分曲线** → **拖拽对齐色谱峰** → **逐峰确认匹配** → **曲线重合** → **生成化学分析报告**
*   **预期时长**: 45-75秒
*   **对应剧情**: Zack将地面油迹采样和Helen家轮椅维护油送到Foster实验室，Foster用简易色谱法（Simple Chromatography）分离两份样本的化学成分，生成成分图谱。Zack在Foster的指导下亲自比对两条色谱曲线。

### 1.1 历史背景 (Historical Context)

```
┌─────────────────────────────────────────────────────────────┐
│              【1920年代色谱分析法】                           │
│                                                             │
│  历史依据：                                                  │
│  • 1903年 Mikhail Tsvet 发明色谱法（Chromatography）        │
│  • 1920年代已用于分离植物色素、石油产品成分                  │
│  • 简易柱色谱（Column Chromatography）可在实验室手工操作     │
│  • 通过不同溶剂洗脱，各成分在滤纸上按分子量分离             │
│                                                             │
│  游戏中的简化：                                              │
│  • Foster将两份油样本分别通过色谱柱分离                     │
│  • 各成分在记录纸上形成"峰"（浓度高的成分峰高）             │
│  • 两份样本各生成一条曲线，如果成分相同则峰的位置/高度一致  │
│  • 玩家通过对齐峰位来直观判断两份样本是否同源               │
│                                                             │
│  设计意义：                                                  │
│  • 波形对齐比静态左右对比更有操作感和发现感                 │
│  • 每个峰对应一种化学特征，逐峰对齐 = 逐项确认             │
│  • "曲线完全重合"的视觉效果比文字结论更具说服力             │
└─────────────────────────────────────────────────────────────┘
```

1. **3311** (33-E-01)「地面油迹采样」— 楼顶地面油迹（石油基润滑剂）
2. **3301** (33-I-01)「轮椅维护油」— Helen家 Standard Oil Lubricant

**视觉说明**:
*   上方曲线（Reference）为**蓝色**，固定不动，代表地面油迹采样的色谱图谱
*   下方曲线（Test）为**橙色**，可左右拖拽，代表轮椅维护油的色谱图谱
*   两条曲线的形状几乎完全相同（因为成分一致），但初始状态下水平偏移，峰位不对齐
*   底部的匹配进度指示器显示4个峰的对齐状态：○ = 未对齐，● (绿色) = 已对齐
*   中间分隔线为虚线，当所有峰对齐后分隔线消失，两条曲线合并到同一坐标系

### 2.2 四个色谱峰的含义 (Peak Definitions)

| 峰位 | 化学特征 | Foster解说 | 对齐判定 |
|:-----|:---------|:-----------|:---------|
| **P1** 基础化合物 (Base Compound) | 石油基润滑剂主成分，峰最高 | "第一个峰——石油基润滑剂的主成分。两份样本的主峰高度几乎一模一样。" | 水平偏差 < 8px |
| **P2** 粘度标记 (Viscosity Marker) | 粘度调节剂，中等高度 | "第二个峰是粘度调节剂。Standard Oil 的中粘度配方...很典型。" | 水平偏差 < 8px |
| **P3** 添加剂特征 (Additive Signature) | 添加剂成分，较矮但独特 | "这个小峰有意思——添加剂的特征峰。比例是14.2%，和Standard Oil的配方表完全吻合。" | 水平偏差 < 8px |
| **P4** 化学指纹 (Chemical Fingerprint) | 微量杂质的特征谱线，最矮 | "最后这个...化学指纹。每一批次油品都有微量杂质差异，这相当于油的'身份证'。两份样本连指纹都一样——不只是同一个品牌，是同一瓶油。" | 水平偏差 < 8px |

## 4. 特殊情况处理 (Edge Case Handling)

### 4.1 玩家长时间无操作 (Idle)

*   **场景**: 进入界面后 15 秒内未拖拽
*   **处理逻辑**:
    *   Foster: "把下面那条曲线往左拖拖看，找到峰位重合的位置。"
    *   下方曲线边缘出现轻微的左右摆动提示动画（0.5秒，振幅3px）

### 4.2 玩家拖拽方向错误 (Wrong Direction)

*   **场景**: 初始偏移为向右偏移，玩家一直往右拖（越来越远离对齐位置）
*   **处理逻辑**:
    *   设置拖拽范围限制：最大偏移不超过初始偏移量的150%
    *   到达边界时曲线弹性回弹，伴随轻微阻力感
    *   超出合理范围后 Foster 提示: "方向反了，往另一边试试。"

### 4.3 玩家拖拽过快 (Speed Overshoot)

*   **场景**: 玩家快速滑过对齐位置，没有触发任何峰位对齐
*   **处理逻辑**:
    *   如前述，对齐需要在阈值范围内停留 > 0.2s 或速度降至阈值以下
    *   如果玩家来回快速滑动3次以上仍未触发对齐:
        *   Foster: "慢一点...色谱比对需要精确。一点一点移。"
        *   同时降低速度阈值（放宽判定条件），使下次更容易触发

### 4.4 玩家中途退出 (Mid-game Exit)

*   **场景**: 玩家在对齐过程中点击退出或按 ESC
*   **处理逻辑**:
    *   弹窗: "色谱比对尚未完成，确定要中断吗？"（继续 / 中断）
    *   若中断: 进度不保存，下次需重新开始
    *   再次触发: 可通过 Foster 实验室中的 [色谱比对] 入口重新进入

### 4.5 证据缺失（防御性设计）

*   **场景**: 因Bug导致玩家只持有一份证据就进入了界面
*   **处理逻辑**:
    *   检测到必要证据缺失时，Foster: "样本不全，没法跑对比。你再查查有没有遗漏。"
    *   自动退出界面

### 4.6 重复进入

*   **场景**: 玩家已完成比对、获得 3312 后再次尝试进入
*   **处理逻辑**:
    *   入口消失（一次性玩法）
    *   玩家可在证据 3312 的详情界面中查看"分析记录"，回顾色谱重合的静态图片和结论文字

*   **语调**: 科学家的客观冷静，但P4的结论句（"同一瓶油"）语气要有明显的分量感
*   **节奏**: 每句解说在对齐音效后0.3秒开始播放，避免音效和语音重叠
*   **字幕**: 所有Foster语音在左下角对话气泡中同步显示文字

### 6.3 峰位数据（用于曲线绘制）

| 峰 ID | 位置 (x%) | 高度 (y%) | 宽度 (σ) | 对应成分 |
|:---|:---|:---|:---|:---|
| P1 | 20% | 90% | 5% | 石油基润滑剂主成分 |
| P2 | 40% | 65% | 4% | 粘度调节剂 |
| P3 | 55% | 40% | 3% | 添加剂 (14.2%) |
| P4 | 80% | 25% | 3% | 微量杂质指纹 |

> 曲线由4个高斯分布叠加生成：`y = Σ height_i * exp(-(x - pos_i)^2 / (2 * σ_i^2))`。Reference和Test使用完全相同的参数，仅水平偏移不同。

## 8. 开发备注 (Dev Notes)

### 8.1 美术资源需求

*   **场景背景**: Foster实验室台面（可与其他Foster实验室小玩法共用背景，如织物比对）
*   **色谱曲线**: 程序化生成（高斯叠加），无需静态图片素材；需手绘质感的线条Shader
*   **峰位标记**: 小三角图标 x2（灰色/绿色两态）
*   **进度指示灯**: 圆形指示灯 x2（灰色/绿色两态）
*   **"完全一致"浮现文字**: 打字机字体贴图
*   **曲线下方半透明填充**: 蓝色/橙色/绿色三色，Shader或遮罩实现
*   **光晕/脉冲特效**: 绿色径向渐变，可用通用粒子或UI动画

### 8.2 技术要点

*   **曲线绘制**: 使用高斯叠加公式程序化生成曲线点集，通过 LineRenderer 或 UI Path 渲染
*   **拖拽系统**: 仅需一维水平拖拽（比常规二维拖拽简单），需实现惯性和边界弹回
*   **对齐判定**: 逐峰检测水平偏差，注意四个峰共享同一偏移量——实质上只有一个自由度
*   **音频时序**: P1-P4的"咔嗒"声和Foster语音需要排队播放，避免重叠；如果多个峰在短时间内连续触发（因为间距一致），用队列系统按序播放
*   **状态机**: IDLE → DRAGGING → ALIGNING(逐峰) → MERGING → COMPLETE，五个状态
*   **性能**: 曲线采样点建议200-300个，帧率影响极低

### 8.3 与L4油迹二方比对的关系

*   L4的小玩法(`L4_油迹二方比对.md`)以本小玩法的输出 3312 作为输入之一
*   L4的交互应当比L3更快速简洁——玩家已做过一次完整的色谱比对，L4可复用相似的UI框架但简化流程
*   建议L4的交互模式为"快速验证"：直接展示已有报告的色谱图谱 + 新采样的色谱图谱，玩家一次拖拽确认即可，不需要逐峰解说
*   两个小玩法共享色谱曲线的渲染系统和拖拽交互代码

### 8.4 本地化

*   所有Foster/Zack对白需支持多语言配置
*   Key命名规范: `GAME_MINI_OIL_CHROMATOGRAPHY_*`
*   峰位标签: `GAME_MINI_OIL_PEAK_BASE`, `GAME_MINI_OIL_PEAK_VISCOSITY`, `GAME_MINI_OIL_PEAK_ADDITIVE`, `GAME_MINI_OIL_PEAK_FINGERPRINT`
*   结论文字: `GAME_MINI_OIL_RESULT_IDENTICAL`, `GAME_MINI_OIL_REPORT_DESC`

### 8.5 可复用性

本色谱峰位对齐交互模式可作为通用"波形比对"框架，适用于:
*   L4 油迹二方比对（简化版）
*   其他需要化学成分比对的场景
*   声纹比对（如果未来有音频类证据）

框架抽象建议: 将峰的数量、位置、高度、解说文本配置为外部数据，交互系统只负责渲染和对齐判定。
