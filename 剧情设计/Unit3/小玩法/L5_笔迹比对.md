# 玩法设计文档：笔迹比对 (Handwriting Comparison)

**文档版本**: v1.0（完整设计）
**隶属章节**: Unit 3 - 循环5
**设计目标**: 玩家亲自比对报纸批注字迹与Thomas日记/遗书字迹，通过主动圈选差异点发现笔迹完全不同——批注者有专业保险背景，而Thomas只有小学文化水平
**玩法类型**: spot_the_difference（交互式笔迹找差异 + 标记）

## 0. 前因后果 (Context & Impact)

### 前因——证据从哪来

| 证据 | 获取场景 | 获取方式 |
|------|---------|---------|
| 3501「标注报纸」(35-D-01) | L5 SC3014 Bernard办公室搜证 | 《芝加哥论坛报》1928年10月版保险版面，有大量**工整专业的保险术语批注**，含ACC.D/BEN.CHG/EXC.CLSE等Continental银行内部缩写 |
| Thomas笔迹样本（日记/遗书） | 跨循环已知信息 | Thomas只有**小学文化水平**——日记拼写错误多（"insuranse""benificiary""mony"）、用词简单口语化、笔迹稚拙不稳 |

**触发动机**: Mickey在L4结尾点出关键疑问——"一个欠债$12,000的失业酒鬼……能想出这么精密的杀人骗保计划？Thomas只有小学文化程度。" Zack在搜证中发现报纸批注明显专业，与Thomas的教育水平严重不符。笔迹比对是将这个直觉判断转化为法医证据的关键步骤。

### 后果——证据用在哪

- **直接产出**:
  - **3502 笔迹比对报告** (35-E-01) — 法医正式结论：报纸批注非Thomas所写，写作者具有专业保险知识背景
  - **35-C-01 Thomas笔迹样本对比照片** — 作为3502报告的附件（不需单独出示）
- **直接用途**: **L5 Expose R1** — Bernard谎称"Thomas来买了标准保单，我没做任何不恰当的事"
  - 出示3502（笔迹比对报告）+ 3304（Mary人寿保险单，L3跨循环引用）
  - 笔迹比对报告证明：批注笔迹(工整专业+银行缩写) ≠ Thomas笔迹(潦草+拼写错误) → **完全不同，写作者有专业保险背景**
  - Bernard被迫承认"确实在Thomas来办公室时用报纸给他讲解过"，但坚称"这是标准的客户讲解"
- **陷阱设计**: 直接出示报纸原件(3501)是**陷阱**——报纸只能证明"有人标注了"，不能证明标注者不是Thomas，更不能证明是Bernard。笔迹比对报告才是正确证据，因为它排除了Thomas。
- **报纸保留**: 3501不被消耗——后续侧光压痕分析仍需要3501作为对比参照物
- **剧情意义**: 首次将Bernard从"普通银行职员"拉入嫌疑圈，揭开银行系统性诱导的冰山一角

## 1. 玩法概述

### 1920s笔迹分析背景

1920年代，笔迹鉴定（Questioned Document Examination）已是法医学的成熟分支。Albert S. Osborn于1910年出版的《Questioned Documents》确立了笔迹比对的科学方法论。核心原则：每个人的书写习惯具有个体性和稳定性，通过字形、笔压、连笔习惯、拼写水平等维度可以判断两份笔迹是否出自同一人。

在本案中，比对维度极端明确——不是同一教育水平的人之间的微妙差异，而是**小学文化水平 vs 专业从业者**之间的鸿沟。任何受过基本训练的法医都能在30秒内给出结论。但对玩家而言，亲手找出每个差异类别的过程就是侦探体验的核心。

### 触发条件

玩家同时持有以下证据时，在Foster实验室选择"笔迹比对"触发：
1. **3501 (35-D-01)**「标注报纸」— 工整字迹、专业保险术语、银行缩写
2. **Thomas笔迹样本**（日记/遗书）— 系统隐式持有，跨循环已知信息

### 核心体验

左右并排查看两份笔迹 → 用放大镜工具检查细节 → **主动圈选差异点** → 每发现一处差异触发Zack评论 → 集齐3个类别的差异 → 确认"非同一人书写" → 生成笔迹比对报告

### 预期时长

45-90秒（取决于玩家探索速度）

### 视觉呈现要点

**左侧面板 —— Thomas日记页面**
- 纸张：泛黄粗糙的便宜笔记本纸，有折痕和污渍
- 笔迹：铅笔/廉价钢笔书写，字迹歪斜不稳，字母大小不一，行距不均匀
- 整体观感：一个勉强识字的人吃力地写下的文字，像小学生作业
- 关键文本展示：
  ```
  I owe mony to the bank.
  Cant pay it back. Mary dont
  no about the insuranse yet.
  benificiary is me. God help us.
  ```

**右侧面板 —— 报纸批注**
- 纸张：《芝加哥论坛报》标准新闻纸，保险版面，印刷体标题清晰
- 笔迹：黑色钢笔书写，字迹工整流畅，字母间距均匀，行列整齐
- 整体观感：受过良好教育的专业人士的从容书写，运笔有节奏感
- 关键文本展示：
  ```
  ACC.D coverage requires 90-day
  exclusion period. BEN.CHG must
  be filed before EXC.CLSE takes
  effect. CLM.48HR mandatory for
  ACC.D claims.
  ```
  以及报纸正文旁的圈画标注："注意90天免赔期""事故报告须在48小时内提交"

### 交互机制

**放大镜工具**
- 玩家移动鼠标/手指时，跟随一个圆形放大镜（直径约120px），放大2倍显示文字细节
- 放大镜内可以看到笔画粗细、抖动痕迹、墨水深浅等细节
- 玩家点击/长按某处文字时，触发"圈选"动作——画面上出现红色圆圈标记

**差异点圈选**
- 界面上预设若干个**可交互热区**（hitbox），分布在两侧面板的关键位置
- 每个热区对应一个差异类别（字形/用词/拼写）
- 玩家点击热区时：
  1. 红色圆圈动画标记该位置
  2. 触发对应的Zack语音评论（见第3节）
  3. 底部进度条对应类别亮起（○ → ●）
- 点击非热区位置：无反应或轻微提示音（"这里没有明显差异"）

**操作总结**
1. 自由移动放大镜探索两侧文本
2. 发现可疑差异处 → 点击圈选
3. 系统判定差异类别 → 触发Zack语音 → 进度更新
4. 三类差异全部发现 → 完成分析

## 3. 差异类别详细设计

玩家需要在三个维度各找到至少一处差异。三个类别可以按任意顺序发现。

### 类别一：字形差异 (Handwriting Form)

**Thomas侧热区位置**: 日记文字的任意一行笔画
**报纸侧热区位置**: 批注文字的任意一行笔画

**可圈选的差异点**（任选其一即可触发）：

| 差异点 | Thomas日记 | 报纸批注 |
|--------|-----------|---------|
| 笔画稳定性 | 字母线条抖动、粗细不均，像是握笔吃力 | 笔画流畅连贯，粗细均匀，运笔从容 |
| 字母大小 | 大小不一，同一个字母前后写法不同（两个"a"长得不一样） | 大小统一，同一字母写法高度一致 |
| 行距 | 行与行之间忽宽忽窄，文字逐渐下斜 | 行距均匀，即使写在无横线的报纸空白处也保持水平 |
| 字间距 | 单词之间间距不规则，有的挤在一起 | 单词间距均匀规整，专业排版感 |

**Zack圈选后语音**:
> "笔迹完全不同。Thomas写字像小学生——歪歪扭扭、大小不一。但报纸上的批注……工整流畅，像是每天都在写文件的人。"

**Foster补充**（可选）:
> "笔画的稳定性反映长期书写习惯。Thomas的笔迹显示他很少写字，握笔姿势不正确。而批注者显然受过系统的书写训练。"

### 类别二：用词差异 (Vocabulary)

**Thomas侧热区位置**: "mony""insuranse""benificiary"等简单用词处
**报纸侧热区位置**: "ACC.D""BEN.CHG""EXC.CLSE""CLM.48HR"等专业术语处

**可圈选的差异点**（任选其一即可触发）：

| 差异点 | Thomas日记 | 报纸批注 |
|--------|-----------|---------|
| 保险术语 | "insuranse"（想写insurance但拼错了）——只知道这个词存在 | "ACC.D"(Accidental Death)、"BEN.CHG"(Beneficiary Change)——Continental银行**内部缩写**，外部人员不可能知道 |
| 表达复杂度 | "Cant pay it back"——最基础的口语句式 | "ACC.D coverage requires 90-day exclusion period"——专业条款语言 |
| 概念层次 | "benificiary is me"——只知道"受益人是我" | "BEN.CHG must be filed before EXC.CLSE takes effect"——理解免责条款与受益人变更的时序关系 |

**Zack圈选后语音**:
> "ACC.D、BEN.CHG、EXC.CLSE……这些是Continental银行的内部缩写。Thomas连'insurance'都拼不对，怎么可能知道这些术语？写这些批注的人在银行系统里工作——而且工作了很久。"

**设计意图**: 这是三个类别中最有叙事冲击力的一个。银行内部缩写不是公开知识，Thomas不可能从任何外部渠道学到。这直接指向Continental银行内部人员。

### 类别三：拼写差异 (Spelling)

**Thomas侧热区位置**: 拼写错误处（"mony""insuranse""benificiary""dont no"）
**报纸侧热区位置**: 对应正确拼写处（整个批注区域零拼写错误）

**可圈选的差异点**（任选其一即可触发）：

| 差异点 | Thomas日记 | 报纸批注 |
|--------|-----------|---------|
| "money" | **"mony"** — 拼错 | 无对应词，但整篇零错误 |
| "insurance" | **"insuranse"** — 拼错（s/c混淆） | "ACC.D coverage"中隐含insurance概念，拼写完全正确 |
| "beneficiary" | **"benificiary"** — 拼错（e/i混淆） | "BEN.CHG"缩写正确，且批注中其他位置完整拼写"beneficiary"无误 |
| "don't know" | **"dont no"** — 语法和拼写双重错误 | 全文语法正确，标点规范 |

**Zack圈选后语音**:
> "'Insuranse'……'benificiary'……'mony'。Thomas连最基本的单词都拼不对。但报纸上的批注——没有一个拼写错误。一个都没有。这两份东西不可能是同一个人写的。"

**设计意图**: 拼写差异是最直观、最"铁证"的类别。教育水平的鸿沟在拼写上无法伪装——小学文化水平的人即使抄写也会犯错，而批注者的零错误率反映了系统性的专业训练。

## 4. 结果与结论

### 三类差异全部发现后

**Foster总结对话**:
> "字形、用词、拼写——三个维度全部不一致。这两份样本的书写者教育程度差距极大。Thomas Sullivan只有小学文化水平，而报纸批注的书写者……具有专业保险知识，熟悉银行内部术语体系，且受过良好的书写训练。"

**Zack**:
> "结论？"

**Foster**:
> "报纸批注绝不是Thomas Sullivan写的。写这些批注的人，是保险行业的专业从业者。"

### 确认按钮

画面底部出现确认按钮：**"这不是同一个人写的"**

玩家点击后：
1. 报告生成动画（打字机音效 + 纸张填充效果）
2. 弹出证据获得提示：
   - **获得 3502「笔迹比对报告」** — "报纸批注非Thomas Sullivan所写。写作者具有专业保险知识背景。"
3. 3501「标注报纸」保留在背包中（不消耗）

### 报告内容摘要（3502）

```
笔迹比对分析报告
报告编号：ME-1928-1120-05-HANDWRITING
日期：1928年11月23日 11:00
法医：Dr. Foster

检测对象A：报纸批注笔迹
检测对象B：Thomas Sullivan笔迹样本（日记、遗书）

【比对结论】
三项维度均完全不一致：
  1. 字形：样本A工整流畅，样本B潦草不稳
  2. 用词：样本A使用专业保险术语及银行内部缩写(ACC.D/BEN.CHG/EXC.CLSE)，
         样本B仅使用基础口语词汇
  3. 拼写：样本A零拼写错误，样本B存在大量拼写错误

最终结论：报纸批注非Thomas Sullivan所写。
写作者具有专业保险知识背景，熟悉Continental银行内部术语体系。

附件：Thomas笔迹样本对比照片 (35-C-01)
```

## 5. 特殊情况处理

### 提示系统（Hint System）

玩家在一定时间内未找到某类差异时，系统逐级提供提示：

| 时间阈值 | 提示级别 | 提示内容 |
|----------|---------|---------|
| 30秒未发现任何差异 | **Level 1 — 方向提示** | Foster: "仔细看两边的文字。不只是内容——注意笔迹本身的特征。" |
| 50秒仍缺某类差异 | **Level 2 — 类别提示** | 根据缺失类别给出针对性提示（见下方） |
| 70秒仍缺某类差异 | **Level 3 — 位置提示** | 缺失差异类别对应的热区微微发光闪烁（持续3秒） |

**Level 2 针对性提示**:

| 缺失类别 | Foster提示 |
|----------|-----------|
| 字形 | "先别看写了什么——看字是怎么写的。笔画稳定吗？大小一致吗？" |
| 用词 | "Thomas只有小学文化。看看报纸上的缩写……你在别的地方见过这些词吗？" |
| 拼写 | "Thomas拼'insurance'拼对了吗？再仔细看看他的其他单词。" |

### 误操作处理

- **点击无差异区域**: 轻微提示音 + 放大镜边缘闪红（0.3秒），无惩罚
- **重复圈选已发现的差异**: 不重复计数，Zack说"这个我已经注意到了"
- **同一类别的不同差异点**: 只要找到该类别下的任意一个差异点即可，不需要全部找到

### 退出/中断

- 玩家可随时点击"返回"退出小玩法
- 进度保存：已发现的差异类别保留，下次进入时不需要重新找
- 不强制完成：但只有完成全部3类差异后才能生成报告

## 6. 视听设计

### 音乐

- **BGM**: 沿用Foster实验室主题音乐——安静的弦乐+轻微的钟表滴答声，营造专注分析的氛围
- 音量较低，不干扰语音和音效

### 音效

| 触发时机 | 音效描述 | 设计意图 |
|----------|---------|---------|
| 进入小玩法 | 纸张展开声 + 放大镜放在桌上的轻响 | 建立"检查文件"的仪式感 |
| 移动放大镜 | 轻微的玻璃滑动声（极低音量） | 反馈移动操作，不喧宾夺主 |

| 圈选（非热区） | 低沉短促的嗡声 | 提示此处无差异，但不惩罚 |
| 找到一类差异 | 进度条填充音效 + 底部对应图标亮起 | 进度正反馈 |
| 三类全部发现 | 三音和弦解决音 + 纸张翻页声 | 完成感，过渡到结论阶段 |
| 报告生成 | 打字机快速敲击声 + 盖章声 | 仪式感，证据正式产出 |

### 镜头/画面效果

- **进入时**: 桌面俯拍，两份文件被摊开在灯下，Foster的手从画面边缘退出
- **圈选差异时**: 红色圆圈标记动画（手绘风格，像侦探用红笔画的），0.5秒完成
- **发现用词差异时**: 银行缩写术语短暂高亮发光（金色），强化"这是内部信息"的视觉冲击
- **三类全发现时**: 两份文件同时微微后退，中间浮现半透明的"≠"符号，然后淡出进入报告生成画面
- **报告生成**: 打字机效果逐行显示报告内容，最后盖上Foster的法医章

## 9. 开发备注

### 美术资产需求

| 资产 | 规格 | 优先级 | 备注 |
|------|------|--------|------|
| Thomas日记页面 | 高分辨率扫描风格，泛黄纸张+潦草笔迹 | P0 | 必须包含完整的可辨认文字，拼写错误需明确可见 |
| 报纸批注页面 | 高分辨率，新闻纸+工整钢笔批注 | P0 | 报纸印刷体+手写批注需层次分明，ACC.D等缩写需醒目 |
| 放大镜UI | 圆形放大镜，带1920s铜质边框 | P1 | 放大2x，边缘有轻微的镜片畸变效果更佳 |
| 红色圆圈标记 | 手绘风格的红色圆圈动画 | P1 | 像侦探用红笔画的，不要太规整 |
| 进度条图标 | 三个类别的小图标（钢笔/书本/字母ABC） | P2 | 底部进度条用 |

### 技术实现要点

1. **热区检测**: 使用Collider2D或RectTransform检测点击位置，每个热区绑定category标签
2. **放大镜实现**: RenderTexture方案——第二个摄像头渲染放大视图到圆形RenderTexture，跟随鼠标位置
3. **圈选动画**: Line Renderer或Sprite Animation，0.5秒完成画圈动画
4. **语音播放**: 圈选后等待画圈动画完成再播放语音，语音播放期间不阻断玩家操作
5. **进度保存**: 将已发现的类别存入MiniGameProgress数据，支持中途退出后恢复
6. **双面板同步**: 左右面板各自独立滚动/缩放，放大镜只作用于当前所在面板

### 与其他小玩法的关系

本小玩法是L5三个连续小玩法中的**第一个**：

| 顺序 | 小玩法 | 输入 | 输出 | 用于 |
|------|--------|------|------|------|
| **1** | **笔迹比对（本文档）** | **3501 + Thomas笔迹** | **3502** | **Expose R1** |
| 2 | 侧光压痕分析 | 3504 + 3501 | 3505, 3506, 3507 | Expose R2 |
| 3 | 花形压痕比对 | 3507 + 3508 | 3509 | Expose R3 |

三个小玩法之间有**严格的证据依赖链**：
- 笔迹比对不依赖其他小玩法，可独立完成
- 侧光压痕分析需要3501（笔迹比对保留了3501，因此必须先完成笔迹比对或至少保证3501不被消耗）
- 花形压痕比对需要3507（侧光分析的额外发现），因此必须在侧光分析之后

### 设计原则遵守记录

| 原则 | 遵守方式 |
|------|---------|
| 原则一：悬疑感优先 | 小玩法不替玩家说出"是Bernard写的"——只证明"不是Thomas写的"。谁写的留给Expose指证时击破 |
| 原则二：信息严密性 | Thomas小学文化+拼写错误与批注专业术语的反差基于人物设定，不存在矛盾 |
| 原则三：逻辑闭环 | 笔迹比对(排除Thomas) + 银行内部缩写(指向内部人员) + 保险单(确认利益关联) = 多源交叉指向Bernard |
| 陷阱设计 | 报纸原件(3501)是Expose R1的陷阱证据——只能证明"有人标注"，不构成对Bernard谎言的反驳 |
