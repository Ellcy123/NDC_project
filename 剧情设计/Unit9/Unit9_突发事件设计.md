## 美术呈现约束

**每一格方格漫画只能展示一个局部画面。** 不能像电影分镜那样展现全景或多人对比构图。每格的"美术画面建议"必须明确写清四件事：

1. **构图机位**：低机位仰角 / 俯视特写 / 贴脸近景 / 侧面 3/4 角度等
2. **画面内**：具体到占画面比例的元素（人物的哪个部位、道具、环境特征）
3. **画面外**（不画但要表达）：通过视线方向、肩膀一角、灯光氛围、声音文字等间接传达
4. **视觉重点**：这一格要传达的核心信息或情绪

不要写"A 和 B 的对比"这种需要全景的描述——拆成"画面只取 A，B 通过 X 暗示"。

**演出语法约定**：
- **方格漫画格子**：用于"过场角色"或"局部动作瞬间"——格子有固定位置（如画面右上方/左侧/中央偏下），可弹出/关闭/复用，伴随音效
- **立绘**：用于"在场角色"——出现在场景里、可与玩家持续交互
- **对话框**：标准 AVG 台词承载形式（NPC 立绘说话 / Zack 主角台词 / 心理旁白用[]标注）
- 同一事件中，过场角色（仅出格子）和在场角色（出立绘 + 可叠加动作格子）可并存——通过"是否出立绘"区分角色重要性与功能

**演出节拍格式约定**（事件详细设计中节奏段落使用）：
- 每个事件用"演出节拍（N 拍）"列出节拍序列；每拍标号 + 一行简述（如"节拍 3 · 失手打碎"）
- 每拍下用 markdown 列表展开，含三类内容：
  - **场景/角色台词**：如 `Zack（对话框）："..."` 或 `Emma（声音，无格子）："..."` 或 `Whale（专属对话框，电话听筒声）："..."`
  - **▸ 演出动作**：以 `▸` 起始 + 一句话——格子弹出/关闭/复用、音效、道具入背包、操作权锁/解、场景切换等所有"非台词的执行动作"
  - **设计意图**（可选）：当本拍含特殊修辞（声画错位、复用节奏、视线方向交接等）时补一行说明
- 格子用 ① ② ③ 编号；同一格子复用时延续原编号（如"酒保格子①再次弹出"）；新格子用新编号
- ▸ 演出动作中可叠加多条（如同一拍里"格子①关闭 + 音效 + 场景立绘恢复"用三行 ▸ 列出）

## 二、事件详细设计

### 事件 0 · L1 酒保拒绝 → Emma 暗语引路

**Loop / 场景**：L1 / 9004 蓝月亮酒吧大门口

**事件类型**：嵌在 L1 Opening cutscene 起始瞬间（无玩家操作）

**叙事作用**：
- 开篇第一组角色基调——确立 Zack"褪色风衣的私家侦探"与 Emma"已经混进上流圈的女记者"的反差关系
- 通过"King Oliver sent us."暗号点明蓝月亮的地下俱乐部属性（禁酒令背景）
- Emma 报暗号的从容感为后续她"主动引路"的角色定位埋伏笔

**触发条件**：游戏开场自动进入 L1 Opening；该方格漫画位于 Opening 最起始，先于 Zack/Emma 进入大堂、先于 23:30 枪响。

**角色呈现形式区分（演出语法关键）**：

| 角色 | 呈现形式 | 位置约定 |
|------|---------|---------|
| 酒保 | 仅动态漫画格子（无立绘） | 格子固定弹在 → 画面右上方——每次弹出位置不变，强化"非角色化的看门人"机械感 |
| Emma | 立绘 + 动态漫画格子 | 立绘从节拍 5 起在场景中（Zack 侧后方半步）；格子弹在 → 画面左侧 |
| Zack | 标准 AVG 主角呈现（玩家主视角） | 不出立绘、不出方格漫画；台词走标准对话框 |

---

**演出节拍（6 拍）**：

**节拍 1** · Zack 停在大门前
- 场景：蓝月亮酒吧门外台阶（静态背景，无角色立绘）
- ▸ 画面右上方弹出**酒保格子①**
- 酒保（格子①）："今晚有包厢吗，先生？"

**节拍 2** · Zack 表明来意
- Zack（对话框）："我来找 Webb 先生。他约了我今晚来。"
- ▸ **酒保格子①关闭**——伴随**关门声脆响**（音效，短促 / 金属质感）

**节拍 3** · Zack 不死心
- Zack（对话框）："喂——我说，Webb 约的我——"
- ▸ 画面右上方**酒保格子①再次弹出**——美术资产复用，台词内容完全相同
- 酒保（格子①）："今晚有包厢吗，先生？"
- 设计意图：酒保只重复同一句程式化台词、不回应——表达 speakeasy 看门人对"非熟客"的机械式拒绝；同位置同台词的复用本身是一种节奏修辞

**节拍 4** · Zack 转身要走
- Zack 沉默一拍，转身（背影朝向门）
- ▸ **格子未出，人声先到**——画面无任何格子，但传来 Emma 的声音
- Emma（声音，无格子）："别急着走。"
- 设计意图：声画错位，先听到再看到，制造 Emma 入场的悬念瞬间

**节拍 5** · Emma 出场报暗号
- ▸ **Emma 立绘出现在场景中**（站在 Zack 侧后方半步）
- ▸ 画面左侧弹出 **Emma 格子**
- Emma（格子）："King Oliver sent us."
- ▸ 画面右上方弹出**酒保格子②**——与①同位置但**新美术资产**：眉毛微挑、视线从画面左侧 Emma 方向移向门外（"会意"瞬间）
- ▸ **酒保格子②关闭**
- ▸ 音效：**三道插销依次从里面拉开**（"咔哒——咔哒——咔哒"，有节奏感 / 由内向外）
- 设计意图：暗号生效的物理凭据由音效承载（门是真的开了），不靠台词解释

**节拍 6** · 进入酒吧
- Zack（对话框）："你是谁？"
- Emma（格子，仍开着）："进来再说。"
- ▸ **Emma 格子关闭**
- ▸ 黑屏过渡 → 切换场景进入 9005 酒吧大堂

---

**美术资产清单**：

| 资产 | 数量 | 用途 |
|------|:-:|------|
| 酒保格子①（拒绝表情） | 1 张 | 节拍 1 + 节拍 3 复用 |
| 酒保格子②（会意表情） | 1 张 | 节拍 5 |
| Emma 格子（报暗号瞬间） | 1 张 | 节拍 5 弹出 / 节拍 6 末关闭 |
| Emma 立绘 | 复用立绘库 | 节拍 5 入场后持续 |
| 9004 静态背景 | 复用场景资产 | 全程 |
| → 方格漫画新美术合计 | 3 张 | |

**音效需求**：
- 节拍 2：关门声脆响
- 节拍 5 末：三道插销依次拉开（"咔哒——咔哒——咔哒"）
- 节拍 4：Emma 入场前奏（脚步声 / 衣料摩擦声等，可选）

---

**美术画面（3 张方格的构图细节）**：

**① 酒保格子（拒绝表情，节拍 1 / 3 复用）**
- 构图机位：低机位 / 仰角，画面只取酒保的上半身
- 画面内：酒保的脸（占 1/3，单眉微挑、视线向下打量）、按门的手（占 1/3，手指紧扣门框）、门缝透出的暖黄灯光与模糊人影（占 1/3，暗示门内是热闹的酒吧大堂）
- 画面外：Zack 在画面下方台阶上——通过酒保俯视的视线方向 + 画面下沿露出的 Zack 风衣肩膀一角暗示
- 视觉重点：酒保"看门狗"姿态——禁酒令时期典型 speakeasy 守门人的拒绝感

**② 酒保格子（会意表情，节拍 5）**
- 构图机位：与①同机位（保持位置一致便于美术沿用构图模板）
- 画面内：酒保的脸（眉毛微挑、视线从画面左侧 Emma 方向移向门外）、按门的手已经松开（手指离开门框，暗示即将拉开门）、门缝里的灯光更明显
- 画面外：Emma 在画面左下角——通过酒保被吸引的视线方向暗示；Zack 在画面下方——通过画面下沿一角风衣暗示
- 视觉重点：从"冷漠"到"会意"的瞬间——眉挑 + 视线方向变化是关键表情符号

**③ Emma 格子（报暗号瞬间，节拍 5）**
- 构图机位：齐胸景，画面只取 Emma 的脸侧
- 画面内：Emma 半张侧脸、嘴形微张正在低声说话（嘴边可有一缕呼出的白气增强"压低声音的私密感"）；眼神沉稳带一丝笑意，明显是熟门熟路
- 画面外：酒保不入画——通过 Emma 视线方向 + 已知她在对酒保说话的语境暗示；Zack 站在 Emma 身后——通过 Emma 肩膀后方一角风衣布料暗示
- 视觉重点：Emma 的从容——一个"我说过这句话"的熟练姿态，与酒保格子②的"会意"形成反应链

### 事件 1 · L2 Emma 接相机失手 → 柜底翻出勒索照片

**Loop / 场景**：L2 / 9006 歌舞厅小包厢

**事件类型**：⚡ 突发事件（自由探索期玩家动作触发）

**叙事作用**：
- 双产出节点——同时获得 9209 便携相机（拍摄工具）+ 9201 勒索照片（仙人跳产物）
- 相机打碎是引出柜底信封的"桥梁动作"，让"找到勒索照片"不显得过于直白——玩家是因为"蹲下捡碎片"才偶然摸到柜底信封
- 揭示蓝月亮 VIP 小包厢的真实用途（勒索拍摄角落）

**触发条件**：玩家点击矮木柜 → 看到柜中便携相机（高亮可点击但未入背包/未显示详情）→ 玩家点击相机道具触发突发事件。

**角色呈现形式区分**：

| 角色 | 呈现形式 | 位置约定 |
|------|---------|---------|
| Zack | 标准 AVG 主角呈现 | 不出立绘、不出方格漫画；台词走标准对话框 |
| Emma | 立绘 + 对话框 | 立绘从节拍 1 起在场景中持续；台词走标准对话框 |
| 方格漫画 | 用于"动作瞬间"特写 | 弹在画面中央偏下，覆盖场景下半；弹出短促，关闭后场景立绘恢复 |

**演出节拍（6 拍，精确匹配 Loop2 草稿 §2 节点 901002050–901002059）**：

**节拍 1** · 拉开矮木柜，发现相机
- 场景：9006 歌舞厅小包厢（Zack + Emma 立绘在场）
- Zack 蹲下拉开角落矮木柜——里面躺着一台落灰的便携相机
- ▸ 视觉提示：相机作为可点击道具高亮显示在矮木柜中（未入背包、未显示详情）
- ▸ 音效：木柜拉开的吱呀声
- 设计意图：此时玩家**未获得相机、也无法查看详情**——只能看到一个"可点击"的视觉钩子

**节拍 2** · 玩家点击相机（触发事件）
- ▸ 玩家点击矮木柜中的相机道具——触发突发事件
- Zack 伸手从柜中取出相机，递给 Emma 看（玩家全程未来得及打开详情）
- Zack（对话框）："这个相机还有底片。"
- Emma（对话框，伸手接过）："35mm 的便携相机？怎么会出现在这——"

**节拍 3** · 失手打碎
- Emma 接相机时失手
- ▸ 画面中央偏下弹出**格子①**——相机滑落瞬间的手部特写
- ▸ 音效：玻璃碎裂声 + 相机金属砸地脆响（同步）
- Emma（对话框，退后半步，懊恼）："这……我也……抱歉……"
- ▸ **格子①关闭**——场景立绘恢复，碎片散落地面

**节拍 4** · Emma 先看到柜底
- Emma 蹲下来收拾碎片时瞥见柜底
- Emma（对话框）："Brennan！柜子底下还藏着这个！"
- Zack（对话框）："嗯？"

**节拍 5** · Zack 抽出信封
- Zack 蹲下，手伸进柜底，指尖碰到一个硬角
- ▸ 画面中央偏下弹出**格子②**——Zack 蹲在柜底从厚信封抽出照片的特写
- ▸ 音效：信封从木柜底缝抽出的纸张摩擦声

**节拍 6** · 收拾 + 双证据入背包
- ▸ **格子②关闭**——Zack 站起身
- Zack 从口袋掏出手帕，把相机残骸 + 散落镜片一起包好放进证据袋
- Zack（对话框，无奈）："碎了也是证据。底片应该还能洗。"
- Emma（对话框，懊恼小声）："……我赔。"
- 信封中是两张暧昧合影（Vivian × Thompson / Coleman），背面有几处淡黄褐色污渍——9201 勒索照片原始未显影态
- ▸ 道具入背包：玩家获得 **9209 便携相机（包好的碎片）** + 9201 勒索照片
- 设计意图：玩家**全程未查看完整相机详情**，背包里只有"包好的碎相机"形态——这是设计上刻意保留的"事件即破坏"瞬间感

**美术画面（2 格的构图细节）**：

**① 局部：相机滑落瞬间的手部特写**
- 构图机位：齐手景，俯视略斜
- 画面内：画面正中是脱手的便携相机（落灰的机身可见、卷片轴一角）、镜头朝上反光；右侧 Emma 的两只手张开但没接到（指尖距离相机有半掌的"差一点点"距离）；左侧 Zack 一只手伸出来已经接到机身但来不及接到一片碎玻璃——一片镜片碎玻璃斜划出画面边缘
- 画面外：两人的脸不入画——通过手部紧张的张力 + 相机滑落的运动模糊轨迹，让观众脑补 Emma 那句"这……我也……"的慌乱
- 视觉重点：失手的瞬间"差一点点"的停格感——相机即将砸地但还未砸到的张力

**② 局部：Zack 蹲在柜底从厚信封抽出照片的特写**
- 构图机位：低机位 / 侧斜角，模拟蹲下时的视角
- 画面内：画面左下角是地上散落的玻璃碎片（占 1/4，作为"为什么会蹲下"的因果暗示）；画面正中是 Zack 一只手伸进矮柜底部、另一只手已经把厚信封抽出来一半——信封口微开，露出黑白照片的一角（背面有几处淡黄褐色污渍，但具体内容看不清）
- 画面外：Emma 也蹲下来——通过 Emma 的一只手伸进画面右上角（指着信封）+ 一句对白框"Brennan！柜子底下还藏着这个！"暗示她在 Zack 旁边
- 视觉重点：信封从隐蔽处现身的"翻出真相"瞬间——以"看不清照片内容"留悬念给后续 9201 分析

### 事件 2 · L2 Tommy 抽屉弹壳惊鸿一瞥

**Loop / 场景**：L2 / 9010 Tommy 办公室

**事件类型**：⚡ 突发事件 + 证据预曝光（道具不入背包，留待 L5）

## 三、后续维护

- 任一突发事件设计有改动，先更新本文档（清单 + 详细设计），再去改 state / 草稿
- 美术资源单（构图、表情、关键道具列表）如需独立成文，建议放在 `美术对接文档/` 下，并从本文档反向引用
- 本文档不替代 state 文件——state 是配置生成源；本文档是"突发事件总览 + 美术呈现规格 + 角色分工说明 + 演出节拍"
- 新增突发事件时，按"事件 N · {Loop} {简述}"标题展开，字段顺序固定为：Loop/场景 → 事件类型 → 叙事作用 → 触发条件 → 角色呈现形式区分（如适用）→ 角色分工与动作（如适用）→ 节奏 / 演出节拍 → 美术资产清单（如有格子复用）→ 美术画面 → 音效需求（如适用）→ 台词建议
- 涉及"格子弹出/关闭/复用 + 立绘 vs 格子 + 音效驱动"的事件（如事件 0），用"演出节拍"格式逐拍展开，每拍标明：场景状态 / 格子动作（弹出/关闭/复用）/ 角色台词 / 音效；其他单纯打断式事件用普通"节奏"格式即可
