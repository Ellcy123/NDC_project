## 三、每 Loop 疑点 / 碎片设计

疑点 ID 格式：`{unit}{loop}{seq}` = `10{loop}{seq2位}` 共 6 位。

### L1 疑点设计

**本 Loop 揭示的真相层**：死者不是 Margaret；Morrison 在误导调查。

**进度条分母**：3 个疑点，100% = 3/3

---

**疑点 10101**
```
id: 10101
isFragment: false
text: "尸体的手上没有戒指——Margaret 从不摘婚戒，这说得通吗？"
condition:
  - type: 1, param: 10101  （持有"尸体手指照片"）
  - type: 1, param: 10102  （持有"Margaret 相册"）
loop: L1
```
推理路径：手上无戒指（照片）+ 相册证明 Margaret 30 年从不摘 → 死者不是 Margaret 或戒指在别处。
此疑点 condition 使用 10101（L1 R1 指证用）和 10102（L1 R1 指证用），时序合规。

**疑点 10102**
```
id: 10102
isFragment: false
text: "首饰盒的戒指凹槽是空的，而现场废墟里也没有找到金属残留——戒指去哪儿了？"
condition:
  - type: 1, param: 10103  （持有"空的首饰盒"）
  - type: 1, param: 10104  （持有"现场地面照片"）

```
推理路径：首饰盒空（戒指不在家里）+ 现场无金属残留（戒指不在废墟里）→ 戒指不见了，需要解释。
此疑点 condition 使用 10103（L1 R1 指证用）和 10104（L1 R2 指证用），时序合规。

> 注：10101 和 10102 分别作为 L1 R1 和 L1 R2 的证据基础，一对一挂载严格分离：10101（尸体照片）+ 10102（相册）→ 疑点 10101；10103（首饰盒）+ 10104（地面照片）→ 疑点 10102。

## 四、每 Loop 指证击穿逻辑

### L1 指证：Morrison（3 轮）

**指证场景**：警局 Morrison 办公区

---

**Round 1：死者身份谎言**

谎言：`1010001`——"死者应该就是 Margaret，基于现场情况判断"
击破证据：ItemStaticData 10101（尸体手指照片）+ 10102（Margaret 相册）
推理链：手上没戒指（照片）→ Margaret 从不摘婚戒（相册）→ 死者不是 Margaret

陷阱选项设计：
- 陷阱：10105（鞋坊营业记录，证明 Frank 是雇员）
- 看似相关理由：玩家可能觉得"证明 Frank 在这里出现过"能帮助指向死者身份
- 实际不成立：营业记录只证明 Frank 有资格进入鞋坊，不能否定"死者是 Margaret"这一具体谎言；击破需要的是"死者的具体特征与 Margaret 不符"，营业记录做不到这一点

---

**Round 2：戒指销毁谎言**

谎言：`1010002`——"火灾很猛烈，戒指可能掉在现场或被烧毁"
击破证据：ItemStaticData 10104（现场地面照片）
推理链：黄金熔点 1064°C，普通建筑火灾约 600-800°C 达不到；废墟无金属残留 → 戒指根本不在现场

陷阱选项设计：
- 陷阱：10103（空首饰盒）
- 看似相关理由：首饰盒是空的，似乎印证"戒指不知道去哪了"
- 实际不成立：空首饰盒能证明戒指不在家里，但不能反驳"戒指在现场被烧毁"这个具体说法；需要的是现场物证证明没有金属残留

---

**Round 3：尸检信息谎言**

谎言：`1010003`——"我有尸检调查信息可以证明死者身份"
击破证据：Testimony 1030001（Foster 通过 Emma 转述"尸检最快明天，连性别都未定"）
推理链：Foster 是案件的法医，她说明天才能出报告 → Morrison 现在根本不可能有尸检信息 → 在虚张声势

陷阱选项设计：
- 陷阱：10106（卧室厨房失火点照片）
- 看似相关理由：Morrison 编造的死因是"在厨房做饭失火"，这张照片似乎能戳穿这个死因
- 实际不成立：当前谎言不是关于死因，而是关于"能否证明死者身份"；照片能质疑死因但无法击破"我有尸检信息"这一维度的谎言。（死因质疑留给 L6）

**证据映射到疑点**：
- 10101 + 10102 → 疑点 10101
- 10104 → 疑点 10102
- 1030001 → 疑点 10103

## 七、大纲已知风险的推理评估

### R9：对账单页眉/正文矛盾过于刻意

**原风险描述**：客户账户对账单"页眉投资公司 vs 正文银行"的内部矛盾过于刻意，现实银行不会犯此低级错误。

**推理策划评估**：

这个证据是 L3 R3 的终极击破件，承担的推理功能是"自证其罪"——Moore 无法辩解（因为这是他自己的文件）。从推理设计角度，这个证据是高效的，但叙事合理性确实是个问题。

**替代方案 A（保留现有结构，改变解释）**：
不是"银行内部犯了错误"，而是"这份文件本来是内部文件，不该被客户看到"。设定：档案室里的对账单是银行内部版本（包含投资公司信息），不是发给客户的版本（客户版只写银行）。Leonard 误将内部对账单归档在客户文件夹里（因为他同时处理两个账本）。这样"矛盾"变成"内部文件泄露了双重身份的秘密"，不是银行的管理失误，而是 Leonard 的文件管理疏失。
优点：证据推理功能保留，叙事合理性大幅提升；缺点：需要在 state 中明确这一设定，让 Leonard 的陈述（L3 银行大厅证词）补一句关于"内部文件和客户文件分开存档"的背景信息。

**替代方案 B（更改物证类型）**：
将对账单改为"内部备忘录"（Internal Memo），内容是 Moore 向 Leonard 发出的业务指令，同时使用了两个抬头（Lakeshore Investment Co. 和 Lakeshore Trust Bank）作为"业务模板"。玩家发现的是这份内部备忘录，而非对外的文件。
优点：更真实，内部文件完全可能有双重抬头。缺点：需要改写物品名称和描述。

**推理策划建议**：采用替代方案 A，代价最小，推理功能完整保留。在 10306（客户账户对账单）的描述中加入"内部存档版本"说明，并在 Leonard L3 大厅证词中预埋"我们的内部文件和对客文件分开管理"的背景。
