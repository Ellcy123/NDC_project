# 玩法设计文档：连号钞票拼图 (Banknote Serial Puzzle)

**文档版本**: v1.0
**隶属章节**: Unit 10 - 循环 2
**设计目标**: 让玩家把两位看似无关的人手里的钞票排在一起，亲眼看出"O'Hara 缺的就是 Vinnie 手里有的"——一笔银行新钞的物理路径就此暴露。

---

## 0. 前因后果

**前因——证据从哪来**

L2 玩家在两个地点拿到两批钞票：
- **10202 O'Hara 贷款钞票**（在 10007 街角，O'Hara 用 5$ 还给 Frank 的还款钞）——五张连号新钞 900B / 901B / 902B / 903B / 904B，序列号 900B–909B 中缺失 **905B / 906B / 907B / 908B / 909B**（O'Hara 实际只拿了前五张）。
- **10206 Vinnie 的钞票**（在 10015 Silver Moon 赌场，Tony 把 Vinnie 上次结账留下的零钱给 Zack 看）——三张钞 905B / 906B / 908B。

两份钞票编号上的关系不靠玩家自己算编号——靠把两摞钞票排进同一条"号段轨道"，缺号补齐的画面会直接告诉玩家。

**后果——证据用在哪**

完成拼图后，10202 + 10206 双件直接进入指证：

- **L2 Expose R2**：击穿 Leonard 谎言 2（2050002）——"我们只是认识，没有什么来往"。O'Hara 偿还的银行新钞流出银行后落到了暴力催债人 Vinnie 手里——Leonard（银行）和 Vinnie（黑帮）之间存在系统性资金往来，不是"只是认识"。
- 钞票拼图本身**不产出派生证据**——10202 和 10206 是 R2 的双件击穿件，玩法只是把"两件原物的关联"可视化。

**疑点 10202**（"O'Hara 缺的钞票怎么会在 Vinnie 手里？"）的解锁 condition 也是 10202 + 10206，玩法完成时疑点同步解锁。

---

## 1. 玩法概述

- **玩法类型**: banknote_serial_alignment（钞票编号排列）
- **核心体验**: 把两摞散乱的钞票拖进同一条号段轨道，缺号位置被另一摞精准填上——画面告诉你"是同一批"，不需要对白解释。
- **预期时长**: ~20–30 秒
- **触发条件**: 持有 10202 **且** 持有 10206，在 CASE BOARD 选择两件钞票"对照"
- **失败惩罚**: 无。玩法本身是引导式的——错位拖动会被吸附回正确位置，不存在"拼错"。

---

## 2. 1920s 考据

1920 年代美国银行新钞按生产批次连号发出，**冠字（letter prefix）+ 三位数字**是固定编号格式（例如 900B、901B……）。一笔新钞被银行从金库取出后，通常会在同一批被付给同一位客户——这是 1920s 美国银行业的常识。

侦探如果想追一笔钱的物理路径，最朴素的办法就是**比对序列号**：当两位看似无关的人手里的钞票编号呈现"我缺你有"的连号关系，几乎可以断定钱出自同一批、在短时间内被同一个人分散花出。

这种"连号追钱"在 1920s 实际办案里被用于打掉抢劫赃款的洗钱链，也是芝加哥地方报纸常报道的"侦探破案"细节。Zack 用得理直气壮。

## 3. 交互设计

```
【界面布局】
  屏幕中央：一条横向"号段轨道"，刻度从 900B 到 909B（十格，每格能放一张钞票）
  屏幕左下：10202 的五张钞票（900B / 901B / 902B / 903B / 904B），略微散乱叠放
  屏幕右下：10206 的三张钞票（905B / 906B / 908B），同样散乱叠放
  屏幕右上：905B、906B、908B 三个号段的"位置剪影"暗示——玩家拖动 10206 时会高亮

【操作】
  阶段 1（自动）：10202 五张钞票从左下自动飞入号段轨道前五格（900B–904B），动作约 1 秒
                  飞入后号段轨道上五格高亮金边，剩余五格（905B–909B）留空
  阶段 2（手动）：玩家把 10206 的三张钞票依次拖进号段轨道
                  拖动时钞票顶部显示编号，对应空格的位置高亮金边
                  正确位置吸附（自动磁吸 ±50px）
                  905B / 906B / 908B 三格被填上
  阶段 3（自动）：907B / 909B 两格依然空缺——但 908B 和左右相邻的 905B/906B 已经形成连号
                  整条号段轨道做一次"扫光"动画（约 0.6 秒，从左到右一道金光）
                  扫光过后整条号段被光晕笼罩，玩家肉眼可见连号关系

【完成判定】
  10206 三张全部就位即触发完成，无需额外确认
  Zack 台词 + 疑点 10202 解锁提示 + CASE BOARD 自动记录关联
```

Zack（看着轨道上对齐的钞票）：
> "O'Hara 还给 Frank 的钞票是 900 到 904，Vinnie 手里的是 905、906、908——这是同一捆从银行出来的连号钞。它经过 Leonard 的手付给了 Vinnie。"

## 4. 结果与产出

**产出证据**: 无（不生成派生证据）

**触发效果**:
- 疑点 10202 解锁（"O'Hara 缺的钞票怎么会在 Vinnie 手里？"）
- 10202 和 10206 在玩家背包内**自动标记关联**——之后在 CASE BOARD 出示其中一件时，另一件会高亮提示"配对件"

**指证用途**:
- L2 Expose R2：10202 + 10206 双件并排出示 → 击穿 Leonard 谎言 2"我们只是认识，没有什么来往"。资金链路从银行 → Leonard → Vinnie 闭合，私交说法不成立。

## 5. 美术 / 音效需求

美术：
- **号段轨道**：横向长条 UI，每格刻度上方印着"900B–909B"号段标签；空格淡灰色，填入钞票后高亮金边
- **10202 钞票样式**：1920 年代美元（金色 5$/10$ 面值不影响玩法，外观素净；冠字编号位置印刷清晰）；五张略带使用磨损感
- **10206 钞票样式**：与 10202 同款印刷规格（同批次），但钞面无明显折痕——刚从 Vinnie 钱夹拿出
- **连号扫光动画**：金黄色光带从左向右扫过号段轨道，过后整条轨道带淡金色光晕（持续约 1.5 秒）
- **磁吸提示**：钞票拖动时，正确位置周围出现淡蓝色光圈（非 UI 文字，纯环境光提示）

音效：
- 钞票自动飞入：轻微"嗖"声 + 落位时纸张接触台面的轻叩
- 拖动钞票：纸张摩擦微响
- 吸附完成：每张正确就位时一声短促"嗒"
- 连号扫光：连续低音"嗡"+ 一道金属共鸣（约 0.6 秒）
- 完成定格：一声轻响 + Zack 台词淡入
