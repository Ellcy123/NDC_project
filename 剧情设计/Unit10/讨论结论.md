## 一、整体变更概述

### 1.1 保留的设计

- 大纲 v5（2026-05-13）的 6 Loop 主结构、指证对象、指证轮次、三重叙事反转、镜像主题、跨章 Whale 伏笔——**全部保留**。
- L1 Morrison（3 轮）→ L2 Leonard（3 轮）→ L3 Moore（3 轮）→ L4 Danny（3 轮）→ L5 Vinnie（1 轮）→ L6 Leonard（4 轮）的击穿序列保持不变。
- L5 律师/侦探二选一分支 + 三幕坏结局保留。
- L6 诱导招供系统（诱导槽 0-100% + 三节点话术选择 + QTE）保留为核心创新机制。

### 1.2 本轮补案（必须落到 state）

- 证据 ID 全面分配（采用推理策划版本为基准，本节 V/X 节有局部修订）。
- 疑点/碎片完整设计（17 疑点 + 1 碎片 + 1 父疑点，共 19 项 DoubtConfig 条目）。
- 跨 Loop 一对一冲突的解决方案（Vinnie 假认罪原话拆为 5080002a / 5080002b 两条独立证词）。
- 父疑点 10401 的合并机制（L2 碎片 10204 + L3 新证词 3060004 → L4 父疑点）。
- 风险点 W1（4090004 Danny 目击证词作为 L5 指证的"指证后知识"无疑点 condition 引用）的处理方案——**重新挂载到 L5 疑点 10501**，见 V 节裁决。
- R9（对账单页眉/正文矛盾）按推理策划替代方案 A 处理，并在 Leonard L3 大厅证词中预埋背景。
- 贷款数学校准（P3 风险）。
- 字条载体（N3）确定为可被检视的演出物，不进推理证据链。
- Leo 妻子（N6）确定为视觉演出角色，无对白。
- 镊子+碎片"用物品交互物品"的实现走 ItemStaticData canCombined 复合方案。

### 1.3 与 Unit9 的衔接

- **Emma**：Unit9 已登场，是 Zack 的朋友（不是助手），Unit10 内沿用 Unit9 NPCStaticData 的 ID。Unit10 内她以"朋友介入帮忙"的姿态参与调查，L3 后跟法医脱离一段时间。
- **Mickey**：Whale = Unit9 已露名但未正面登场的反派。Unit10 是 Mickey 这个"律师外壳"的首次正面登场，**Unit10 内不暴雷**。Mickey 立绘和声纹可独立设计，Unit11+ 揭示真身时使用"卸下伪装"反差呈现。
- **Foster**：Unit10 首次登场，由 Emma 引荐，Unit9 无前史。
- **Morrison**：跨 EPI01/EPI09/EPI10。EPI01 栽赃 Zack 的旧仇是 L1 误导动机的根。沿用既有 NPCStaticData ID。
- **Zack 事务所位置**：南区与北区交界——贴近 Margaret 鞋坊（步行 10 分钟）、远离 Moore 银行总部（电车 20 分钟）。此设定支撑 L1 末尾 Mickey 发电报、L5 末尾 Zack 给 Foster 打公用电话的物理距离。

## 五、关键设计问题与裁决

### 5.1 推理策划 Q1（P0）：跨 Loop testimony 引用机制

**争议点**：指证 testimony 能否引用非当前 Loop 疑点 condition 的证词？

**判断依据**：`docs/游戏系统/核心玩法/疑点系统.md` 第六节硬约束："每一个指证步骤所需的道具/证词，必须至少被某个疑点或碎片的 condition 引用。"

**裁决**：不豁免。4090004 必须挂在疑点 condition 中。

**下一步**：将 4090004 挂入 L5 疑点 10501 的 condition（10501 由 2 件改为 3 件 condition）。时序合规：4090004 在 L4 末取得，疑点 10501 在 L5 出现。

### 5.2 推理策划 Q2（P0）：贷款数学一致性

**裁决**：改为"年利率 12% 单利 + 月管理费 50$"。5 年合计 ≈ 3,960$（与当前欠款 3,200$ 合理接近）；折合年化（含管理费）112%，远超伊利诺伊州法 36% 上限——违法特征保留。

### 5.3 推理策划 Q3 + 叙事策划 Q3：Vinnie 字条性质（合并裁决）

**裁决**：
1. 字条仅为情感载体，不进推理证据链——不分配 ItemStaticData ID
2. 字条出现时机为 R4 后、Lula 悼念前——Foster 在指证完成后到场，递交字条
3. 字条文本保留四句中文版本
4. 前序铺线：L2 Tony 试探 Vinnie 时 Vinnie 否认用力 + L3 童年合照 10310 + L6 空相框 10611

### 5.4 推理策划 Q4：R9 对账单矛盾过于刻意

**裁决**：采用替代方案 A（保留现有结构，改变解释）。
- 10306 描述明确为"内部存档版本对账单"
- Leonard L3 大厅证词 3050001 预埋背景
- Leonard 误将内部对账单归档在客户文件夹里（管理疏失暴露）

### 5.5 推理策划 Q5 / 原则二 P5：钞票缺号设计

**裁决**：Tony 证词 2070001 补一句："上周给姑娘点烟那天，他还花掉了两张新票"——解释 907B/909B 去向，账目闭环。

### 5.6 系统策划 P3：字条载体

**裁决**：合并 5.3——字条不进 ItemStaticData，仅演出。

### 5.7 系统策划 P8：镊子+碎片"用物品交互物品"逻辑

**裁决**：采用 canCombined 模式包装。
- 10605 设置 canCombined=true，combineParameter=[10605, 10603]
- 玩家"发现地板缝隙有反光"，需持有 10603 才能取出 10605
- 等价于"合成"——输入"地板缝隙+镊子"，输出"碎片实物"

### 5.8 系统策划 P10：钥匙登记

**裁决**：10308/10309/10408 三件钥匙独立登记为 ItemStaticData，itemType=key。

### 5.9 叙事策划 Q2 / 系统策划 P4 / 大纲 N6：Leo 妻子

**裁决**：完全哑角。
1. 端茶上楼（进场时）
2. Leonard R4 招供前望向楼梯时，Zack 视线跟着 Leonard，看到楼梯顶端她的裙角，然后她消失

不分配独立 NPCStaticData，作为 1017 场景的"环境叙事 NPC"。

### 5.10 叙事策划 Q4：Foster/Emma 的 Miller 集团伏笔密度

**裁决**：间接暗示。
- L1 Emma 转述 Foster 时主动说"她受到 Miller 集团的资助"
- L4 Emma 在 1006 透露"医院是 Miller-funded"
- 此外不再增加 Miller 直接提及

### 5.11 叙事策划 Q1：Vinnie 字条语言

**裁决**：保持中文（与游戏整体一致）；声纹的简短粗糙质感落到对白作者编写指令里。

### 5.12 叙事策划 Q5：律师坏结局末白视角

**裁决**：末白"这个城市需要的不是律师"是第三方叙述视角。

### 5.13 叙事策划 Q6：Danny 种族歧视处理

**裁决**：暗示而非明写。Danny 用"勾引""骗子"等词，语气和语境让玩家自己读出种族维度。

### 5.14 系统策划 P1（P0）：跨章 NPC 复用

**裁决**：Morrison / Emma 复用 Unit9 NPCStaticData ID。Foster / Mickey / Leonard / Moore / Vinnie / Lula / Margaret / Tony / O'Hara / Danny 按 Unit10 新建。

Mickey 在 NPCStaticData 上是新条目（与 Unit9 的 Whale 是不同 NPCStaticData 实例，Unit11+ 再处理身份合并）。

### 5.15 系统策划 P2（P0）：场景 ID 格式

**裁决**：暂用 4 位格式 1001–1017。如果与 Unit1 4 位场景 ID 冲突，升级为 5 位。升级到用户拍板（U-P0-1）。

### 5.16 系统策划 P5：错误选项是否清零诱导槽

**裁决**：不减少也不增加。保证玩家最终能到达 100%，避免卡关。错误选项的"惩罚"是叙事层面的不适感。

### 5.17 系统策划 P6：律师坏结局回到选择点

**裁决**：坏结局末尾跳回选择 talk_id，存档点设在选择之前。需程序确认 Talk 链 ParameterInt 回指功能。

### 5.18 系统策划 P7：赌博小玩法跨章持久化

**裁决**：Unit10 内仅做单 Loop 入口，跨章持久化降级为"未来需求"。

### 5.20 推理策划 Q9：5080002a / 5080002b 拆分实现

**裁决**：在对话 JSON 中实现为同一段对白的两句连续台词，分别打上不同 testimony_id。两句之间用对话系统的句段分隔（不同 talkitem）。

### 5.21 推理策划 Q6 / 原则二 P6：Leonard 电话记录技术合理化

**裁决**：10608 来源改为"Vinnie 公寓接听端的电话账单（Leonard 留副本作为对 Vinnie 的把柄）"。

### 5.22 原则二 P1：戒指熔点

**裁决**：在 10104 现场地面照片描述中明确"勘验现场地面已细筛，无任何金属残留物（包括戒指、衣物金属配件等）；火灾烟尘高度未达熔金温度"。同时补一条 O'Hara/Morrison 的环境证词支撑（火势有限，由消防迅速扑灭）。

### 5.23 原则二 P2：眼镜碎片偏黄

**裁决**：将所有涉及描述统一改为"烟熏黄色"。
- 10605 描述：右眼旧镜片，火灾烟尘留下烟熏黄色着色
- 10610 眼镜维修单据描述：左眼新换镜片（透明），右眼旧镜片（未换）

### 5.24 原则二 P4：法医鉴定报告措辞

**裁决**：10601 法医报告措辞改为"头骨伤口形状与圆底钝器一致，烛台底部吻合（不排除其他同类工具）"。

### 5.25 原则二 P7：戒指脱落物理合理化

**裁决**：综合两层处理：
1. 10103 空首饰盒描述补一句"凹槽上方有今天临时取出过的轻微划痕"
2. Danny 目击证词 4090004 在 L4 末附加一句"看见戒指从她手旁滚出来反光"

### 5.26 推理策划：L5 单轮指证的设计合理性

**裁决**：保留 L5 单轮，明确豁免理由："Vinnie 是 Frank 已死后才到场的从犯，他有充分理由快速认罪止血，且 L6 会继续追问。"
